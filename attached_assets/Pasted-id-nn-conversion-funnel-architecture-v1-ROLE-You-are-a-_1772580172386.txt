ROLE
You are a senior growth engineer + full-stack product architect implementing NurseNest’s end-to-end conversion funnel and retention loop.

MISSION
Build a complete, measurable, and automated funnel that connects:
- Free 50-question trial
- Results report + readiness bands
- AI Study Path Generator
- Remediation sessions
- Adaptive CAT engine
- Premium Exam Console
- Subscription + BNPL checkout
- Lifecycle emails
- Ongoing retention + upsell

Non-negotiables:
- Do NOT claim to replicate official exam algorithms; use “adaptive measurement principles.”
- Keep US vs Canada content separate when relevant (units/scope/blueprints).
- NP is never “NCLEX-style.”
- Preserve existing roles/paywalls.
- Everything is event-driven, logged, and A/B testable.

========================================================
A) FUNNEL USER JOURNEYS (PRIMARY FLOWS)
========================================================

FLOW 1 — Anonymous → Trial → Paid
1) User lands on track page (e.g., /rex-pn, /nclex-rn, /np-canada, /allied/mlt)
2) CTA: “Start Free 50-Question Trial”
3) Email verification gate
4) Trial runs (15Q diagnostic + 35Q simulation)
5) Results report + readiness band
6) AI Study Plan preview (Week 1 visible; rest gated if not paid)
7) Upgrade screen → Checkout (Stripe + BNPL)
8) On success: unlock full study plan, remediation, CAT mode, exhibits, deep rationales

FLOW 2 — Trial incomplete → Nudge → Finish → Convert
1) User starts trial, leaves before completion
2) Trigger email at 1h + 24h
3) Resume link deep-links to /trial/question
4) On completion: same conversion steps as Flow 1

FLOW 3 — Paid → Retention Loop
1) User completes sessions
2) Weekly readiness check-in
3) Adaptive CAT retest every 7 days
4) Smart Study Plan updates automatically
5) Reactivation triggers for inactivity
6) Upsell to higher tier (CAT + premium reports) if on basic

========================================================
B) CORE FEATURES TO CONNECT
========================================================

1) FREE 50-QUESTION TRIAL (Conversion Driver)
- 50 questions total per examKey
- 15 diagnostic + 35 simulation
- Save/resume, progress meter, optional timer
- Explanations policy (BEST DEFAULT):
  - During trial: partial explanation only
  - After completing all 50: unlock full trial explanations + full results report
- Anti-abuse: email verify + device fingerprint + rate limits

Routes:
- /trial
- /trial/start?examKey=
- /trial/question/:attemptId
- /trial/results/:attemptId
- /trial/upgrade/:attemptId

2) RESULTS REPORT (Authority + Conversion)
Must show:
- Readiness Band: Not Ready / Borderline / Ready / Strong
- Confidence/Stability score (based on SE and consistency)
- Domain breakdown vs blueprint weighting
- Difficulty exposure summary
- Time management breakdown
- “What to do next” personalized actions

Premium preview:
- Show “Full report locked” components blurred (or collapsed) when not subscribed

3) AI STUDY PATH GENERATOR (Retention + Conversion)
Trigger on:
- trial completion
- mock completion
- readiness/CAT completion
- subscription purchase

Inputs:
- examKey, region, exam date, domain scores, weak topics, pace

Outputs:
- 6-week default plan (or dynamic weeks until exam date)
- weekly plan with:
  - lesson modules
  - question targets
  - flashcards
  - 1 mock/CAT checkpoint
- daily “Next session” button

Gating:
- Free users see Week 1 only + upgrade CTA
- Paid users get full plan + auto-updates

4) REMEDIATION PATH (Behavior change)
Button everywhere:
- “Generate Remediation Path (15 Q)”
Rules:
- Use tags from missed items + weak domains
- Difficulty ±1
- Avoid repeats
- Create new attempt with remediation label

5) PREMIUM EXAM CONSOLE + CAT ENGINE (Product superiority)
- Console layout (left question, right explanation/exhibit/notes)
- Exhibit viewer tool (zoom/pan/fullscreen/labels)
- Strike-through and highlight
- Keyboard shortcuts
- CAT engine uses adaptive measurement principles, blueprint weighted selection, stopping rules
- Never claim official algorithm match

6) CHECKOUT + BNPL (Revenue lift)
- Stripe checkout with:
  - cards
  - Klarna/Afterpay (CAN + US)
  - Affirm (US)
- Show installments on pricing pages if price ≥ $99
- Post-checkout redirect to “Unlocked” success page showing study plan

========================================================
C) PAYWALL + PACKAGING (DEFAULT RECOMMENDED)
========================================================

Plans:
- Basic QBank
- QBank + CAT
- Premium (CAT + deep rationales + exhibits + remediation + study plan)

Free:
- trial only + limited practice page access

Unlock rules:
Basic:
- standard practice mode + limited reports
QBank+CAT:
- CAT mode + full reports
Premium:
- CAT + exhibits + “why wrong” diagnostics + study plan + generate similar questions + visuals

BNPL:
- enabled for QBank+CAT and Premium

========================================================
D) EVENT TRACKING (MANDATORY)
========================================================

Create event schema and log everything for analytics:

Events:
- page_view(trackKey)
- trial_start(examKey)
- trial_progress(questionsUsed)
- trial_complete
- results_view
- study_plan_generated
- remediation_generated
- pricing_view(plan)
- checkout_start(plan, method)
- checkout_success(plan, bnplUsed)
- subscription_active
- session_complete(type=practice|mock|cat|remediation)
- inactivity_7d
- reactivated

Store:
user_id, examKey, region, timestamp, metadata

Admin metrics:
- trial → completion rate
- completion → upgrade rate
- upgrade by plan
- BNPL usage
- retention (week 1, week 4)
- study plan engagement
- remediation usage

========================================================
E) LIFECYCLE EMAILS (AUTOMATED SEQUENCES)
========================================================

Email triggers:
- signup
- trial started (not completed)
- trial completed (not upgraded)
- trial completed (upgraded)
- weekly study plan check-in
- inactivity 7 days
- renewal reminders

Sequences (minimum):
1) Trial Started (incomplete)
- +1 hour: resume link + “finish in 20 min”
- +24 hours: value reminder + resume
- +3 days: “get your readiness score”

2) Trial Completed (not upgraded)
- Day 0: results summary + study plan preview
- Day 2: weak areas + 20-min remediation recommendation
- Day 4: feature spotlight: CAT + exhibits
- Day 7: trial expiration + upgrade prompt (no scare tactics)

3) Subscriber onboarding
- Day 0: how to use study plan + next session CTA
- Day 3: how remediation works
- Day 7: first CAT readiness check

Rules:
- personalize by examKey + region
- never promise passing
- one CTA per email
- preference center/unsubscribe
