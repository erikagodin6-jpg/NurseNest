Redesign the NurseNest frontend UI to feel like a premium, high-trust exam preparation platform similar in polish and clarity to leading medical/nursing study tools, while fully preserving the NurseNest soft clinical pastel brand.

This is not a full rebrand. Keep the NurseNest identity feminine, modern, calm, and professional. The result should feel more premium, more credible, and more conversion-focused, with stronger hierarchy, cleaner layouts, better spacing, and more confidence-building design patterns.

Work on the published production site only. Do not point anything to development database. Do not break current auth, subscriptions, question delivery, lesson rendering, flashcards, mock exams, admin, or billing flows.

CORE GOAL

Increase user trust, perceived value, and subscription conversion by making the platform look more like a polished exam platform and less like a generic educational website.

DESIGN DIRECTION

Target feel:
• premium
• clean
• clinically professional
• modern SaaS
• calm and focused
• high trust
• optimized for nursing learners

Keep:
• NurseNest pastel/lilac/soft clinical aesthetic
• rounded corners
• gentle feminine-modern brand identity
• soft but not childish design language

Avoid:
• cartoonish visuals
• clutter
• oversized icons everywhere
• too many colors competing
• flat amateur card layouts
• excessive gradients
• low-contrast tiny text
• playful app-like gimmicks in study content

--------------------------------------------------
PART 1 — GLOBAL VISUAL SYSTEM UPGRADE
--------------------------------------------------

Refine the design system across the site.

1. Typography
Use a premium academic SaaS typographic hierarchy:
• primary font should remain DM Sans or closest current equivalent
• stronger hierarchy between page title, section heading, card heading, body text, labels, and microcopy
• improve line height and spacing
• reduce visual heaviness in dense learning areas
• use lighter font weights where appropriate, especially in educational image labels and supporting UI text
• ensure buttons and navigation labels are clean and modern, not chunky

Create a clear scale such as:
• Hero heading
• Section heading
• Card title
• Body text
• Secondary text
• Caption/meta text

2. Spacing and layout
Increase whitespace and visual breathing room throughout:
• standardize padding on cards
• improve vertical rhythm between sections
• align cards and content blocks to a consistent grid
• reduce crowded feeling on dashboards and lesson pages
• improve mobile spacing especially on dense screens

3. Color system
Refine current theme usage:
• keep pastel clinical palette
• use one primary accent per theme
• add more neutral balance with whites, off-whites, and soft greys
• ensure premium contrast for text and controls
• use color more intentionally for state and emphasis, not decoration

4. Surfaces and depth
Upgrade cards and containers:
• subtle premium shadows
• softer borders
• more consistent radius system
• distinct surface hierarchy between page background, cards, active cards, and overlays
• avoid flat amateur boxes

--------------------------------------------------
PART 2 — HIGH-CONVERSION HOMEPAGE / HERO IMPROVEMENTS
--------------------------------------------------

Redesign the homepage hero and top sections to increase credibility and conversion.

1. Hero section
Create a premium hero that clearly communicates:
• who the platform is for
• what it helps users achieve
• why it is better than generic study tools

Include:
• stronger headline hierarchy
• concise high-value subheading
• primary CTA
• secondary CTA
• trust/supporting copy
• cleaner image or screenshot showcase area

2. Hero proof section
Create a polished screenshot showcase or carousel using existing NurseNest screenshots if available.

The screenshot area should:
• look premium and not cluttered
• use framed mockups or polished browser/device containers
• include short benefit-focused captions
• highlight question bank, flashcards, analytics, mock exams, rationales, progress tracking, and lesson quality

3. Trust-building section
Add a section near the top of the homepage with trust signals such as:
• comprehensive question bank
• realistic exam-style practice
• rationale-rich learning
• multiple nursing tiers
• progress tracking
• mock exam readiness

Design this with premium feature cards, not cheesy marketing tiles.

4. CTA structure
Improve homepage CTA flow:
• primary CTA above the fold
• another CTA after feature explanation
• conversion CTA near pricing
• make CTAs visually strong but elegant

--------------------------------------------------
PART 3 — DASHBOARD UX SHOULD FEEL LIKE A PREMIUM EXAM PLATFORM
--------------------------------------------------

Redesign student dashboards so they feel clean, focused, and product-led.

1. Dashboard hierarchy
Each dashboard should immediately show:
• current tier
• progress snapshot
• quick start options
• recently used tools
• recommended next action

2. Replace cluttered cards with structured modules
Use modules such as:
• Continue studying
• Today’s progress
• Weak areas
• Recent mock exam
• Flashcards due
• Recommended lessons

3. Add strong premium visual hierarchy
Most important actions should be obvious at a glance.

4. Reduce icon overload
Keep icons only where they improve scanning:
• navigation
• quick action cards
• analytics summaries
Remove unnecessary decorative icons in educational content.

--------------------------------------------------

--------------------------------------------------

Upgrade all question-related screens.

1. Question page layout
Improve the layout of:
• question stem
• answer options

• cleaner reading experience

--------------------------------------------------
PART 5 — FLASHCARDS SHOULD LOOK MORE PREMIUM AND LESS BASIC
--------------------------------------------------

Upgrade the flashcard experience.

1. Card design
Flashcards should feel premium:
• cleaner front/back transitions
• stronger spacing
• better visual framing of prompt and answer
• polished image placement if image exists
• more elegant button layout

2. Theme integration

3. Randomization and flow
If not already present, keep flashcards feeling dynamic:

• smooth transitions
