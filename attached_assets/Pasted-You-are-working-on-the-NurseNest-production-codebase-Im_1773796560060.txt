You are working on the NurseNest production codebase.

Implement PHASE 2: CORE RESILIENCE ARCHITECTURE.

This phase builds the central systems that make the platform robust, recoverable, and easier to operate.

PRIMARY GOAL
Create a durable resilience foundation across all premium and critical user flows.

BUILD THESE SYSTEMS

1. Central Entitlement Resolver
Create a single source of truth returning a normalized access object:
- hasAccess
- accessSource
- planId
- productId
- productType
- region
- locale
- fallbackEligible
- backupModesAvailable
- substituteEligible
- lastVerifiedVersion
- expiresAt
- provisionalAccess
- decisionReason

Replace scattered entitlement logic everywhere critical.

2. Access Delivery Orchestrator
Create a routing/orchestration layer that selects:
- primary experience
- safe fallback mode
- last-known-good version
- backup snapshot
- substitute equivalent content
- downloadable/static fallback

3. Publish-Time Validation Pipeline
Before premium content is exposed:
- validate schema
- validate required fields
- validate question/deck/lesson structures
- validate translations
- validate media/assets
- validate supported renderer compatibility

If invalid:
- block publish
- preserve old verified version
- mark item for repair
- do not expose broken content live

4. Snapshot + Versioning System
At publish time generate:
- verified render payload
- backup payload
- static backup where applicable
- immutable version record
- last-known-good pointer

5. Content Quarantine System
If published content is discovered to be malformed:
- quarantine it immediately
- stop serving it in main path
- auto-route users to backup or substitute
- preserve trust messaging

6. Admin Reliability Dashboard
Show:
- failing routes
- fallback usage
- quarantined content
- broken content IDs
- provisional access events
- entitlement mismatches
- recent incidents
- affected users

Include actions:
- disable content
- restore version
- force backup mode
- replay entitlement sync
- regenerate backup artifacts
- extend access

7. Alerting
Trigger alerts for:
- failure spikes
- fallback spikes
- entitlement anomalies
- backup generation failures
- quarantine events
- excessive protected recovery screen events

8. Synthetic Monitoring
Create synthetic user flows for:
- login
- exam open
- CAT start
- flashcards
- lessons
- premium downloads

TESTS REQUIRED
- access resolver handles subscription, bundle, one-time purchase, promo, legacy, and provisional access
- orchestrator routes correctly on primary failure
- invalid publish is blocked
- last-known-good is served automatically
- quarantined content is removed from main route
- admin dashboard surfaces incidents correctly
