SECTION 2 — AI ACCURACY VERIFICATION (Must-Have)
Implement an AI verification workflow for user-created flashcards.

When a user creates or edits a flashcard:
1) Backend endpoint verifies content (front/back text + optional image metadata + tags).
2) AI returns ONLY STRUCTURED JSON in this schema:
{
  "status": "VERIFIED"|"PARTIALLY_ACCURATE"|"INACCURATE"|"UNSAFE",
  "confidence": 0-100,
  "issues": [
    { "field":"front"|"back"|"both",
      "type":"FACT_ERROR"|"OUTDATED"|"AMBIGUOUS"|"MISSING_CONTEXT"|"UNSAFE"|"IMAGE_ACCURACY",
      "snippet":"...",
      "explanation_short":"(2-4 sentences max)",
      "suggested_fix":"(short correction, not a lesson)"
    }
  ],
  "corrected_version": { "front":"...", "back":"..." },
  "safety_flags": [
    { "category":"DOSING"|"CONTRAINDICATION"|"EMERGENCY_ACTION"|"CRITICAL_VALUE"|"ACID_BASE"|"OTHER",
      "severity":"HIGH"|"MED"|"LOW",
      "note_short":"..."
    }
  ],
  "publish_eligibility":"PUBLIC_OK"|"PRIVATE_ONLY"|"ADMIN_REVIEW_REQUIRED",
  "disclaimer_short":"One sentence safety disclaimer."
}

Rules:
- If status is UNSAFE or publish_eligibility is ADMIN_REVIEW_REQUIRED => card cannot be public; route to admin queue.
- Store verification results with model name/version and timestamps.
- Show badges:
  "AI Verified", "AI Flagged (Private)", "Not Verified"
- Only VERIFIED + PUBLIC_OK cards may be included in PUBLIC sets that are indexed.

Free vs Paid gating (Revenue-protective, but keep core usable):
- Free: limited verification per day (configurable 5–10/day). Show status + minimal issue summary only.
- Paid: unlimited verification, full concise fix suggestions, and bulk set verification.

IMAGE VERIFICATION:
- Users can upload images for flashcards (front/back). For paid users, run additional AI image-accuracy check (lightweight). If fails => PRIVATE_ONLY.
- Do NOT embed text in generated images. (For user uploads, allow text but do not generate text-in-image.)
