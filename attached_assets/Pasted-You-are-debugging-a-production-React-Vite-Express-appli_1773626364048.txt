You are debugging a production React + Vite + Express application deployed on Replit. The site currently loads a completely blank white page on desktop and mobile browsers.

Your task is to perform a systematic diagnosis and repair. Do NOT restructure the project, change navigation systems, or modify tier menus. Only locate and fix the root cause of the blank screen.

Follow this process in order.

STEP 1 — Verify server response

1. Confirm the Express server is running and returning index.html.
2. Confirm that the network request for `/` returns status 200.
3. If the server is returning HTML correctly but the page is blank, continue to frontend diagnostics.

STEP 2 — Inspect browser console errors
Open the browser console and identify any fatal errors preventing React from rendering.

Look specifically for errors such as:

* "Cannot read property of undefined"
* "Failed to fetch"
* "Module not found"
* "Hydration error"
* "Uncaught ReferenceError"
* "Unexpected token"

Log the exact file and line number of the first error.

STEP 3 — Check React mount point
Verify that index.html contains a valid root element:

`<div id="root"></div>`

Then confirm the entry file (main.tsx or main.jsx) contains:

```
ReactDOM.createRoot(document.getElementById("root")).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
)
```

If the root element ID does not match, fix it.

STEP 4 — Verify router configuration
If using Wouter or React Router:

• Confirm that the main App component actually returns JSX.
• Confirm that no route renders `null` or `undefined`.
• If routes depend on authentication or role checks, add a fallback loader instead of returning nothing.

Example safe fallback:

```
if (!userLoaded) {
  return <div>Loading...</div>;
}
```

STEP 5 — Check failing imports
Scan all components imported into App.tsx.

Common causes of white screens:
• importing a component from a file that does not exist
• circular imports
• importing server-only code into the client bundle

Fix any broken imports.

STEP 6 — Validate environment variables
If the frontend references environment variables (Stripe keys, API URLs, etc.), ensure they use Vite format:

`import.meta.env.VITE_VARIABLE_NAME`

Replace any usage of:

`process.env`

with the correct Vite syntax.

STEP 7 — Disable crashing components temporarily
If a component causes the crash, wrap it in a temporary error boundary to confirm.

Example:

```
try {
  return <Component />
} catch (err) {
  console.error(err)
  return <div>Component failed to load</div>
}
```

Once identified, fix the underlying issue.

STEP 8 — Confirm bundle builds correctly
Run a production build.

`npm run build`

Fix any TypeScript or bundling errors.

STEP 9 — Verify assets load correctly
Ensure the following load successfully:

• main.js bundle
• CSS bundle
• images and logos

If any return 404, correct the asset path.

STEP 10 — Final verification
After fixes:

1. Reload the site with a hard refresh.
2. Confirm React mounts successfully.
3. Confirm the homepage renders.
4. Confirm navigation works.

IMPORTANT RULES
• Do not modify the navigation tier system.
• Do not change pricing or Stripe code.
• Do not move pages or routes.
• Only repair the cause of the white screen.

Return a short report listing:

1. Root cause
2. Files modified
3. Exact fix applied.
