B) Watermarked preview system (end-to-end)

This is the real missing piece. You need:

DB columns for preview storage key/url

Admin endpoint to generate preview (sample pages + watermark)

Public endpoint to stream preview

UI to show preview (public product page) and “Regenerate preview” (admin)

B1) DB schema changes (minimal, safe)

Add storage-key fields, not raw URLs, so you can change storage later.

Migration:

preview_storage_key TEXT NULL

preview_updated_at TIMESTAMP NULL

preview_page_count INT NOT NULL DEFAULT 3

(If you already have pdf_storage_key for the full PDF, great. If not, you must.)

If you truly want a URL field:

But best practice: store the key and generate URL in code.

B3) Server: generate preview (admin-only)

Create route:

POST /api/admin/products/:id/generate-preview

Important: do not require a new upload. Use the existing full PDF storage key.

Here is a working pdf-lib watermark generator (Node):

import { PDFDocument, StandardFonts, rgb, degrees } from "pdf-lib";

async function generateWatermarkedPreviewPdf(fullPdfBytes: Uint8Array, pageCount: number) {
  const src = await PDFDocument.load(fullPdfBytes);
  const dst = await PDFDocument.create();

  const font = await dst.embedFont(StandardFonts.HelveticaBold);
  const pagesToCopy = Math.min(pageCount, src.getPageCount());
  const copied = await dst.copyPages(src, [...Array(pagesToCopy)].map((_, i) => i));

  const watermarkText = "NURSENEST PREVIEW • DO NOT DISTRIBUTE";

  for (const page of copied) {
    dst.addPage(page);

    const { width, height } = page.getSize();
    const fontSize = Math.max(28, Math.min(56, Math.floor(width / 18)));

    page.drawText(watermarkText, {
      x: width * 0.08,
      y: height * 0.45,
      size: fontSize,
      font,
      color: rgb(0, 0, 0),
      rotate: degrees(-32),
      opacity: 0.14,
    });

    // Optional second watermark for stronger deterrence
    page.drawText(watermarkText, {
      x: width * 0.08,
      y: height * 0.25,
      size: fontSize,
      font,
      color: rgb(0, 0, 0),
      rotate: degrees(-32),
      opacity: 0.10,
    });
  }

  const bytes = await dst.save();
  return bytes; // Uint8Array
}

Then in your route:

app.post("/api/admin/products/:id/generate-preview", requireAdmin, async (req, res) => {
  const id = req.params.id;
  const pageCount = Math.max(1, Math.min(10, Number(req.body?.pageCount ?? 3)));

  const product = await db.products.findById(id);
  if (!product) return res.status(404).json({ error: "Not found" });
  if (!product.pdfStorageKey) return res.status(400).json({ error: "No full PDF for product" });

  // 1) Load full PDF bytes from private storage
  const fullPdfBytes = await storage.readBytes(product.pdfStorageKey);

  // 2) Generate preview bytes
  const previewBytes = await generateWatermarkedPreviewPdf(fullPdfBytes, pageCount);

  // 3) Save preview to storage
  const previewKey = `previews/${id}/preview-${Date.now()}.pdf`;
  await storage.writeBytes(previewKey, previewBytes, "application/pdf");

  // 4) Persist pointer
  await db.products.update(id, {
    previewStorageKey: previewKey,
    previewPageCount: pageCount,
    previewUpdatedAt: new Date(),
  });

  res.json({ ok: true });
});

Storage adapter requirement: you need storage.readBytes(key) and storage.writeBytes(key, bytes, mime) for local disk/S3/etc. If you don’t have an abstraction yet, implement local disk first.

B4) Server: stream preview publicly

Add:

GET /api/products/:slug/preview

app.get("/api/products/:slug/preview", async (req, res) => {
  const slug = req.params.slug;
  const product = await db.products.findBySlug(slug);
  if (!product || product.isDeleted) return res.status(404).end();
  if (!product.previewStorageKey) return res.status(404).end();

  const bytes = await storage.readBytes(product.previewStorageKey);

  res.setHeader("Content-Type", "application/pdf");
  res.setHeader("Content-Disposition", "inline"); // preview should display
  res.setHeader("Cache-Control", "public, max-age=3600");
  res.send(Buffer.from(bytes));
});

Critical: your full PDF download route must NOT be publicly accessible. Keep it:

admin-only or

entitlement-only (purchasers)

D) Quick sanity warnings (to prevent rework)
1) Don’t add only previewUrl unless your storage is truly public

If you store previewUrl as an actual public URL, you’ll eventually leak full PDFs the same way. Prefer storage keys + streaming endpoints.

2) Cap preview pages

Use a hard max like 10 pages to prevent abuse.

3) Don’t touch translations right now

It’s a separate risk surface. Finish preview + downloads + image lab first.
