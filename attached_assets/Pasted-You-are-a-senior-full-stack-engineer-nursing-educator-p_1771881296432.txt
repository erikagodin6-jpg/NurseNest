You are a senior full-stack engineer + nursing educator + psychometrics consultant.

Build a “Content Generator” module for my nursing education platform (Next.js/React + TypeScript). It must generate:
1) Free Anatomy quizzes/exams
2) Free Pre-Nursing foundational quizzes/exams
3) Large question bank expansions
4) Full mock exams assembled from the bank

Key requirements:

A) Content blueprint first (must be explicit)
Create a blueprint JSON schema that defines:
- Course: anatomy or pre-nursing
- Exam type: quiz, unit test, cumulative exam, mock exam
- Target learner level: pre-nursing / RPN / RN / NP (default: pre-nursing for free section)
- Domains (topics) and subtopics with target weight %
- Difficulty distribution (easy/medium/hard)
- Cognitive level distribution (recall / understanding / application / analysis) based on Bloom
- Item types distribution (MCQ single-best, SATA, ordering, fill-in-blank, hotspot/labeling)
- Time limit, number of questions, pass mark

B) Question item spec (must be standardized)
Define a single Question object with:
- id, course, topic, subtopic
- stem
- options[] (if MCQ/SATA)
- correctAnswer (single) or correctAnswers (multiple)
- rationaleCorrect (2–5 sentences)
- rationaleIncorrect (brief bullets for why each distractor is wrong)
- difficulty (1–3)
- bloomLevel
- clinicalCorrelation (optional but encouraged even for anatomy; 1–2 sentences)
- references field (text-only; do not browse web)
- tags[] (e.g., “musculoskeletal”, “cranial nerves”, “cell biology”)
- estimatedTimeSeconds
- version, createdAt

C) Item-writing quality controls (non-negotiable)
Implement automated QA checks:
- Exactly one correct answer for single-best MCQs
- Distractors must be plausible and grammatically parallel
- No “all of the above / none of the above”
- No trick wording
- Correct spelling and consistent units
- Ensure stem + options match scope for pre-nursing/anatomy
- Detect duplicates/near-duplicates (semantic similarity)
- Flag ambiguous items (too-short stem, multiple plausible answers)

D) Generation process
Implement a generator that:
1) Creates/reads a blueprint
2) Produces N new questions matching blueprint constraints
3) Runs QA checks and reports failures
4) Regenerates failed items until passing
5) Saves to local JSON files and exposes an API route to return batches

E) Mock exam assembler
Build an assembler that:
- Pulls from the bank by topic weights + difficulty
- Ensures no duplicates
- Produces a mock exam object with section breakdown + timer
- Can create multiple unique forms (Form A/B/C)
- Exports as JSON

F) UI (admin tools)
Create a simple admin UI page:
- Choose course (anatomy / pre-nursing)
- Choose blueprint or generate blueprint from a topic list
- Set number of new questions to generate
- Generate + view QA report
- Approve/reject items
- Build mock exams from approved items
- Export JSON

G) Provide starter blueprints
Ship at least:
- Pre-nursing foundations: 60-question cumulative exam + 3 unit quizzes
- Anatomy: 100-question mock exam + 5 unit quizzes (skeletal, muscular, cardio, resp, neuro)

Output:
Generate the full code, folder structure, schemas, UI, sample blueprints, and sample question bank JSON (at least 200 items total across both courses, generated in batches with QA).
