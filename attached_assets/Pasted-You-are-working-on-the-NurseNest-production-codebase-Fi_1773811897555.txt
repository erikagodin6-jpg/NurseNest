You are working on the NurseNest production codebase.

Fix the build-memory problem caused by oversized i18n source files being processed by Vite/TypeScript.

GOAL
The client build must stop scanning or bundling large locale source files. All locales, including English, must use a compiled JSON runtime-loading architecture.

REQUIRED CHANGES

1. MOVE I18N SOURCE FILES OUT OF CLIENT SOURCE TREE
- Move all translation authoring files out of client/src
- Put them in a non-bundled directory such as:
  - tools/i18n/source/
  or
  - scripts/i18n/source/
- These files must not remain anywhere inside the Vite app source tree

2. COMPILE SOURCE FILES TO JSON
- Keep or improve the existing compileI18n() process
- Generate runtime locale JSON files into:
  public/locales/{locale}.json
  or namespace form:
  public/locales/{locale}/{namespace}.json

3. REMOVE DIRECT CLIENT IMPORTS OF LARGE LOCALE TS FILES
- Search entire codebase for any direct imports of locale source files
- Remove all direct imports of translation source TS files from client code
- Replace with runtime JSON loading via fetch or dynamic resource loader

4. ENGLISH MUST ALSO FOLLOW JSON LOADING
- Do not keep English as a giant directly imported TS object
- Convert English to compiled JSON and load it the same way as other locales
- Maintain fallback behavior, but do not bundle the full English dictionary into the initial app build unless absolutely required
- If a minimal fallback is needed, keep only a tiny emergency fallback object in code

5. EXCLUDE SOURCE I18N AUTHORING DIRECTORY FROM TYPESCRIPT
- Update tsconfig.json exclude list so the authoring directory is excluded
- Ensure Vite does not treat those files as part of the client graph

6. VERIFY RUNTIME I18N LOADING
- App should load only the active locale at runtime
- Support lazy loading
- Support fallback to English JSON if a locale file is missing
- Preserve existing translation behavior

7. OPTIONAL BUT RECOMMENDED: NAMESPACE SPLITTING
- Split very large locale dictionaries by namespace:
  - common
  - home
  - pricing
  - exams
  - lessons
  - dashboard
  - admin
- Load namespaces on demand to reduce memory and initial payload

8. CLEANUP + VALIDATION
- Remove dead imports and stale locale barrels
- Confirm build no longer references old TS locale files
- Confirm app still renders correctly in English and at least one secondary language
- Confirm no mixed-language regressions

DELIVERABLES
- Refactored i18n architecture
- Updated compile step
- Updated tsconfig/vite-safe setup
- Runtime locale loader using compiled JSON only
- Brief summary of what was changed and which files were moved
