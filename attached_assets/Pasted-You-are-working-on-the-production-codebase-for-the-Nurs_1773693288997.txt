You are working on the production codebase for the NurseNest website.

Your task is to build a **FULL BACKUP AND EXPORT SYSTEM** so the entire project can be packaged and saved externally in case anything happens to the Replit environment.

Do NOT explain reasoning.
Do NOT create placeholder scripts.
Build the complete backup/export system.

────────────────────────

PRIMARY GOAL

Allow the site owner to create a **complete downloadable archive of the entire website** including:

• frontend code
• backend code
• database schema
• migrations
• content manifests
• question banks
• lessons
• flashcards
• scenario simulations
• configuration files
• translation files
• assets
• scripts
• deployment configs

The archive must be usable to restore the project outside Replit if needed.

────────────────────────

PART 1 — CREATE BACKUP DIRECTORY

Create a new folder:

/backup-system

Inside it create:

backup-full.ts
backup-code.ts
backup-db.ts
backup-assets.ts
export-deployment.ts
validate-backup.ts

These scripts will generate backup archives.

────────────────────────

PART 2 — FULL PROJECT BACKUP

Create a script:

npm run backup:full

This command must:

1. create a timestamp
2. create a folder

/backups/YYYY-MM-DD-HHMMSS/

3. copy all critical directories into the backup folder

Include:

/client
/server
/shared
/scripts
/public
/assets
/i18n
/config
/package.json
/package-lock.json or yarn.lock
/tsconfig files
/vite config
/tailwind config
/database schema files
/migration files

Exclude:

node_modules
.cache
dist builds

After copying, compress the backup into:

nursenest-full-backup-YYYY-MM-DD-HHMMSS.zip

────────────────────────

PART 3 — DATABASE BACKUP

Create command:

npm run backup:db

This must export:

database schema
migrations
table structure
content tables (non-sensitive)

Output files:

/backups/db/schema.sql
/backups/db/table-summary.json
/backups/db/core-content.json

Do NOT export user passwords or sensitive user data.

────────────────────────

PART 4 — ASSET BACKUP

Create command:

npm run backup:assets

This copies:

logos
icons
hero images
PDF templates
generated images
translation files
SEO assets

Output folder:

/backups/assets/

────────────────────────

PART 5 — DEPLOYMENT EXPORT

Create command:

npm run export:deployment

This generates a clean export of the entire site for redeployment elsewhere.

Create folder:

/exports/deployment/

Include:

source code
package.json
lock files
schema
migrations
.env.example
deployment instructions

Create file:

DEPLOYMENT_README.md

Include steps:

1 install dependencies
2 configure environment variables
3 run migrations
4 build project
5 start server

────────────────────────

PART 6 — ENVIRONMENT TEMPLATE

Generate file:

.env.example

List all required variables:

DATABASE_URL
STRIPE_SECRET_KEY
STRIPE_PUBLIC_KEY
OPENAI_API_KEY
PAYPAL_CLIENT_ID
SESSION_SECRET
SMTP_CONFIG
ANALYTICS_KEYS

Do not include actual secrets.

────────────────────────

PART 7 — BACKUP VALIDATION

Create command:

npm run backup:validate

This must verify the archive contains:

client code
server code
schema
config files
translations
assets

If anything is missing, show warnings.

────────────────────────

PART 8 — ADMIN BACKUP PANEL

Add an admin page:

/admin/backups

This page must allow:

Generate Full Backup
Generate Deployment Export
Download Latest Backup
Validate Backup

Display:

latest backup time
backup size
validation status

────────────────────────

PART 9 — BACKUP LOGS

Create logs folder:

/backups/logs

Each backup must log:

timestamp
backup type
archive size
files included
validation status

────────────────────────

PART 10 — OUTPUT

After implementing the system show:

scripts created
commands available
backup file locations
example archive path

────────────────────────

FINAL RESULT

The project owner must be able to run:

npm run backup:full

and receive a **complete downloadable zip of the entire site** that can be saved externally.
