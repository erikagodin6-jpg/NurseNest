====================================================

GOAL:
Stop guessing difficulty levels. Make them data-driven using real performance metrics.

A) DATABASE CHANGES

Create new fields in questions table:
- difficulty_declared (1–5)  // what admin sets
- difficulty_calibrated (1–5)  // data-driven result
- discrimination_index (float)
- average_time_seconds (float)
- attempt_count (int)
- percent_correct (float)
- standard_deviation_time (float)
- last_calibrated_at (datetime)

Create table:
question_attempt_stats
- question_id
- user_id
- timestamp
- time_spent_seconds
- correct (bool)
- difficulty_at_attempt
- exam_mode (tutor|strict|cat)

B) DIFFICULTY METRICS

For each question compute:

1) Percent Correct
2) Average Time Spent
3) Discrimination Index:
   - Correlation between question correctness and overall user performance
   - High performers should answer correctly more often than low performers

4) Difficulty Score Formula:

raw_difficulty_score =
  (1 - percent_correct) * 0.5
  + normalized_avg_time * 0.2
  + (1 - discrimination_index_normalized) * 0.3

Convert raw_difficulty_score into 1–5 scale using quantiles across tier/exam.

C) CALIBRATION RULES

- Only calibrate if attempt_count >= 100
- Exclude tutor mode attempts from calibration
- Recalculate nightly via cron job
- Store calibrated value in difficulty_calibrated
- If difficulty_calibrated differs from difficulty_declared by >=2 levels, flag in admin QC dashboard

D) ADMIN UI

Add “Item Analytics” view:
- percent correct
- discrimination index
- avg time
- difficulty declared vs calibrated
- attempt count
- confidence in calibration

Add “Recalibrate All” job button.

E) USE CALIBRATED DIFFICULTY IN:

- Mock exam assembly
- CAT algorithm
- Probability model weighting
- Study plan escalation
