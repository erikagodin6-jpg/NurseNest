PART 2 — USER SEGMENTATION (RULE-BASED FIRST)
====================================================

Segment users using deterministic rules (no ML required initially):

Segments:
A) Exam Soon High Intent
- time_to_exam_days <= 30
- diagnostic_complete = true

B) Moderate Risk Strivers
- pass_probability between 60–75 OR readiness band borderline
- completed >= 200 questions

C) Content Explorers
- reading lessons, low qbank usage

D) Heavy Practicers
- high question volume + mocks completed

E) Returning / At-risk
- inactivity > 7 days
- previously engaged

Store segment in user_revenue_profile and update daily.

====================================================
PART 3 — PROPENSITY + PRICE SENSITIVITY SCORES
====================================================

Compute two scores (0–1), simple and explainable:

propensity_score (likelihood to upgrade)
Signals:
- diagnostic completion
- study plan created
- mock exams taken
- simulator clicks (locked)
- time_to_exam proximity
- repeated sessions over 7 days

price_sensitivity_score (likelihood to respond to discounts)
Signals:
- multiple upgrade modal dismissals
- low income region indicator (optional; do not infer sensitive traits)
- prefers monthly over annual historically
- time spent on pricing page without converting

No personal attribute inference. Use only behavioral signals.

====================================================
PART 4 — OFFER DECISION ENGINE
====================================================

Create a centralized service:
getBestOffer(user_id, context)

Contexts:
- after_diagnostic
- after_first_mock
- when_prob_moderate_risk
- when_exam_soon
- winback_after_inactivity
- after_simulator_click_locked

Decision rules:

1) If segment=Exam Soon High Intent AND propensity high:
   show premium monthly + optional 7-day trial

2) If propensity medium AND price_sensitivity high:
   show 3-month bundle discount (paid up front) OR first-month discount

3) If user is Heavy Practicer but not converting:
   emphasize premium features:
   - strict CAT simulator
   - probability confidence band
   - improvement simulator

4) If user is Content Explorer:
   show Core tier first, then Premium upsell after first mock

5) Winback:
   if inactive 7–14 days:
     show “come back plan” + small limited-time discount (optional)

Guardrails:
- Max 1 modal/day
- Max 3 offer impressions/week
- Respect “don’t show again for 7 days” option

Return:
- offer_id
- tier emphasized
- localized copy keys
- CTA targets

Gate Premium features at “aha moments”:
