SYSTEM / STYLE INSTRUCTION
Build features for a professional nursing education platform called NurseNest.
Audience: RPN, RN, NP learners and practicing nurses.
Quality constraints:
• No shallow summaries
• No generic textbook phrasing
• Prioritize clinical reasoning and mechanisms
• Explanations must emphasize WHY and HOW, not memorization
• Reflect licensing exam logic (NCLEX / Next Gen / NP exams)
• UX must feel modern, minimal, cognitively clean, low-friction
Design language:
• Soft clinical aesthetic
• Rounded UI
• Calm, high-trust interface
• Avoid childish or cartoonish visuals
Educational philosophy:
Teach mental models, pattern recognition, and decision logic — not fact lists.
1. Clinical Decision-Making Trainer
PROMPT
Create a Clinical Decision-Making Trainer module.
Functionality:
Generate interactive patient scenarios where users must make sequential clinical decisions under uncertainty.
Requirements:
• Cases must evolve dynamically based on user choices
• Include subtle cues, competing priorities, incomplete data
• Simulate realistic nursing dilemmas (deterioration, safety risks, prioritization)
• Each decision must have clinically accurate consequences
• Include detailed rationales explaining cognitive errors and correct reasoning
Cognitive design:
• Force prioritization thinking
• Include distractors resembling licensing exam traps
• Avoid obvious correct answers
UI Behavior:
• Present information progressively (like unfolding cases)
• Allow "What would you do next?" interactions
• Provide post-decision breakdowns
2. Charting & Documentation Simulator
PROMPT
Build a Charting and Clinical Documentation Simulator.
Functionality:
Simulate realistic patient documentation tasks.
Include:
• Narrative nursing notes
• Focused assessments
• Incident documentation logic
• Legal defensibility principles
• Common charting mistakes
Educational logic:
• Users submit charting entries
• System evaluates for safety, accuracy, omissions, risk
• Provide corrective feedback explaining medico-legal reasoning
Avoid:
• Generic EMR replicas
• Oversimplified templates
Goal:
Teach how nurses think when documenting, not just formatting.
3. Abnormal Labs Interpretation Engine
PROMPT
Create an Abnormal Laboratory Interpretation Engine.
Functionality:
Allow users to explore abnormal lab patterns and interpret physiological meaning.
Requirements:
• Present labs in clusters, not isolation
• Explanations must focus on mechanisms and compensatory responses
• Include misleading / borderline scenarios
• Emphasize clinical implications and dangers
Example reasoning depth:
Not “High potassium = hyperkalemia”
But:
Explain membrane excitability, conduction risk, causes, and prioritization.
Include:
• Pattern recognition challenges
• Clinical consequence mapping
• Common exam pitfalls
4. Rapid Response & Deterioration Recognition Trainer
PROMPT
Build a Rapid Response and Clinical Deterioration Recognition Trainer.
Functionality:
Simulate early signs of patient instability.
Include scenarios involving:
• Sepsis evolution
• Internal bleeding
• Respiratory compromise
• Neurological decline
Design logic:
• Prioritize subtle early warning signs
• Avoid dramatic obvious collapse
• Require escalation decisions
Feedback engine:
Explain why deterioration occurs physiologically and why certain actions are time-critical.
