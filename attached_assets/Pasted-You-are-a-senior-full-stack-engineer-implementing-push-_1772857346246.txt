You are a senior full-stack engineer implementing push notifications for NurseNest, a nursing exam preparation platform.

Goal:
Add push notifications to the NurseNest website/PWA first, with a structure that can later support a native mobile app.

Current platform context:
- NurseNest has lessons, question banks, flashcards, progress tracking, subscriptions, and study streaks
- users log in with accounts
- subscribed users should receive personalized study reminders
- the platform should support daily engagement notifications such as daily questions, flashcard review reminders, streak reminders, and weak-area review prompts

Build the push notification system in phases.

PHASE 1 — WEB PUSH / PWA
Implement web push notifications using:
- service worker
- browser Notifications API
- Push API
- Firebase Cloud Messaging for web delivery

Tasks:
1. Add or update the service worker for push support
2. Add notification permission request flow
3. Do not request permission immediately on first page load
4. Trigger permission prompts only after meaningful user engagement, such as:
   - after completing 5 questions
   - after finishing a lesson
   - after starting a study streak
5. Register the browser/device for push notifications
6. Store the push token or subscription in the backend tied to the authenticated user
7. Support multiple devices per user

PHASE 2 — BACKEND NOTIFICATION SYSTEM
Create backend infrastructure for sending notifications.

Requirements:
1. Create a notifications table and device subscriptions table
2. Store:
   - user_id
   - device or subscription token
   - platform
   - permission status
   - created_at
   - last_active_at
3. Create server-side notification sending endpoints or background jobs
4. Support scheduled notifications and triggered notifications
5. Add retry logic and failure handling
6. Add unsubscribe / opt-out handling

PHASE 3 — NOTIFICATION TYPES
Implement these initial NurseNest notification types:

1. Daily Question Reminder
Example:
Your daily NCLEX question is ready

2. Flashcard Review Reminder
Example:
You have 12 flashcards due for review

3. Study Streak Reminder
Example:
Your 4-day study streak ends today

4. Weak Area Review Reminder
Example:
Review cardiac rhythms before they fall behind

5. New Content Reminder
Example:
A new sepsis case study is available in your study path

Each notification should deep-link the user into the correct page in the web app.

PHASE 4 — USER PREFERENCES
Create notification preference settings so users can control:
- daily reminders on/off
- flashcard reminders on/off
- streak reminders on/off
- new content alerts on/off
- preferred reminder time
- exam tier preference: RPN, RN, NP

PHASE 5 — FUTURE NATIVE APP COMPATIBILITY
Design the system so it can later support a React Native / Expo app using the same backend notification logic.

Do not hardcode the architecture only for web.
Make token storage and notification event handling extensible for:
- web push
- iOS native push
- Android native push

DATABASE DESIGN
Recommend and implement tables such as:
- user_notification_preferences
- user_push_subscriptions
- notification_events
- notification_delivery_logs

SECURITY AND UX
- require authenticated users for subscription registration
- validate tokens/subscriptions
- avoid spammy notification frequency
- add rate limits
- make notifications useful and personalized
- ensure clear opt-out and preference controls

OUTPUT
Provide:
1. architecture overview
2. frontend implementation steps
3. backend implementation steps
4. service worker setup
5. database schema
6. deep-linking logic
7. example notification payloads
8. testing instructions
9. production-ready code where possible
