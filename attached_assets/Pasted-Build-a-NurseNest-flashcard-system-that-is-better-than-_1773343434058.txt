Build a NurseNest flashcard system that is better than UWorld-style flashcards while remaining original to NurseNest.

This is not a simple flashcard feature. It is a premium, tier-specific, adaptive clinical study engine for nursing learners.

IMPORTANT
Do not copy UWorld branding, wording, UI, or proprietary design exactly.
Create an original NurseNest product that is stronger in functionality and more personalized for RPN, RN, NP, and other tiers.

CORE PRODUCT GOAL

Turn flashcards into a high-value clinical reasoning study mode.

Each “flashcard” should really function as:
- an untimed exam-style question
- with answer choices
- with full rationale
- with incorrect option explanations
- with a related lesson link
- with optional supportive image
- with spaced repetition and weakness targeting
- with tier-specific difficulty and logic

This should feel like:
- exam prep
- remediation
- active recall
- personalized reinforcement

The system should be more adaptive, more educational, and more tier-aware than generic flashcard tools.

PART 1: TIER-SPECIFIC FLASHCARD ARCHITECTURE

Flashcards must not be shared across all tiers.

Every flashcard/question must include:
- examTier
- topic
- subtopic
- difficulty
- blueprintCategory
- cardType
- questionStem
- options
- correctAnswer
- rationale
- incorrectOptionRationales
- clinicalPearl
- relatedLesson
- rationaleImage
- tags
- sourceType

Valid examTier values:
- RPN
- RN
- NP
- RRT
- other allied health tiers if supported later

Tier rules:
- RPN cards = fundamentals, safety, common meds, delegation basics, stable patient care, recognition of findings
- RN cards = deeper pathophysiology, prioritization, interpretation, acute changes, more complex pharmacology, multi-step clinical reasoning
- NP cards = differential diagnosis, diagnostics, treatment selection, prescribing considerations, management planning, advanced interpretation

No user should see cards from the wrong tier unless explicitly marked as shared/foundational.

PART 2: FLASHCARDS SHOULD BE EXAM-STYLE QUESTIONS

Eliminate generic definition-style flashcards as the primary experience.

Replace with:
- single best answer multiple choice
- select all that apply where supported
- bowtie/case-based later if supported
- priority questions
- pharmacology questions
- interpretation questions
- scenario-based nursing judgment questions

Each flashcard should feel like:
“practice question with unlimited time and built-in teaching.”

This is the primary value.

PART 3: MODES THAT MAKE IT BETTER THAN UWORLD-STYLE FLASHCARDS

Build multiple study modes inside the flashcard system:

1. Learn Mode
- answer after selection
- full rationale shown immediately
- incorrect option explanations
- image shown if relevant
- lesson link shown
- confidence rating after each card
- mark as mastered / review again

2. Test Mode
- untimed
- answer hidden until user submits
- rationale shown only after answering
- optional streak and performance analytics
- still lower pressure than a mock exam

3. Rapid Review Mode
- compact high-yield questions only
- quick answer/reveal flow
- designed for short study bursts
- optimized for mobile

4. Weak Areas Mode
- automatically pulls cards from systems/topics the learner misses most
- weighted by recency and exam importance

5. Before-Exam Cram Mode
- high-yield blueprint-based question cards
- prioritizes weak + high-frequency exam topics
- short rationale summaries plus expand-for-more

6. Spaced Repetition Mode
- cards resurface based on mastery, misses, and review intervals
- stronger clinical memory retention than static flashcards

PART 4: ADAPTIVE ENGINE

This is one of the biggest ways to make it better.

Create a flashcard adaptation engine that tracks:
- correctness
- response confidence
- time spent
- number of repeats
- flagged status
- mastered status
- topic weakness
- recent misses
- pattern of distractor mistakes

For each user, generate a mastery profile:
- by topic
- by subtopic
- by exam blueprint category
- by question type
- by tier-specific competency domain

Adaptive logic should prioritize:
1. recently missed cards
2. high-yield topics
3. weak systems
4. cards answered correctly but with low confidence
5. cards not reviewed recently
6. cards tagged as exam-critical

Example:
User keeps missing hyperkalemia ECG and potassium management questions
→ system automatically serves more:
- hyperkalemia
- ECG changes
- calcium gluconate vs insulin/dextrose
- renal potassium regulation
- cardiac complications

PART 5: CONFIDENCE-BASED LEARNING

After each card, ask the user:
- Got it
- Somewhat unsure
- Need more review

Or use:
- Confident
- Unsure
- Guess

Use this to improve adaptation.
A user who guessed correctly should still see similar cards again sooner.

This makes the system smarter than simple right/wrong tracking.

PART 6: DEEP RATIONALES

Every card must have a strong rationale structure.

Required rationale sections:
- Correct Answer
- Why this is correct
- Why the other answers are incorrect
- Clinical pearl / exam tip
- Related lesson link
- Optional memory trick / key pattern
- Optional image or chart if it helps

Rationale quality requirements:
- specific, not generic
- teach exam logic
- teach clinical reasoning
- explain why distractors fail
- written at the correct tier level

RPN rationale style:
- clearer and more foundational

RN rationale style:
- more nuanced clinical reasoning

NP rationale style:
- diagnostic and management-oriented depth

Do not use filler text such as:
- “Although this option may seem reasonable...”
unless followed by specific, meaningful explanation.

PART 7: LESSON INTEGRATION

Every flashcard should connect into the learning ecosystem.

Each card should include:
- relatedLessonTitle
- relatedLessonSlug
- relatedLessonUrl
- optional relatedFlashcardDeck
- optional relatedQuestionBankSet

Display this as:
- Review lesson
- Study this topic
- Open full lesson

When a user misses a card repeatedly, recommend the lesson more aggressively.

Possible behavior:
- Missed 3 times → show “You may want to review the lesson on Hyperkalemia Management”

PART 8: IMAGE ACCURACY RULES

Images must be exact and educationally relevant.

Image mapping order:
1. explicit question-level image
2. exact subtopic image
3. exact diagnosis/concept image
4. no image

Never use broad category fallback that can become clinically wrong.

Examples:
- asthma question cannot show COPD
- cardiac tamponade cannot show torsades unless directly relevant
- neonatal sepsis should use neonatal sepsis-related content only

If no exact image exists, show none.

PART 9: REVIEW EXPERIENCE THAT FITS ONE VIEWPORT

The reviewed flashcard must fit within one standard desktop screen for clean screenshot capture.

Desktop reviewed layout:
- left panel: question + options
- right panel: rationale + image + lesson + pearl

Do not stack everything vertically after answering on desktop.

Optimize for:
- 1440x900
- common laptop heights
- screenshot-ready review frame

While in active session:
- reduce global nav height
- suppress giant footer/promotional sections
- keep the study screen focused
