Unify the existing NurseNest dashboard features into one cohesive lifecycle-driven dashboard experience for Nursing and Allied Health users.

Context:
We already have or are building these features:
- readiness predictor
- analytics dashboard
- study streak / momentum system
- exam countdown
- AI study planner
- post-exam follow-up
- new grad transition flow

Goal:
Do not add another disconnected feature. Instead, connect the existing systems into a single intelligent dashboard flow that adapts to where the user is in their exam and career journey.

Main product goal:
Make NurseNest feel like one premium adaptive platform rather than a collection of separate widgets.

IMPLEMENTATION OBJECTIVES

1. Create one primary dashboard “command center” card
This card should be the main personalized card shown near the top of the dashboard.

It should change based on user state.

Possible states:
- No exam date set
- Target date set
- Booked exam upcoming
- Exam date approaching soon
- Awaiting results
- Passed
- Did not pass
- Postponed
- New Grad active

2. State logic
Determine card state using:
- examDate
- examDateType
- examFollowupCompleted
- examResultStatus
- careerStage
- countdown hidden/shown
- study planner hidden/shown
- tier type

3. Command center behavior by state

STATE: No exam date
Show:
- Set Your Exam Date
- Personalize your countdown, study plan, and pacing
Buttons:
- Add Exam Date
- Plan Without a Date
- Hide for Now

STATE: Exam upcoming
Show:
- Days until exam
- current study phase
- weekly target summary
- readiness snapshot
- momentum/streak message
Buttons:
- View Full Study Plan
- Edit Date

STATE: Exam approaching soon
Show:
- countdown
- supportive urgency messaging
- mock exam recommendation
- weak area focus
- momentum reminder

Example:
- 18 days until your exam
- Focus on high-yield review and timed practice
- Keep your 5-day streak going

STATE: Awaiting results
Show:
- Results pending message
- light review suggestions
- ability to update exam result later

STATE: Passed
Show:
- Congratulations message
- transition to New Grad / career support
- discount or offer if configured
Buttons:
- Explore New Grad Support
- Claim Offer

STATE: Did not pass
Show:
- supportive recovery message
- weak area review summary
- rebuild plan CTA
Buttons:
- Build My New Plan
- Review Weak Areas
- Update Exam Date

STATE: Postponed
Show:
- update exam date prompt
- restore study plan

STATE: New Grad active
Show:
- first-year clinical support
- report/prioritization tools
- shift prep
- interview/career tools

4. Connect countdown + momentum + readiness
The command center card must combine these systems into one message block.

Do not show them as disconnected widgets if avoidable.

Examples:
- 63 days until your exam. You’re in foundation phase. Maintain your 4-day streak and aim for 18 questions today.
- 21 days until your exam. Your readiness is improving. Prioritize pharmacology and one timed mock this week.
- Your exam date has passed. Tell us how it went so we can guide your next step.

5. Reduce dashboard clutter
Audit the dashboard and reduce duplicate cards or overlapping modules.
If multiple widgets repeat the same concept, consolidate them.

Examples:
- countdown + study planner + readiness summary should feel connected
- post-exam follow-up should replace countdown when relevant
- new grad transition should replace exam-focused prompts when relevant

6. Shared design system
Use a consistent visual design for all state cards:
- same border radius
- same shadow style
- same badge system
- same CTA styling
- same spacing rhythm
- same typography hierarchy

Match the existing premium NurseNest dashboard style.

7. Messaging rules
Use calm, supportive, premium language.
Avoid:
- panic language
- guilt-based messaging
- robotic text

Use:
- next step
- stay on track
- recommended focus
- build confidence
- personalized plan

8. Tier-aware adaptation
The same dashboard engine should work for:
- RPN
- RN
- NP
- Allied Health tiers

Content recommendations and weak areas should be tier-appropriate.

9. Data/model support
Ensure dashboard state can be driven by:
- examDate
- examDateType
- examResultStatus
- examFollowupCompleted
- careerStage
- studyPlanIntensity
- examCountdownHidden
- studyPlannerHidden
- newGradResourcesActivated

10. QA requirements
Verify:
- only one primary command center card appears at a time
- card changes correctly as the user moves through stages
- countdown, momentum, and readiness are integrated cleanly
- dashboard feels less cluttered
- passed users see career-transition content
- users who did not pass see recovery planning
- mobile and desktop layout remain clean
- no repeated prompts or conflicting states

Final instruction:
This is a dashboard cohesion and lifecycle orchestration task, not a new isolated feature. Refactor the user experience so NurseNest feels like one intelligent, adaptive learning platform from exam prep through new grad transition.
