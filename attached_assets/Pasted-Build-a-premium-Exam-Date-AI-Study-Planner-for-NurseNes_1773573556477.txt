Build a premium Exam Date AI Study Planner for NurseNest that expands on the existing Exam Countdown feature for Nursing and Allied Health tiers.

Goal:
Turn the dashboard from a simple analytics page into a personalized exam prep command center by using the user’s exam date or target date to generate a dynamic study plan with weekly goals, pacing targets, and adaptive recommendations.

Context:
- Platform: NurseNest
- Stack: Vite + React + Express
- Existing or planned feature: Exam Countdown card on user dashboard
- Audience: Nursing tiers (RPN, RN, NP) and Allied Health tiers
- This should feel high-end, supportive, personalized, and motivating
- It should help users answer: “What should I study next, how much should I do, and am I on track for my exam date?”

Core product vision:
Create an AI-powered study pacing system that uses the user’s exam date, current progress, and weak areas to recommend a realistic weekly prep plan.

MAIN FEATURE SET

1. Exam Date + Study Planner Integration
Use the exam date countdown as the anchor for a personalized study planning engine.

If exam date exists, the system should generate:
- days until exam
- current study phase
- weekly targets
- daily pacing suggestions
- recommended content mix
- readiness messaging

If no exam date exists:
- prompt user to add a booked date or target date
- allow user to hide the planner
- optionally allow a “build me a plan without a date” fallback based on estimated timeline

2. Study Planner Card / Module
Add a prominent dashboard module called:
- Your Study Plan
or
- Personalized Exam Plan

This module should show:
- exam countdown
- current study phase
- weekly target summary
- recommended next actions
- study load pacing guidance
- ability to view full plan

3. Support both:
- booked exam date
- target exam date

Add a clear label:
- Booked Exam Date
- Target Exam Date

4. Study phase logic
Generate planning phases based on time remaining.

Suggested phase model:
- 90+ days: Foundation + Content Building
- 45–89 days: Practice + Reinforcement
- 14–44 days: Timed Exams + Weak Area Review
- 0–13 days: Final Review + High-Yield Focus

Each phase should change:
- recommendation language
- content priority
- pacing targets
- CTA focus

Example:
90+ days:
- Build core concepts
- Focus on lessons, flashcards, untimed question practice

45–89 days:
- Increase question volume
- Reinforce weak systems
- Begin more mixed practice

14–44 days:
- Prioritize mock exams
- Focus on weak areas and timed review
- Simulate exam conditions

0–13 days:
- High-yield rapid review
- Lighter new content load
- Focus on confidence, repetition, and final weak-area closure

5. Personalized pacing logic
Generate pacing targets using current progress and time remaining.

If available, use:
- total questions completed
- total flashcards completed
- lessons completed
- weak areas / analytics categories
- mock exam performance
- readiness score
- target goals already in the system

Generate outputs like:
- Aim for 25 questions/day this week
- Complete 2 mixed quizzes and 1 mock exam
- Review cardiovascular and pharmacology weak areas
- Finish 3 lessons and 40 flashcards this week

If detailed progress data is unavailable, generate simpler recommendations based on exam phase and tier.

6. AI recommendation engine
Create a recommendation layer that suggests:
- what to study next
- which subject areas need attention
- whether to focus on lessons, flashcards, or questions
- when to take mocks
- what intensity the user should maintain

Recommendation examples:
- Your exam is 52 days away. This week, focus on mixed practice and weak-area reinforcement.
- You’ve built a good content base. Shift toward timed question sets and one mock exam this week.
- Your pharmacology and respiratory scores are below target. Prioritize these areas before your next mock.

Do not use guilt-based language.
Avoid:
- you are behind
- time is running out
- you failed to meet target

Use:
- to stay on track
- recommended focus this week
- next best step
- this week’s priority

7. Weekly Study Plan View
Add a dedicated expandable view or modal called:
- Weekly Plan
or
- View Full Study Plan

This view should display:
- this week’s focus
- target questions
- target flashcards
- target lessons
- recommended mock exams
- top weak areas to review
- suggested study rhythm

Optional display:
Monday–Sunday structure or flexible weekly bucket model

Example:
This Week’s Plan
- 120 practice questions
- 2 system review lessons
- 60 flashcards
- 1 timed mock exam
- Focus areas: Cardiac, Pharmacology, Prioritization

8. Flexible planning mode
Allow users to choose or later adjust:
- light plan
- balanced plan
- intensive plan

This adjusts pacing recommendations but keeps the same exam date.

Example:
Light:
- lower daily volume
- gentler weekly targets

Balanced:
- standard recommended plan

Intensive:
- higher daily targets
- more mocks and mixed sets

9. User controls
Allow users to:
- set or edit exam date
- set target vs booked date
- hide/show countdown
- hide/show study planner
- select study intensity
- regenerate plan
- reset plan

10. Data model additions
Add fields as needed to user profile/preferences:
- examDate
- examDateType
- examCountdownHidden
- studyPlannerHidden
- studyPlanIntensity
- generatedStudyPhase
- weeklyStudyTargets
- plannerLastUpdated

If needed, create a structured object for recommended weekly plan data.

11. Tier-aware recommendations
Recommendations should adapt to the user’s product tier.

Nursing tiers:
- RPN
- RN
- NP

Allied Health tiers:
- respiratory therapy
- paramedic
- pharmacy technician
- medical laboratory
- diagnostic imaging
- other supported allied verticals

The planner should use the correct content pools and analytics categories for each tier.

12. UI/UX requirements
Design style:
- premium
- calm
- motivating
- consistent with NurseNest analytics dashboard
- rounded cards
- soft shadows
- clean typography
- subtle accent colors
- not cluttered

The planner should feel like part of a serious exam prep ecosystem.

Dashboard module should include:
- countdown
- current study phase badge
- next action summary
- weekly targets snapshot
- button to view full plan

13. Smart empty states
If no exam date:
Show:
- Set Your Exam Date
- Personalize your countdown, pacing, and study plan
Buttons:
- Add Date
- Plan Without a Date
- Hide for Now

If exam date passed:
Show:
- Your exam date has passed
- Update your exam date to continue personalized planning
Buttons:
- Update Date
- Remove Date

14. Optional “Plan Without a Date” fallback
If user has not booked:
allow them to choose:
- 4-week prep
- 8-week prep
- 12-week prep
- custom target month

This generates a planning framework without a fixed exam date.

15. Analytics integration
If possible, integrate with current analytics to show:
- on track / needs focus / strong momentum
- strongest area
- weakest area
- recommended content mix this week

But keep this encouraging, not punitive.

Examples:
- Strong momentum in med-surg. Prioritize pharmacology this week.
- Your fundamentals performance is improving. Add one mixed exam this week.
- You’re building consistency. Keep focusing on high-yield review.

16. Reusable architecture
Build this in a reusable way so it can later support:
- push reminders
- milestone notifications
- AI coaching
- study schedule generation
- calendar integrations
- exam readiness forecasting

17. QA requirements
Verify:
- exam date logic is accurate
- weekly plan updates as time passes
- phase logic changes correctly by date range
- user can hide/show planner
- planner supports mobile and desktop
- planner is visible only on Nursing and Allied Health tiers
- recommendations do not break if some analytics data is missing
- no harsh or anxiety-inducing wording
- resetting or changing date recalculates the plan correctly

18. Final product expectation
This should not feel like a generic countdown widget.
It should feel like a polished premium exam planning system that helps users understand:
- how long they have
- what phase they are in
- what to focus on next
- how much to do each week
- how to study smarter based on their actual timeline and progress

Please implement this as a production-quality feature, consistent with the NurseNest brand and dashboard experience.
