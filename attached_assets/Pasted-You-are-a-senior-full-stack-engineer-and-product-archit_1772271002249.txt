You are a senior full-stack engineer and product architect.

Goal:
Implement a freemium flashcard system with the following rules:

FREE PLAN:
- Users can create up to 300 total flashcards across all decks.
- After reaching 300 cards, they cannot create new flashcards unless they upgrade.
- They can still view/edit existing cards.
- They can delete cards to free up space.

PAID PLAN:
- Unlimited flashcards.
- Must be cheaper than Quizlet+ (assume Quizlet+ ≈ $7.99–$8.99/month).
- Set pricing to:
    Option A: $4.99/month
    Option B: $39/year
- Must integrate with Stripe.
- Must support subscription status tracking.

Technical Requirements:

1) Database Changes
Add fields to users table:
- plan_type (free|pro)
- flashcard_limit (default 300 for free, null for pro)
- stripe_customer_id
- stripe_subscription_id
- subscription_status

Add flashcard counter logic:
- Count total flashcards per user.
- Do NOT trust client-side counts.
- Enforce limit server-side before insert.

2) Enforcement Logic
Before creating a flashcard:
- Query total count for user.
- If plan_type=free AND count >= 300:
    return 403 with message:
    "You’ve reached your free 300 card limit. Upgrade to continue."
- If user deletes cards, recalculate count dynamically.

3) Upgrade Flow
- Add /upgrade page.
- Show pricing comparison:
   "Cheaper than Quizlet+"
- Stripe Checkout integration.
- On successful payment:
   - Set plan_type=pro
   - Remove flashcard limit

4) UI Changes
- Display current count:
   "247 / 300 free cards used"
- Add progress bar.
- Show upgrade CTA at 80% usage.
- Hard stop modal at 300.

5) Abuse Protection
- Prevent multiple free accounts via:
   - Email verification
   - Optional device fingerprint check
- Prevent API bypass of limit.
- Rate-limit flashcard creation endpoint.

6) Admin Controls
- Admin can override limits.
- Admin can set custom limits.
- Admin can comp premium accounts.

7) Future Scalability
- Make limits configurable via environment variables:
   FREE_FLASHCARD_LIMIT=300
   PRO_FLASHCARD_LIMIT=null

Deliverables:
- DB migrations
- API enforcement code
- Stripe integration
- UI update
- Test cases:
   - User at 299 creates card
   - User at 300 blocked
   - User deletes card and creates new
   - User upgrades and can exceed 300
