You are implementing a **Site Issue Reporting System** across the entire NurseNest platform (https://www.nursenest.ca).

The goal is to allow users to **easily report problems with any page, lesson, question, translation, or feature** directly from the interface using a small flag/report button.

This system must work across **all pages of the site**, including:

Lessons
Practice questions
Allied health sections
New grad resources
Blog posts
Study guides
Exam simulations

Do NOT break the current navigation system or page layout.

---

CRITICAL RULES

Do NOT modify the navigation structure.
Do NOT move existing page elements.
Do NOT slow down page performance.
Do NOT allow spam submissions.
Do NOT break multilingual support.

The reporting feature must be **lightweight and unobtrusive**.

---

STEP 1 — ADD A REPORT ICON TO EVERY PAGE

Add a small flag icon labeled:

"Report Issue"

Place this icon in a consistent location on every page, such as:

Bottom-right corner of the page
Near the lesson header
Near question explanations

The icon must be visible but not distracting.

---

STEP 2 — OPEN A REPORT MODAL

When users click the flag icon, open a modal window titled:

"Report a Problem"

The modal should include a short message:

"Help us improve NurseNest. If you notice an issue with this page, please let us know."

---

STEP 3 — ISSUE TYPE SELECTION

Provide selectable issue categories:

Incorrect information
Question error
Explanation unclear
Broken link
Translation issue
Technical problem
Typo or grammar mistake
Other

Users must select at least one category.

---

STEP 4 — USER COMMENT FIELD

Include a text box where users can describe the issue.

Label:

"Describe the problem"

Allow up to 1000 characters.

---

STEP 5 — AUTOMATIC PAGE CONTEXT

When the report is submitted, automatically attach:

Page URL
Lesson ID or question ID
Language selected
User account ID (if logged in)
Timestamp

This ensures developers know exactly where the issue occurred.

---

STEP 6 — OPTIONAL SCREENSHOT CAPTURE

If possible, allow users to attach a screenshot.

Screenshots should help developers identify UI problems.

---

STEP 7 — SEND REPORT TO ADMIN SYSTEM

All reports must be stored in a database table such as:

issue_reports

Fields should include:

report_id
page_url
content_id
issue_type
user_comment
language
user_id
timestamp
status

---

STEP 8 — ADMIN REVIEW DASHBOARD

Create an admin dashboard where developers can review reports.

Features should include:

Issue list
Sorting by date
Filtering by issue type
Filtering by page
Status labels

Status options:

Open
In review
Resolved
Dismissed

---

STEP 9 — NOTIFICATION SYSTEM

When a report is submitted:

Send a notification to the admin panel.

Optionally send email alerts for critical issues such as:

Broken pages
Incorrect answers in questions
System errors

---

STEP 10 — MULTILANGUAGE SUPPORT

Ensure the report button and modal are translated in all supported languages.

Example:

English
Report Issue

French
Signaler un problème

Spanish
Reportar un problema

---

STEP 11 — PREVENT SPAM

Implement basic protection:

Rate limit submissions per user
Block repeated submissions
Use CAPTCHA if necessary

---

STEP 12 — SUCCESS MESSAGE

After submission display:

"Thank you for helping improve NurseNest. Our team will review your report."

---

FINAL RESULT

After implementation, every page on the NurseNest platform will allow users to quickly report issues.

This will allow the team to rapidly detect:

Content errors
Broken links
Translation problems
Technical bugs

The system will continuously improve the quality and accuracy of the platform.
