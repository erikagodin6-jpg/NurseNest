We need to redesign the NurseNest practice question / flashcard interface to make it clean, modern, and similar in usability to professional exam-prep platforms (UWorld-style experience) while maintaining the NurseNest aesthetic.

The current UI has major problems:
• The layout forces a two-column design with a large empty left side.
• Images appear beside the question and create visual imbalance.
• The interface looks sparse and unfinished.
• The rationale box appears disconnected from the question.
• The question card wastes too much horizontal space.
• The UI does not feel like a focused exam experience.

GOAL
Create a single-column, exam-focused question layout where the learner's attention stays on the question and answers.

IMPORTANT RULE
Educational images should ONLY appear in the rationale section unless the image is required to answer the question.

STRUCTURAL CHANGES

1. REMOVE THE CURRENT TWO-COLUMN LAYOUT
Delete the left image column entirely.

The question UI must become a centered single-column layout.

Max width:
max-width: 820px
margin: auto
padding: 24px

2. QUESTION CARD STRUCTURE

The card should appear in this order:

Question container
------------------

Question header
• question number
• exam mode indicator

Question stem
• large readable text
• line-height optimized for reading

Answer options
• full width stacked buttons
• soft rounded corners
• subtle hover state

Submit button

Rationale panel (hidden until answer submitted)

3. IMAGE DISPLAY RULES

Images must NOT appear beside the question anymore.

Images should only render inside the rationale section unless the image is required for answering.

Add a schema distinction in the question model:

questionImage
rationaleImage
imageRequiredForAnswer (boolean)

Rendering logic:

IF imageRequiredForAnswer = true
    show image ABOVE the question stem

IF imageRequiredForAnswer = false
    hide image during question phase
    show image inside rationale panel after submission

4. RATIONALE PANEL DESIGN

The rationale panel should expand after the learner answers.

Structure:

RATIONALE CARD

Header
"Explanation"

Optional rationale image
(if present)

Explanation text

Clinical takeaway / pearl section

Optional lesson link

Styling requirements:

• soft card background
• rounded corners
• padding 20px
• comfortable reading width
• subtle divider lines
• image max width 100%

The rationale must feel like a teaching panel, not a plain text box.

5. ANSWER OPTION DESIGN

Answer choices should be modern and clean.

Requirements:

• full width
• rounded corners
• soft border
• hover state
• selected state
• correct/incorrect state after submission

Correct answer:
soft green highlight

Incorrect answer:
soft red highlight

User selected answer:
accent outline

Avoid harsh black backgrounds.

6. TYPOGRAPHY HIERARCHY

Question stem:
font-size: 20px
font-weight: 600

Answer options:
font-size: 16px

Rationale text:
font-size: 15px
line-height: 1.6

7. VISUAL CLEANUP

Remove:
• the empty left image space
• oversized containers
• disconnected rationale box

Add:
• better spacing between sections
• subtle card shadows
• improved reading width
• consistent margins

8. MOBILE BEHAVIOR

Mobile must remain single column.

Ensure:
• no horizontal scroll
• buttons are full width
• rationale images scale properly
• readable text spacing

9. DATABASE HANDLING

If current questions only have one image field:

If imageRequiredForAnswer = false
move image into rationaleImage

If imageRequiredForAnswer = true
keep image as questionImage

Update rendering logic accordingly.

10. RESULT

The final UI should feel like a professional exam prep interface:
• focused
• centered
• modern
• visually balanced
• optimized for studying

No empty columns.
No floating images beside the question.
Images only appear in rationale unless required to answer.

Implement this refactor in the current question component.
