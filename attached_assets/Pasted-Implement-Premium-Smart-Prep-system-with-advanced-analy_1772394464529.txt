Implement Premium Smart Prep system with advanced analytics, peer comparison, gated features, and performance intelligence.

CRITICAL RULES:
- No feature is visible without passing data integrity checks.
- Peer statistics must be hidden until minimum sample size reached.
- All analytics must be derived from real stored attempt data.
- No fabricated percentages.
- All premium features must be properly gated.

================================================

================================================
PART 2 — QUESTION ANALYTICS ENGINE
================================================

For each question track:
- totalAttempts
- totalCorrect
- percentCorrect
- lastUpdated
- uniqueUserCount

Update stats on every submission.

Compute:
percentCorrect = (totalCorrect / totalAttempts) * 100

Difficulty classification:
- Easy: >70%
- Moderate: 40–70%
- Difficult: <40%

================================================
PART 3 — MINIMUM SAMPLE SIZE GATE (CRITICAL)
================================================

Peer stats must NOT be shown unless:
- totalAttempts >= 30
- uniqueUserCount >= 15

If below threshold:
Display:
"Peer performance data available once more students answer this question."

This prevents misleading early percentages.

================================================
PART 4 — DISPLAY LOGIC (POST-ANSWER ONLY)
================================================

After user submits answer:
Show:
- Correct / Incorrect
- Rationale
- Clinical pearl

If premium AND sample size threshold met:
- Show: "43% of users answered this correctly"
- Show difficulty badge
- Show topic percentile

If free user:
- Blur peer stats
- Show upgrade prompt

Do NOT show peer stats before answer submission.

================================================
PART 5 — PERCENTILE RANKING SYSTEM
================================================

After full-length exam:

Store:
- examScore
- domainScores
- timestamp

Compute percentile:
percentile = rank among users with >= 1 completed exam
Exclude:
- users with < 50 total answered questions

Premium users see:
- Percentile rank
- Domain percentile
- Trend over time

Free users see:
- Raw score only
- Upgrade prompt for percentile insight

================================================
PART 6 — READINESS SCORE (PREMIUM ONLY)
================================================

Compute Readiness Score:

readinessScore =
(RecentExamAverage * 0.6) +
(DomainBalanceScore * 0.2) +
(ImprovementTrendScore * 0.2)

Display levels:
- <65: Not Ready
- 65–75: Developing
- 75–85: On Track
- >85: Exam Ready

Always include disclaimer:
"Readiness score is an internal performance estimate, not a guarantee of exam outcome."
