Implement an Exam Completion Follow-Up system for NurseNest that triggers after a user’s exam date and collects feedback while guiding them to the next stage of their journey.

Goal:
Create a supportive post-exam flow that:
• asks how the exam went
• gathers insight into weak areas
• congratulates successful users
• offers next-step support such as New Grad resources
• optionally offers a transition discount
• improves retention and product analytics

Context:

* Platform: NurseNest
* Stack: Vite + React + Express
* Existing features: exam date countdown and AI study planner
* Audience: nursing tiers (RPN, RN, NP) and allied health tiers

This system should feel encouraging and celebratory, not transactional or stressful.

---

PART 1: Trigger Logic

When the current date is greater than the user’s stored examDate:

1. Detect that the exam date has passed.
2. Wait 24–48 hours before triggering the follow-up prompt.
3. Display a modal or dashboard card titled:

"How Did Your Exam Go?"

Do not show this repeatedly if already completed.

Add a field to the user model:

examFollowupCompleted: boolean

---

PART 2: Follow-Up Modal

The modal should include supportive language:

Headline:
How did your exam go?

Subtext:
Your feedback helps us improve NurseNest and guide you to the right next step in your nursing or healthcare career.

Provide selectable responses:

• I passed
• I’m waiting for results
• I didn’t pass
• I decided to postpone

Allow the user to skip if they prefer.

---

PART 3: Conditional Follow-Up Paths

PATH A: User Passed

Display celebratory message:

"Congratulations! You did it."

Offer next steps:

• Transition to New Grad Support
• Access career tools
• Prepare for your first healthcare job

Offer a limited discount:

Example:
"Celebrate your success with 20% off NurseNest New Grad tools."

Buttons:
Explore New Grad Resources
Claim Discount
Dismiss

Add optional features:

* badge or confetti animation
* celebratory dashboard message

---

PATH B: Waiting for Results

Message:

"Waiting for results can be stressful. While you wait, you can review key concepts or explore next-step resources."

Offer:

• light review content
• new grad resources preview
• ability to update results later

Buttons:
Update Result Later
Explore Resources

---

PATH C: Did Not Pass

Important: messaging must be supportive.

Message example:

"Many great nurses and healthcare professionals need more than one attempt. We’re here to help you succeed next time."

Ask follow-up question:

"What areas felt most challenging?"

Allow multi-select:

• Pharmacology
• Prioritization
• Fundamentals
• Clinical judgement
• Med-surg
• Pediatrics
• Maternity
• Mental health
• Time management
• Exam anxiety

After selecting:

Offer a personalized restart plan:

"We can build a new study plan focused on your weak areas."

Buttons:
Create a New Study Plan
Review Weak Areas
Retake Preparation

---

PATH D: Exam Postponed

Message:

"No problem. Let’s update your target exam date and keep your study plan on track."

Buttons:
Update Exam Date
Continue Study Plan

---

PART 4: Analytics Collection

Store user responses to improve platform insights.

Add fields:

examResultStatus
examWeakAreas
examFollowupCompleted
examResultDate

These data points will help generate:

• pass rate estimates
• weak area trends
• product improvement insights

Do not expose individual results publicly.

---

PART 5: Dashboard Integration

After completing the follow-up flow:

If user passed:
Replace the exam countdown card with:

"Next Step: Launch Your Healthcare Career"

Show:
• New Grad tools
• career guides
• clinical transition resources

If user did not pass:
Replace countdown with:

"Your Next Study Plan"

Show:
• new targeted study plan
• weak area review
• suggested mock exams

If user postponed:
Re-enable the exam planner with new date.

---

PART 6: Optional Email Follow-Up

If the user has not responded within 3 days after the exam date:

Send a gentle email:

Subject:
How did your exam go?

Body:
We hope your exam went well. Let us know how it went so we can help you with the next step of your journey.

Include link to the follow-up modal.

---

PART 7: Discount Logic

If the user selects "I passed":

Generate a one-time coupon code:

NEWGRAD20

Valid for:
New Grad resources

Expiration:
14 days

Show the discount visually in the modal.

---

PART 8: UX Requirements

Design should match the NurseNest dashboard aesthetic:

• soft pastel palette
• rounded cards
• supportive tone
• not overly celebratory or overwhelming
• responsive for mobile

Avoid dark red error messaging or negative language.

---

PART 9: QA Checklist

Ensure:

• modal appears only after exam date passes
• user can skip and return later
• responses persist in database
• correct flow appears for each result
• discount appears only for passing path
• mobile layout works correctly
• no infinite loops of the modal
• dashboard updates correctly after completion

---

PART 10: Product Vision

This feature should transform NurseNest into a full professional journey platform:

Exam Prep → Exam Result → Career Launch → New Grad Support

The system should guide users smoothly through this transition while providing value at each stage.
