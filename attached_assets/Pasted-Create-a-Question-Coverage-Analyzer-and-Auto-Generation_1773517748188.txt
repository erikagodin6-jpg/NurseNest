Create a **Question Coverage Analyzer and Auto-Generation System** for the NurseNest platform.

The goal is to ensure every tier, specialty, and exam category has sufficient question coverage and that missing questions are automatically generated to meet minimum targets.

This system must analyze the entire question database and generate missing questions where needed.

The analyzer must work across all learning products including:

• Test Bank questions
• Exam questions
• Mock exam questions
• Flashcard-derived questions

The system should run as an admin tool and also optionally as a scheduled maintenance job.

ADMIN DASHBOARD

Add a new section:

Admin → Content Coverage

This dashboard should display:

• total questions per tier
• total flashcards per tier
• questions per topic
• questions per body system
• questions per exam category
• missing or underrepresented topics

Provide visual charts and lists.

COVERAGE TARGETS

Set minimum coverage thresholds.

Default targets:

Nursing tiers (RPN, RN, NP):

• minimum 300 questions per major body system
• minimum 500 questions per specialty
• minimum 100 questions per subtopic

Allied health programs:

• minimum 300 questions per profession
• minimum 75 questions per topic

Flashcards:

• minimum 25 flashcards per topic

These targets should be configurable from the admin panel.

DATABASE ANALYSIS

Scan the question database and group questions by:

• tier (RPN, RN, NP, Allied)
• specialty
• body system
• topic
• subtopic
• difficulty level

Calculate coverage statistics.

Detect areas where coverage is below the target threshold.

Examples:

Cardiology RN:
Current questions = 212
Target = 500
Missing = 288

Respiratory therapy pharmacology:
Current questions = 34
Target = 75
Missing = 41

These deficits should appear on the Content Coverage dashboard.

AUTO QUESTION GENERATION

For each deficit detected, automatically generate new exam-quality questions until the target count is reached.

Generated questions must include:

• clinical scenario
• question stem
• four or five answer options
• realistic distractors
• correct answer
• detailed rationale
• clinical pearls
• difficulty rating
• topic metadata
• tier metadata

Questions must reflect realistic nursing exam styles such as:

• NCLEX style
• REx-PN style
• NP certification style

Use clinical reasoning questions rather than simple recall where possible.

QUESTION QUALITY RULES

Generated questions must:

• avoid duplicate question stems
• vary patient demographics and scenarios
• include realistic clinical data such as lab values or vital signs
• avoid overly obvious distractors
• maintain clinical accuracy

DIFFICULTY DISTRIBUTION

Ensure question difficulty distribution remains balanced:

• Easy: 30%
• Moderate: 50%
• Difficult: 20%

Difficulty should be determined based on clinical reasoning complexity.

TOPIC BALANCING

When generating questions for a topic, vary subtopics within that system.

Example for cardiology:

• heart failure
• myocardial infarction
• arrhythmias
• cardiogenic shock
• valve disorders

Avoid producing large numbers of questions on only one subtopic.

FLASHCARD GENERATION

If a topic has fewer than the required number of flashcards:

Automatically generate flashcards including:

• term-definition format
• clinical concept format
• exam recall format

Each flashcard must include a clear explanation.

FLASHCARD TO QUESTION CONVERSION

Allow flashcards to automatically generate corresponding test bank questions when appropriate.

Example:

Flashcard concept:
ACE inhibitors reduce afterload.

Generated question:
Which medication class reduces afterload by inhibiting angiotensin-converting enzyme?

BATCH PROCESSING

Generation must occur in batches to avoid timeouts.

Recommended batch size:

• 50 questions per batch
• 25 flashcards per batch

Allow multiple batches to run sequentially until targets are reached.

Do not terminate generation due to long runtime.

Allow generation to continue until completion.

DUPLICATE DETECTION

Before inserting a new question:

Check the database for similar question stems.

If similarity exceeds a defined threshold, regenerate the question.

This prevents near-duplicate content.

IMAGE INTEGRATION

If a question topic would benefit from a diagram or visual reference, attach an appropriate existing image or generate a new one.

Examples:

• ECG rhythms
• anatomical diagrams
• lab interpretation charts

Images must be relevant and clinical.

PRODUCTION DATABASE

All newly generated content must be written directly to the production database.

Ensure:

• proper tier assignment
• proper topic tagging
• proper difficulty labeling
• correct metadata fields populated

Do not write incomplete questions to the database.

VALIDATION PIPELINE

Each generated question must pass validation checks:

• stem length within acceptable range
• rationales present
• answer choices valid
• metadata present

If validation fails, regenerate the question.

ADMIN CONTROLS

Add controls to the Content Coverage dashboard allowing admin to:

• regenerate missing questions
• regenerate flashcards
• adjust coverage targets
• disable generation for specific topics
• run the coverage analyzer manually

CONTENT COVERAGE REPORT

After each generation cycle produce a report showing:

• number of questions generated
• number of flashcards generated
• topics that reached coverage targets
• topics still below threshold

Display this report in the admin dashboard.

GOAL

The final system should ensure that:

• every tier has robust question coverage
• every major specialty has hundreds of exam questions
• flashcards exist for all key topics
• gaps in content are automatically filled
• the question bank continues expanding over time
• NurseNest maintains exam-level depth comparable to major exam prep platforms.
