Implement a Career Stage Engine for NurseNest that automatically adapts the user dashboard and product experience based on where the user is in their professional journey.

Goal:
Create a lifecycle-aware platform that transitions users smoothly from exam prep to career development.

Stages to support:

1. Pre-Exam Student
2. Exam Candidate
3. Awaiting Results
4. Passed Exam
5. New Grad
6. Practicing Clinician

The system should update dashboard content, messaging, recommendations, and offers automatically as the user moves between these stages.

Platform context:

* Stack: Vite + React + Express
* Existing features:

  * Readiness Predictor
  * Exam Countdown
  * AI Study Planner
  * Post-Exam Follow-Up Flow
* Audience:

  * Nursing tiers (RPN, RN, NP)
  * Allied health tiers

The goal is to keep users inside the NurseNest ecosystem throughout their entire professional journey.

---

PART 1: Career Stage Data Model

Add a careerStage field to the user profile.

Possible values:

pre_exam_student
exam_candidate
awaiting_results
passed_exam
new_grad
practicing_clinician

Additional fields:

examDate
examResultStatus
careerStageLastUpdated
newGradResourcesActivated

Career stage should update automatically based on:

* exam date
* post-exam survey response
* user selections

---

PART 2: Stage Detection Logic

Stage 1: Pre-Exam Student

Criteria:

* user signed up
* no exam date entered

Dashboard shows:

* readiness predictor
* study tools overview
* prompt to add exam date
* AI study planner preview

Primary messaging:
"Start building your exam preparation plan."

---

Stage 2: Exam Candidate

Criteria:

* examDate exists
* exam date still in future

Dashboard shows:

* exam countdown
* AI study planner
* readiness score
* weekly targets
* weak area recommendations

Primary messaging:
"Prepare confidently for your upcoming exam."

---

Stage 3: Awaiting Results

Criteria:

* exam date passed
* user selected "waiting for results"

Dashboard shows:

* results pending card
* light review recommendations
* encouragement messaging

Primary messaging:
"Waiting for results? Keep your knowledge sharp while you wait."

---

Stage 4: Passed Exam

Criteria:

* user selects "I passed"

Dashboard shows:

* celebration banner
* career transition card
* new grad resources introduction

Primary messaging:
"Congratulations. You're ready for the next stage of your career."

---

Stage 5: New Grad

Criteria:

* user activates new grad resources
* user accepts new grad offer
* or manually selects new grad stage

Dashboard shifts from exam prep to career support.

Show:

* clinical shift guides
* prioritization practice
* report templates
* transition-to-practice resources
* job interview preparation
* confidence-building modules

Primary messaging:
"Build confidence in your first year of practice."

---

Stage 6: Practicing Clinician

Criteria:

* user has been in new grad stage for a defined period
* or manually selects advanced career stage

Dashboard shows:

* continuing education
* specialty certifications
* advanced practice learning
* leadership resources

Primary messaging:
"Continue growing your clinical expertise."

---

PART 3: Dashboard Adaptation

The dashboard layout should remain visually consistent, but the content modules should adapt based on careerStage.

Examples:

Exam Candidate dashboard:

* countdown
* study planner
* readiness predictor
* weak area analytics

New Grad dashboard:

* shift survival tools
* prioritization practice
* patient scenario simulations
* workplace readiness tools

Practicing Clinician dashboard:

* specialty learning
* certification prep
* advanced clinical modules

---

PART 4: Transition Prompts

Transitions should feel natural and supportive.

Examples:

Exam Candidate → Awaiting Results
"Your exam date has passed. How did it go?"

Awaiting Results → Passed
"Congratulations! Let's help you succeed in your first healthcare role."

Passed → New Grad
"Ready to transition from exam prep to real-world practice?"

New Grad → Practicing Clinician
"Continue developing your clinical expertise."

---

PART 5: Offer Logic

Offer targeted upgrades based on stage transitions.

Examples:

Passed → New Grad
Offer:
20% off New Grad resources for 14 days.

New Grad → Advanced Learning
Offer:
Specialty certification preparation tools.

Ensure offers appear as helpful suggestions, not aggressive sales prompts.

---

PART 6: Dashboard Messaging System

Implement a stage-aware messaging component that updates automatically.

Example:

If careerStage = exam_candidate:
"Your exam is in 42 days. Stay on track with your study plan."

If careerStage = new_grad:
"Focus on real-world patient care and clinical decision-making."

---

PART 7: Analytics Integration

Track stage transitions.

Data points:

careerStageHistory
examOutcome
timeSpentInEachStage
conversionToNewGradResources

Use this data to improve:

* retention
* feature development
* educational insights

---

PART 8: UX Design Requirements

Maintain NurseNest visual style:

* pastel clinical palette
* rounded cards
* soft shadows
* calm typography hierarchy

Transitions between stages should feel smooth and encouraging.

Avoid alarming colors or anxiety-inducing messaging.

---

PART 9: QA Checklist

Verify:

* careerStage updates automatically when appropriate
* manual overrides work correctly
* dashboards update correctly for each stage
* offers trigger at the correct time
* no infinite loops of transition prompts
* mobile and desktop layouts remain clean
* readiness predictor and study planner continue functioning properly
* transitions do not break existing analytics features

---

PART 10: Long-Term Vision

This Career Stage Engine should support a complete professional journey:

Exam Preparation
→ Exam Success
→ Career Launch
→ Clinical Development
→ Advanced Practice

NurseNest should evolve from a simple exam prep tool into a long-term healthcare professional development platform.
