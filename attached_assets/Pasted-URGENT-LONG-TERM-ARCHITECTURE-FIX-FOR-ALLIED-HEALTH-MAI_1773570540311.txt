URGENT LONG-TERM ARCHITECTURE FIX FOR ALLIED HEALTH + MAIN NURSENEST

Problem:
The Allied Health experience currently behaves like a disconnected site/subdomain instead of a seamless part of NurseNest.

Confirmed UX problems:
- some allied pages route to a different-looking shell
- some links still go to the allied subdomain unexpectedly
- there is no clear path back to the main NurseNest site from allied pages
- navigation consistency is broken across main NurseNest and allied pages
- users can get trapped in allied pages with limited brand continuity
- footer and menu links are inconsistent or broken across sections

Goal:
Refactor the site architecture so Allied Health is fully integrated into the NurseNest ecosystem, with consistent branding, navigation, routing, and cross-linking.

PRIMARY PRODUCT RULE:
Allied Health must feel like a section of NurseNest, not a disconnected website.

--------------------------------------------------
SECTION 1 — AUDIT CURRENT ARCHITECTURE
--------------------------------------------------

1. Audit the current architecture for:
- main NurseNest domain
- allied subdomain
- allied routes on main domain
- route redirects
- shared layout components
- navigation components
- footer components
- header components
- locale routing behavior

2. Document:
- which allied pages are on main domain
- which allied pages are on subdomain
- which pages use shared layout
- which pages use a different shell
- which redirects are intentional vs legacy leftovers

3. Identify the root cause of the split experience:
- legacy subdomain architecture
- hardcoded links
- different router config
- different app shell
- different layout wrapper
- partial migration not completed

--------------------------------------------------
SECTION 2 — CHOOSE ONE CONSISTENT STRUCTURE
--------------------------------------------------

4. Normalize Allied Health into one architecture.

Preferred structure:
Keep Allied Health inside the main NurseNest experience with clean section-based routes such as:

/allied-health
/allied-health/paramedic
/allied-health/respiratory-therapy
/allied-health/pharmacy-tech
/allied-health/mlt
/allied-health/radiologic-tech

/allied-health/cardiac-sonography
/allied-health/occupational-therapy-assistant
/allied-health/physiotherapy-assistant
/allied-health/health-information-management

5. If the subdomain must remain temporarily, then:
- make it use the same NurseNest global shell
- make cross-navigation explicit
- preserve branding and layout
- remove all “disconnected site” behavior

6. Preferred end state:
Users should not need to understand whether they are on a subdomain or not.
It should feel like one product.

--------------------------------------------------

--------------------------------------------------

- logo

- language switcher
- account/menu behavior

9. If there are legacy templates without shared layout, migrate them into the shared layout wrapper.

--------------------------------------------------

--------------------------------------------------

10. Add a persistent return path from every allied page back to main NurseNest.

Required:
- NurseNest logo links to main homepage
- ecosystem nav links back to:
  - Exam Prep
  - New Grad Support

- footer includes:
  - main homepage
  - pricing

  - new grad
  - FAQ

11. Add breadcrumb structure where appropriate, e.g.:
NurseNest > Allied Health > Pharmacy Technician

12. Ensure users can get from any allied page back to the main site in one click.

--------------------------------------------------

14. Audit all allied-related routes and fix:

or

16. Do not leave mixed routing behavior where one allied page behaves one way and another behaves differently.
