The Real Fix (Architectural)
You do NOT need to rewrite your whole generator.
You need 3 structural protections:
1. Required Section Hard Validator (Server Side)
After AI generation, before compiling pages:
function validateSections(data: any, blueprint: TemplateBlueprint) {
  const errors: string[] = [];

  for (const section of blueprint.sections) {
    if (!section.required) continue;

    const content = data[section.id];

    if (!content || typeof content !== "string") {
      errors.push(section.id);
      continue;
    }

    const cleaned = content.trim();
    const bulletCount = (cleaned.match(/^- /gm) || []).length;

    if (cleaned.length < 800 && bulletCount < 6) {
      errors.push(section.id);
    }
  }

  return errors;
}
If errors.length > 0:
return res.status(422).json({
  error: "EMPTY_SECTION",
  sections: errors,
  rawPreview: raw.substring(0, 1500)
});
DO NOT compile if validation fails.
2. Question Batching (You Cannot Generate 300 at Once)
Even GPT-4 class models struggle with 300 questions + rationales in one pass.
Server logic should be:
async function generateQuestions(total: number) {
  const batchSize = 25;
  let allQuestions: any[] = [];

  while (allQuestions.length < total) {
    const remaining = total - allQuestions.length;
    const count = Math.min(batchSize, remaining);

    const batch = await callLLM(count);

    if (!Array.isArray(batch)) throw new Error("Invalid batch");

    allQuestions = [...allQuestions, ...batch];
  }

  return allQuestions;
}
If you don’t do this, the model truncates.
That’s exactly why you got Q1–Q5 and then nothing.
3. Enforce Structured JSON Format Strictly
Your current parseAIJsonResponse just tries to extract {...}.
That is fragile.
You need to force this response shape:
{
  "sections": {
    "learning-objectives": "...",
    "pathophysiology": "...",
    ...
  },
  "questions": [
    {
      "stem": "",
      "options": ["", "", "", ""],
      "correct": "A",
      "rationale": ""
    }
  ]
}
Then validate:
if (!parsed.sections || !parsed.sections["pathophysiology"]) throw new Error();
Right now, your generator probably expects top-level keys, and if AI nests them differently, content silently disappears.
