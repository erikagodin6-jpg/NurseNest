You are acting as a senior technical SEO engineer and performance architect. Your job is to perform a complete structural, indexing, crawlability, and performance overhaul of my educational platform so that Google can efficiently crawl, index, and rank the highest-value pages.

My site is a large nursing education platform with thousands of programmatically generated pages including lessons, study guides, practice questions, exams, and category hubs. Google Search Console recently showed a sharp drop in impressions (from around 120 per day to about 13), along with a very large number of URLs labeled “Discovered – currently not indexed”. This indicates that Google is discovering many URLs but choosing not to index them.

The objective is to correct the root cause by implementing a quality-first indexing strategy, improving crawl efficiency, strengthening page quality signals, fixing canonical and sitemap issues, optimizing performance, and restructuring internal linking so Google clearly understands which pages are most important.

You must perform the following tasks across the codebase.

First perform a complete crawl and indexation audit of the site. Identify every type of URL currently generated or accessible, including lesson pages, study guides, question pages, exam pages, search or filter pages, tag or archive pages, category pages, language variants, parameterized URLs, preview routes, and pagination routes. Determine which of these should be indexable and which should not. Create a clear classification of indexable pages versus pages that should be excluded from search engines.

Implement a strict indexing policy where only high-value educational pages are indexable. All low-value, repetitive, or utility pages must be excluded from search indexing. Pages that should remain indexable include core lessons, long study guides, educational blog pages, category hubs, and major exam preparation landing pages. Pages that should not be indexed include individual question pages, test session interfaces, account pages, admin tools, filtered search pages, query parameter URLs, duplicate language routes, pagination pages, and other thin or utility pages.

For all pages that should not appear in search results, implement the correct exclusion strategy. Depending on the page type, apply either noindex meta tags, robots directives, canonical consolidation, or robots.txt exclusions so that Google does not waste crawl budget on those URLs.

Next audit canonical tags throughout the application. Ensure every indexable page uses a correct self-referencing canonical URL. Fix any incorrect canonical tags that point to the wrong page, duplicate language versions, alternate routes, or dynamically generated duplicates. If multiple URLs currently serve the same content, consolidate them so that only one canonical version remains indexable.

After correcting canonical issues, rebuild the sitemap system. Generate a clean sitemap architecture that contains only indexable canonical URLs. Exclude any pages that are noindex, parameterized, duplicate, or low-value. Include last modified timestamps so search engines can identify fresh content. If the sitemap grows large, split it into multiple files and create a sitemap index file. Ensure that the sitemap only represents the strongest pages we want Google to crawl and rank.

Improve the internal linking architecture so that search engines clearly understand the site hierarchy. Each lesson page should link to related lessons, its parent category hub, and related study guides. Category pages should link to all child lessons and major topic clusters.
