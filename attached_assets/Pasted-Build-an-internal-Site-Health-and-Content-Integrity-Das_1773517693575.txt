Build an internal **Site Health and Content Integrity Dashboard** inside the NurseNest admin system and run a full audit of the platform to detect missing pages, incomplete lessons, broken links, SEO issues, and unfinished advertised content.

The system must also automatically generate missing lesson content when pages are advertised but do not exist.

The goal is to ensure that the public site **never advertises content that does not exist** and that the platform remains technically healthy.

This system must work across the entire NurseNest ecosystem including:

• Nursing tiers (RPN, RN, NP)
• Allied Health
• New Grad
• Lessons
• Test Bank
• Flashcards
• Exams
• Mock Exams
• Blog content
• Pricing pages

ADMIN DASHBOARD

Create a new admin section:

Admin → Site Health

This dashboard should display real-time diagnostics including:

• Broken links detected
• Pages returning 404 errors
• Missing lesson pages
• Lessons referenced but not generated
• Missing flashcards
• Missing exam questions
• Missing images
• Missing SEO metadata
• Sitemap inconsistencies
• Revenue vs expenses tracking

The dashboard should include color-coded indicators:

Green = healthy
Yellow = warning
Red = critical issue

BROKEN LINK SCANNER

Create a system that scans all internal links across the site including:

• navigation menus
• dropdown menus
• hero buttons
• CTA buttons
• feature cards
• lesson links
• flashcard links
• exam links
• program pages
• breadcrumbs
• blog internal links
• sitemap entries

The scanner should detect:

• 404 routes
• routes that do not exist
• links pointing to pages without content

Broken links should appear on the Site Health dashboard.

MISSING PAGE DETECTION

Detect pages that are referenced but do not exist.

Examples:

• a lesson listed in a curriculum but missing its page
• a topic card that links to an undefined route
• an exam topic with no questions
• a flashcard topic without cards

The system should automatically flag these as **content integrity errors**.

CONTENT COMPLETION SYSTEM

If a page is advertised but does not exist, automatically generate the missing content.

Examples:

If a lesson title appears in navigation but the page is missing, automatically generate:

• full lesson page
• structured headings
• clinical explanations
• exam-focused summaries
• images where appropriate
• related flashcards
• related test bank questions

The system should prioritize content generation for:

• lessons referenced in navigation
• topics shown on program pages
• topics linked from flashcards
• topics linked from test bank questions

LESSON GENERATION STANDARDS

Generated lessons must include:

1. Clear clinical explanation of the condition or topic
2. Pathophysiology section
3. Signs and symptoms
4. Diagnostic evaluation
5. Nursing interventions
6. Pharmacology considerations
7. Clinical pearls for exam preparation
8. Key takeaways

Images should be included when appropriate.

Avoid cartoon-style illustrations. Use realistic clinical diagrams when possible.

TIER-SPECIFIC CONTENT

Ensure content difficulty matches the tier:

RPN lessons should emphasize:

• fundamentals
• safety
• medication administration
• core nursing care

RN lessons should include:

• advanced pathophysiology
• multi-system reasoning
• pharmacology depth

NP lessons should include:

• differential diagnosis
• advanced clinical decision making
• diagnostic workup

Do not mix tier content incorrectly.

FLASHCARD COMPLETION

Scan all lesson topics.

If a topic has no flashcards, automatically generate flashcards including:

• term-definition cards
• clinical scenario cards
• exam-style recall cards

Flashcards should include:

• concise explanations
• exam relevance

TEST BANK COMPLETION

If a topic exists but has no test bank questions:

Generate exam-style questions including:

• multiple choice options
• realistic distractors
• correct answer
• detailed rationale
• clinical pearls

Ensure questions align with licensing exam formats.

IMAGE VALIDATION

Scan lesson pages for missing images.

If an image placeholder exists but the image file is missing, regenerate the image using the existing image generation system.

Ensure images are relevant to the lesson topic.

SEO VALIDATION

Verify that every public page includes:

• title tag
• meta description
• canonical URL
• open graph metadata

Detect duplicate metadata and correct it.

ROUTE VALIDATION

Compare all internal links with the router configuration.

Detect links pointing to routes that are not defined.

Fix navigation references automatically when possible.

SITEMAP VALIDATION

Ensure the sitemap only includes valid pages.

Remove URLs that return 404 errors.

Ensure all major public pages are included.

CONTENT COVERAGE REPORT

Generate a report showing:

• total lessons per tier
• lessons missing content
• flashcard coverage by topic
• test bank coverage by topic

This report should appear on the Site Health dashboard.

REVENUE AND COST TRACKING

Add a section to the dashboard that tracks:

• total revenue from subscriptions
• Stripe revenue totals
• total expenses entered by the admin

Allow manual expense entry such as:

• domain costs
• hosting costs
• development tools
• AI generation costs

Display:

• total expenses
• total revenue
• break-even point

This allows the admin to monitor profitability.

AUTOMATED CONTENT GUARD

Implement a rule:

No page should appear publicly if the corresponding content does not exist.

If a page is missing content, either:

• generate the content automatically
or
• hide the link until content exists.

FINAL REPORT

After implementing the system generate a summary including:

• number of broken links repaired
• number of missing lessons generated
• number of flashcards generated
• number of test bank questions generated
• number of images regenerated
• SEO issues corrected

GOAL

The final system should ensure that:

• the NurseNest platform never links to empty pages
• all advertised lessons exist and contain full content
• the site automatically detects structural issues
• the admin dashboard provides a clear view of site health and profitability
