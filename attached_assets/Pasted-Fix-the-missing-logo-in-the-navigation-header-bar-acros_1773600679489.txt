Fix the missing logo in the navigation/header bar across NurseNest.

Problem
- The NurseNest logo is not displaying in the navigation bar/header.
- This needs to be fixed in the actual shared header/nav component, not patched on only one page.

Goal
- Make the correct logo render consistently in the navigation bar on all relevant pages and subdomains.
- Keep the current layout clean and aligned after the logo is restored.

What to inspect
1. Check the shared header/navigation component used across:
   - main NurseNest site
   - allied site
   - new grad site
   - tier pages
2. Verify whether the logo asset path is broken:
   - wrong file path
   - wrong import
   - missing asset
   - case-sensitive filename mismatch
   - bad environment/base URL handling
   - subdomain path issue
3. Check whether the logo is being hidden by CSS:
   - width/height set to 0
   - overflow hidden
   - display none
   - opacity 0
   - z-index issue
   - flex shrink/cropping
   - text color/background making it appear invisible
4. Check whether the component is conditionally rendering a fallback state that is suppressing the logo.
5. Check whether recent navigation/header edits broke the logo container sizing or asset loading.

Required fix
- Restore the correct logo in the header.
- Use the existing official NurseNest logo asset already intended for the site.
- Do not substitute a random placeholder.
- Do not redesign the header.
- Do not change the brand name styling unless needed to restore the intended logo lockup.
- Ensure the logo displays crisply and proportionally.
- Ensure the logo links to the correct home page for that site/subdomain.

Subdomain behavior
- Main site should show the NurseNest logo and link to the main home page.
- Allied site should show the correct allied/NurseNest branding and link to the allied home page if that is the intended behavior.
- If there are different approved logo variants for different sections, use the correct existing variant for each one.
- Do not mix logos between sites.

Layout rules
- Keep vertical centering correct.
- Keep spacing between logo and navigation items balanced.
- Do not let the logo crowd the menu or wrap awkwardly.
- Verify desktop, tablet, and mobile layouts.
- Make sure restoring the logo does not reintroduce the duplicate menu issue.

QA checklist
- Logo is visible in header on all intended pages
- Logo asset loads successfully with no broken image
- Correct logo variant appears on each site/subdomain
- Logo is aligned and sized properly
- Clicking the logo goes to the correct home page
- No header spacing regressions
- No duplicate nav controls return

Acceptance criteria
- The logo displays properly in the navigation bar everywhere it is supposed to
- The correct asset and route are used
- The header remains clean, consistent, and responsive
