- exam status is active/in progress

PART 3 — BACK BUTTON BEHAVIOR DURING ACTIVE CAT EXAM
When the user presses the browser Back button during an active CAT exam:
- Do not immediately allow navigation away.
- Intercept the navigation attempt.
- Show a modal dialog.

Modal content:
Title:
Leave exam?

Body:
You are currently in an active CAT exam. Leaving this session may submit or end your exam. Your progress, timer, and exam state could be affected.

Buttons:
- Continue Exam
- Submit and Exit

Behavior:
- Continue Exam:
  - dismiss modal
  - keep user on current exam screen
  - preserve timer and question state
- Submit and Exit:
  - run the proper exam submit/end flow
  - save final state safely
  - navigate to the correct post-exam results/summary/exit page
  - do not leave the exam in a corrupted or partial state

Important:
- Do not allow browser Back to reveal prior exam questions or previous adaptive states.
- Do not allow the user to step backward through question history in CAT mode.
- Do not rely on brittle hacks that create infinite history loops.
