QUESTION QUALITY REQUIREMENTS
------------------------------------------------

Every generated question must include:

question stem  
answer options  
correct answer  
detailed rationale  

Rationales must:

- explain why the correct answer is correct
- explain why distractors are incorrect when relevant
- match the learner level
- reflect clinical reasoning
- reinforce exam preparation

No question may publish without a rationale.

------------------------------------------------
QUESTION FORMATS
------------------------------------------------

Supported formats include:

MCQ  
SATA  
ORDERED / prioritization  
HOTSPOT (image-based where applicable)  
SCENARIO / CASE questions  

If advanced formats such as BOW_TIE or CASE_SERIES are supported by the system, the generator may include them.

------------------------------------------------
GENERATION BATCH RULES
------------------------------------------------

Questions must generate in small batches.

Default batch size: 5  
Maximum batch size: 10  

Never generate massive batches in one request.

------------------------------------------------
TAXONOMY RULES
------------------------------------------------

The generator must NOT create:

new categories  
new topics  
new navigation items  
new menu sections  

All generated content must map to the existing canonical taxonomy.

If a topic does not match exactly:

1. normalize synonyms
2. fuzzy match to an existing topic
3. fallback to parent body system category

Never auto-create taxonomy entries.

------------------------------------------------
PUBLISHING REQUIREMENTS
------------------------------------------------

All generated content must:

save into the production database  
publish to the live site automatically  
appear in the correct question banks  
appear in the correct study categories  
update analytics counts  

Generated content must never appear in orphan locations.
