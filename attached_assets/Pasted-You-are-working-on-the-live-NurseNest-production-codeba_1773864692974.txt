SECTION 1 — INCIDENT INVESTIGATION
==================================================

Perform a deep audit specifically focused on the exam system.

Investigate all of the following:

A. EXAM ROUTES
- inspect all exam-related frontend routes
- identify routes that hang, crash, overfetch, or fail entitlement checks
- inspect loading states, suspense boundaries, route guards, and error boundaries
- detect infinite loading loops
- detect repeated fetch loops
- detect hydration/render failures
- detect routes loading too much exam data at once
- detect pages importing heavy modules unnecessarily

B. EXAM API / BACKEND
- inspect all exam-related endpoints
- identify slow endpoints, timeout-prone endpoints, error-prone handlers, malformed responses, and memory-heavy operations
- identify any request handler that loads entire exam datasets when only a subset is needed
- identify whether rationale, metadata, translations, media, analytics, progress, and exam configuration are being loaded in a single oversized request
- inspect whether the API is doing synchronous heavy work during exam startup
- identify whether entitlement checks are expensive or brittle
- identify whether invalid payloads are crashing exam initialization

C. EXAM DATA ACCESS
- inspect all queries related to:
  - exam sessions
  - question delivery
  - question banks
  - CAT exams
  - user progress
  - exam results
  - subscriptions/entitlements
  - multilingual exam content
- identify N+1 query patterns
- identify missing indexes
- identify queries returning oversized result sets
- identify whether full question/rationale content is loaded when only active item data is needed
- identify whether exam startup is blocked by too many sequential queries

D. MEMORY / PERFORMANCE IMPACT ON EXAMS
- inspect whether exam startup or live exam sessions are causing memory spikes
- inspect whether question payloads are too large
- inspect whether too many questions are loaded into memory at once
- inspect whether locale data, media, rationale, and explanation content are bloating memory
- inspect whether admin or generator logic is leaking into exam runtime
- inspect whether exam serialization/deserialization is too heavy

E. ENTITLEMENT / ACCESS CONTROL
- confirm that tier logic still works correctly:
  - RPN users cannot access RN/NP-only exams unless entitled
  - RN users cannot access NP-only exams unless entitled
  - region/country gating remains correct
  - premium exam access remains correct
- identify whether entitlement bugs are blocking valid subscribers from exams
- identify whether broken entitlement checks cause false denials or blank states

F. PUBLISH / DEPLOY RISK
- inspect whether exam failures are worsened by slow or risky publish steps
- identify whether deploy includes migrations, repair scripts, or content jobs that create instability
- identify whether exam-related fixes can be shipped in a safer, more isolated way
