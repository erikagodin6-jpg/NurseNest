















Build a “MicroLecture Generator” feature for my NurseNest web app.

Goal:
For every topic or disease listed under the lessons sections (disease or symptom), generate:
1) a 10-30 minute narration script (tiered: RPN, RN, NP)
2) an 8-15slide lecture deck (PowerPoint .pptx)
3) a voice-over audio file (mp3) reading the script
4) a lecture player page where users can watch slides and listen to audio

Hard requirements:
- Script must be mechanism-driven (cellular for RN/NP), clinically accurate, and exam-relevant
- No shallow summaries; must explain WHY signs/symptoms happen and what treatments target at the cellular level (tier dependent level concepts)
- Slide content must match NurseNest aesthetic: premium, soft pastel palette, minimalist, clean typography, no cartoons
- Output must be generated as files and stored per-lecture: script.json, slides.pptx, audio.mp3
- Add metadata: title, slug, tier, duration estimate, keywords

Implementation requirements:
- Add an admin-only page: /admin/microlectures
- Admin inputs: topic name, tier (RPN/RN/NP), optional focus (e.g., ECG, shock), “Generate” button
- When clicked, backend calls an LLM to generate structured JSON: {title, script, slides:[{title, bullets, speakerNotes, visualPrompt}], flashcards, questions}
- Generate PowerPoint using PptxGenJS (or another reliable .pptx library) from the slides JSON
- For voice-over:
  Option 1 (preferred): integrate a text-to-speech API to generate mp3 from the narration script
  Option 2 (fallback): implement browser SpeechSynthesis for playback if no API key exists
- Save files to storage and record lecture in database with URLs
- Create lecture page: /microlecture/:slug that loads the PPTX slides as images or embeds them and plays MP3 with controls

Deliverables:
- All code changes required (backend + frontend)
- Database table schema for microlectures
- Security: only admins can generate; users can view based on subscription tier
- Include clear setup instructions for environment variables for TTS provider
- allow users to write notes and have the notes save to their user account 
- do not allow users to screenshot or print any of the information on any of these lectures
