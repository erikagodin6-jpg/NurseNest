Yes—do not store base64 PDFs in the DB. That’s the right correction. Now lock the preview system to a clean, low-risk pattern:

Store preview as a file/object (key/path), not base64

Serve preview via a streaming endpoint

Keep full PDFs private and only stream for admin/purchasers

Never “public URL” the full PDF

Below is the exact implementation pattern you should follow, including the two endpoints and storage scheme, plus the auth tightening for admin download.

1) Data model changes (minimal + correct)

Add these columns to digital_products (or your products table):

preview_storage_key TEXT NULL

preview_page_count INT NOT NULL DEFAULT 3

preview_updated_at TIMESTAMP NULL

If you already have storage_key for the full PDF, keep:

pdf_storage_key TEXT NOT NULL

Do not add previewUrl base64. Avoid it.

2) Storage scheme (object storage or local private dir)
Preferred: object storage keys

Use deterministic keys so regenerate overwrites or versions cleanly:

Full PDF: products/{productId}/full.pdf

Preview PDF: products/{productId}/preview-{pageCount}.pdf
(or include timestamp if you want history)

Example:

products/abc123/preview-3.pdf

Store that key in preview_storage_key.

3) Generate preview endpoint (admin-only)

Route

POST /api/admin/products/:id/generate-preview

Body

{ "pageCount": 3 } (cap to 1–10)

Implementation (pdf-lib)
This version assumes you have:

storage.getBytes(key) → Buffer|Uint8Array

storage.putBytes(key, bytes, mime) (or equivalent)

db.updateProduct(id, patch)

import { PDFDocument, StandardFonts, rgb, degrees } from "pdf-lib";

async function makePreviewPdf(fullPdfBytes: Uint8Array, pageCount: number) {
  const src = await PDFDocument.load(fullPdfBytes);
  const dst = await PDFDocument.create();
  const font = await dst.embedFont(StandardFonts.HelveticaBold);

  const n = Math.min(pageCount, src.getPageCount());
  const copied = await dst.copyPages(src, Array.from({ length: n }, (_, i) => i));

  const watermark = "NURSENEST PREVIEW • DO NOT DISTRIBUTE";

  for (const page of copied) {
    dst.addPage(page);
    const { width, height } = page.getSize();
    const size = Math.max(28, Math.min(56, Math.floor(width / 18)));

    // One strong watermark
    page.drawText(watermark, {
      x: width * 0.06,
      y: height * 0.42,
      size,
      font,
      color: rgb(0, 0, 0),
      rotate: degrees(-32),
      opacity: 0.14,
    });

    // Optional second lighter watermark
    page.drawText(watermark, {
      x: width * 0.06,
      y: height * 0.25,
      size,
      font,
      color: rgb(0, 0, 0),
      rotate: degrees(-32),
      opacity: 0.10,
    });
  }

  return await dst.save();
}

app.post("/api/admin/products/:id/generate-preview", requireAdmin, async (req, res) => {
  const productId = req.params.id;
  const pageCountRaw = Number(req.body?.pageCount ?? 3);
  const pageCount = Math.max(1, Math.min(10, Number.isFinite(pageCountRaw) ? pageCountRaw : 3));

  const product = await db.getDigitalProductById(productId);
  if (!product) return res.status(404).json({ error: "Not found" });
  if (!product.pdf_storage_key) return res.status(400).json({ error: "Missing full PDF storage key" });

  const fullBytes = await storage.getBytes(product.pdf_storage_key);
  const previewBytes = await makePreviewPdf(fullBytes, pageCount);

  const previewKey = `products/${productId}/preview-${pageCount}.pdf`;
  await storage.putBytes(previewKey, previewBytes, "application/pdf");

  await db.updateDigitalProduct(productId, {
    preview_storage_key: previewKey,
    preview_page_count: pageCount,
    preview_updated_at: new Date(),
  });

  res.json({ ok: true, previewKey });
});
Key points

capped pages

watermark every page

writes to storage

stores preview key in DB

4) Public preview streaming endpoint (no auth, preview only)

Route

GET /api/products/:slug/preview

app.get("/api/products/:slug/preview", async (req, res) => {
  const slug = req.params.slug;
  const product = await db.getDigitalProductBySlug(slug);

  if (!product || product.is_deleted) return res.status(404).end();
  if (!product.preview_storage_key) return res.status(404).end();

  const bytes = await storage.getBytes(product.preview_storage_key);

  res.setHeader("Content-Type", "application/pdf");
  res.setHeader("Content-Disposition", "inline");
  res.setHeader("Cache-Control", "public, max-age=3600");
  res.send(Buffer.from(bytes));
});

Do not allow this to fall back to the full PDF. If no preview exists, return 404.

5) Admin download auth: fix GET auth correctly

You said requireAdmin supports query params. That’s not the real requirement.

The real requirement: requireAdmin must validate admin status based on:

session cookie (preferred), or

bearer token header

GET requests often don’t include a JSON body, so do not depend on body fields.

Correct admin download route

GET /api/admin/products/:id/download

app.get("/api/admin/products/:id/download", requireAdmin, async (req, res) => {
  const id = req.params.id;
  const product = await db.getDigitalProductById(id);
  if (!product) return res.status(404).end();
  if (!product.pdf_storage_key) return res.status(400).json({ error: "Missing PDF" });

  const bytes = await storage.getBytes(product.pdf_storage_key);

  const safeName = (product.title || `product-${id}`)
    .replace(/[^\w\-]+/g, "_")
    .slice(0, 80);

  res.setHeader("Content-Type", "application/pdf");
  res.setHeader("Content-Disposition", `attachment; filename="${safeName}.pdf"`);
  res.send(Buffer.from(bytes));
});

Object storage is already set up: use it.

If you instead store previews under a “private directory” on disk:

Object storage + streaming endpoint is the cleanest.
