PART 2 — SERVER-SIDE AI GENERATION SERVICE
Create: services/aiItemWriter.ts (or similar)

A) Two-stage generation for higher quality (required)
Stage 1: Create a blueprint per item (structured JSON)
- For exam questions:
  - learningObjective
  - patient vignette key facts
  - what is being tested (concept)
  - distractor logic
  - correct answer reasoning
  - tier/exam alignment notes
Stage 2: Expand into final exam-ready item
- stem, choices, correct answer, rationales, clinical pearl, tags, difficulty

B) Use strict JSON schema validation (required)
Use zod (or existing validator) to enforce output structure.
If invalid JSON:
- One repair attempt only
- If still invalid, fail batch item and log.

C) Add “Quality Rubric” scoring (required)
Before saving, score each question against rubric (0–100):
- Clarity of stem
- Single best answer
- Distractors plausible and tier-appropriate
- Rationale quality
- Safety/clinical accuracy flags
- No ambiguity
Reject or mark “Needs Revision” automatically if score < 85.
Store score and rubric breakdown in DB.

D) Duplicates prevention
- Generate a stable hash for each item (tier + stem normalized + correctAnswer + key concepts).
- If duplicate detected vs last 90 days, discard and regenerate replacement.

E) Topic distribution + exam realism
- Ensure the batch includes realistic exam mix:
  RPN/LVN: safety, infection control, basic pharm, stable clients, data collection
  RN: prioritization, unstable clients, labs, delegation, acute deterioration, med-surg breadth
  NP: differential diagnosis, diagnostics, prescribing, red flags, referral thresholds, guideline-based management
- SATA frequency realistic (e.g., 15–25% RN, 10–20% PN, 10–15% NP depending on blueprint)
- Include “who do you see first” and “first action” items.

F) Do NOT use real patient identifiers or PHI.
