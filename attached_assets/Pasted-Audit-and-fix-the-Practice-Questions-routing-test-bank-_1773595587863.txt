Audit and fix the Practice Questions routing, test bank exposure, and SEO structure for specialty guides such as ICU Nursing.

This is a high priority issue involving both conversion optimization and paywall protection.

CURRENT PROBLEM

When users click "Practice Questions" from a specialty guide such as:

/guides/icu-nursing

they are routed directly to:

/test-bank

The test bank page exposes internal functionality including:

tier selectors (RPN / RN / NP / Allied)
system filters
exam filters
question navigation
question counts
test modes

This is incorrect behavior because the test bank is the internal learning environment and should only be visible to authenticated users with the correct subscription.

Allowing guide pages to route directly to the test bank creates the following risks:

users probing locked content
users switching tiers
confusion between marketing pages and learning dashboards
potential paywall bypass vectors
poor UX funnel design

OBJECTIVE

Implement a proper preview funnel for specialty guides while protecting the test bank behind authentication.

The system should follow this structure.

GUIDE PAGE
example:

/guides/icu-nursing

↓

PREVIEW PAGE
example:

/preview/icu

↓

PAYWALL

↓

FULL TEST BANK
/test-bank

STEP 4

Restrict access to the test bank.

redirect to login

redirect to the pricing page

STEP 5

STEP 6

Tier switching (RPN, RN, NP, Allied) should only exist inside the authenticated dashboard.
