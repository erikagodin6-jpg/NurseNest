You are working on the live NurseNest production codebase.

Your task is to perform a full SEO and technical indexing cleanup specifically focused on fixing the Google Search Console issue:

“Alternative page with proper canonical tag”

There are currently hundreds of affected URLs across many language folders, including examples like:

/en/learn/...
/en/lessons/...
/en/blog
/en/anatomy
/en/lab-values
/es/index.html
/fa
/zh/
/fr/privacy
/pt/blog
/ja/disclaimer
and many localized copies with timestamped slugs such as:
-1772056430904
-1772067874506
-1772068216542

GOAL

Reduce duplicate URL generation, tighten canonical logic, improve hreflang correctness, prevent low-value duplicate pages from being indexed, and ensure only the correct version of each page is indexable.

IMPORTANT

Do not just patch one page.
Do not hardcode isolated fixes.
Do not create SEO regressions.
Do not break the multilingual routing system.
Do not remove existing high-value content.
Implement this as a systematic production-grade cleanup across the whole site.

==================================================
PHASE 1 — AUDIT THE CURRENT INDEXING ARCHITECTURE
==================================================

Audit and document how the site currently handles:

1. Canonical tags
2. hreflang tags
3. language routing
4. trailing slash behavior
5. index.html variants
6. duplicate slug generation
7. translated page generation
8. sitemap generation
9. robots directives
10. meta robots on low-value pages
11. pagination or hub-page indexing
12. internal links pointing to duplicate URLs

Find the root causes of why Google is classifying so many pages as:
“Alternative page with proper canonical tag”

==================================================
PHASE 2 — FIX CANONICAL LOGIC GLOBALLY
==================================================

Implement a strict canonical URL policy.

Requirements:

1. Every indexable page must output exactly one self-referencing canonical tag if that page is intended to be indexed.
2. Pages that should not be indexed must not pretend to be primary pages.
3. Canonicals must always resolve to the clean final URL:
   - no duplicate parameters
   - no timestamp slug variants unless that exact page is the primary permanent slug
   - no /index.html canonicals
   - no mixed trailing slash inconsistencies
   - no canonicals pointing to redirected URLs
4. Canonicals must be absolute URLs using https://www.nursenest.ca
5. Canonicals must be generated centrally from one canonical URL utility, not scattered manually across components.

Create or refactor a shared SEO URL helper so every route uses the same canonical normalization rules.

==================================================
PHASE 3 — REMOVE OR CONSOLIDATE DUPLICATE SLUGS
==================================================

A major problem appears to be duplicate article/page URLs with timestamp suffixes, for example:

/learn/high-anion-gap-metabolic-acidosis-explained
/learn/high-anion-gap-metabolic-acidosis-explained-1772067874506

Fix this system-wide.

Requirements:

1. Detect where duplicate timestamped slugs are being generated.
2. Prevent new timestamp-appended slugs from being created unless absolutely required for uniqueness in the CMS backend.
3. If uniqueness is needed internally, keep the unique ID in the database, not in the public SEO slug whenever possible.
4. Choose one permanent public slug for each content item.
5. Redirect all duplicate slug variants to the chosen primary slug with 301 redirects.
6. Ensure sitemap output only includes the primary slug.
7. Ensure internal links always point to the primary slug.
8. Ensure canonicals always point to the primary slug.

If two pages are materially the same content, consolidate them into one canonical public URL.

==================================================
PHASE 4 — CLEAN UP MULTILINGUAL INDEXING
==================================================

The site has many language folders such as:
en, es, fr, pt, ar, hi, ur, zh, ja, ko, vi, fil, ht, fa, pa

You must clean up multilingual SEO architecture.

Requirements:

1. Every truly localized page must have correct hreflang tags referencing all equivalent language versions, plus x-default where appropriate.
2. hreflang clusters must only include genuine equivalents of the same page.
3. Do not link unrelated pages together via hreflang.
4. If a translated page is weak, unfinished, machine-generated, duplicate, or not strategically important, consider marking it noindex until quality is improved.
5. Utility pages duplicated across all languages with very little search value should be evaluated carefully:
   - privacy
   - terms
   - disclaimer
   - refund-policy
   - faq
   - start-free
   - generic hub pages
6. For low-value or thin translated utility pages, implement a deliberate policy:
   either keep one canonical version indexed
   or set the low-value localized versions to noindex, follow.
7. Remove accidental duplicate home variants such as:
   /es/index.html
   and normalize them to the clean route:
   /es/
8. Enforce one clean URL structure per locale.

Do not let thin translated copies flood the sitemap or internal search index.

==================================================
PHASE 5 — DEFINE WHAT SHOULD AND SHOULD NOT BE INDEXED
==================================================

Create a clear indexing policy by page type.

Review all main page groups, including but not limited to:

1. lesson pages
2. learn articles
3. blog posts
4. study guides
5. anatomy
6. lab-values
7. simulators
8. mock exams
9. case simulations
10. privacy/terms/disclaimer/refund pages
11. hub/category pages
12. empty or incomplete lesson pages
13. paywalled or near-duplicate lesson teaser pages

For each type, decide whether it should be:

- index, follow
- noindex, follow
- canonicalized elsewhere
- redirected to a better parent or primary page

Rules:
1. Empty pages must never be indexable.
2. Placeholder pages must be noindex or removed.
3. Very thin duplicate translated pages should not all be indexed.
4. Pages with little standalone search value should not dilute crawl budget.
5. Pages intended to rank should have unique metadata, self-canonical, clean internal linking, and inclusion in sitemap.

==================================================
PHASE 6 — FIX SITEMAPS
==================================================

Audit the sitemap generator and rebuild it properly.

Requirements:

1. Only include canonical, indexable, live 200-status URLs.
2. Exclude:
   - redirected URLs
   - noindex URLs
   - duplicate timestamp-slug URLs
   - /index.html variants
   - thin placeholder pages
   - broken or empty localized pages
3. Ensure hreflang-aware alternates are correct if sitemap alternates are used.
4. Separate sitemap sections logically if needed:
   - pages
   - lessons
   - learn
   - blog
   - study guides
   - locales
5. Ensure canonical URL generation and sitemap URL generation use the same normalization utility.

==================================================
PHASE 7 — FIX INTERNAL LINKING
==================================================

Audit internal links across the site.

Requirements:

1. Remove links to duplicate slug variants.
2. Remove links to /index.html paths.
3. Ensure nav, cards, breadcrumbs, related posts, lesson lists, XML generators, and localized switchers all point to the canonical destination.
4. Ensure language switchers map equivalent pages only.
5. Prevent inconsistent linking between:
   /en/page
   /en/page/
   /en/index.html
   duplicate timestamp versions
6. Ensure all internal links reinforce the canonical version Google should index.

==================================================
PHASE 8 — HANDLE LOW-VALUE TRANSLATED PAGES STRATEGICALLY
==================================================

Do not blindly keep every translated page indexable.

Create a strategic rule set such as:

1. Keep English fully indexable as the primary content base.
2. Keep only high-quality, fully rendered, strategically valuable translated pages indexable.
3. Set weak or low-value translated utility/hub pages to noindex, follow.
4. If a page exists in many languages but has almost no unique search intent or content differentiation, do not waste crawl budget on all versions.
5. Preserve user access to translations, but do not force Google to index everything.

This must be implemented carefully so the site still supports multilingual UX while improving SEO quality.

==================================================
PHASE 9 — TECHNICAL NORMALIZATION
==================================================

Implement route normalization rules.

Requirements:

1. 301 redirect /xx/index.html to /xx/ or /xx
2. Normalize trailing slash policy consistently
3. Force lowercase paths if the site allows accidental case variation
4. Strip duplicate slashes
5. Remove malformed URLs from sitemap generation
6. Ensure canonical generation matches the final routed URL exactly
7. Ensure non-existent old variants return proper 404 or 301, not soft duplicate responses
