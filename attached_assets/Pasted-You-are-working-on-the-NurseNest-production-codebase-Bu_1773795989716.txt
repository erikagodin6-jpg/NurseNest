You are working on the NurseNest production codebase.

Build a COMPLETE PLATFORM RESILIENCE, FAILOVER, AND DISASTER RECOVERY SYSTEM that ensures:

A paying or entitled user NEVER loses access to usable study functionality, even during:
- frontend crashes

==================================================
PRIMARY OBJECTIVE
==================================================

Guarantee CONTINUITY OF ACCESS.

If the primary system fails, the platform must:
1. Automatically detect the failure
2. Degrade gracefully
3. Route to a safe backup experience
4. Preserve user progress where possible
5. Maintain user trust with clear messaging
6. Alert admin systems immediately
7. Recover or rollback automatically where applicable

SECTION 1 — GLOBAL EMERGENCY MODE (READ-ONLY SAFE MODE)
==================================================

Implement a platform-wide “Emergency Mode” that activates automatically when critical systems fail.

Triggers:
- exam load failure rate spike
- subscription/entitlement failures
- frontend crash spike
- API failure rate threshold
- DB query failures
- deployment instability

Behavior:
- switch platform into READ-ONLY SAFE MODE
- disable unstable features (CAT engine, heavy UI, complex interactions)
- serve ONLY:
  - static lessons
  - static exam backups
  - flashcard list mode
  - downloadable resources
  - last-known-good snapshots
- prevent write operations that could corrupt data
- preserve login + entitlement checks
- show system banner:
  “We’re running in backup mode. Your access is protected.”

Admin controls:
- manual toggle ON/OFF
- auto-trigger with threshold
- audit log for activation

Implement safe deployment strategy.
