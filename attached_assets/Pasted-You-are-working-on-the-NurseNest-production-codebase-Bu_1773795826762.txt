You are working on the NurseNest production codebase.

Build a full subscription resilience, entitlement fail-safe, and backup access architecture across the entire platform so paying users and valid free users never lose access to the core value they purchased or unlocked, even if a page, feature, API, database query, content payload, integration, or frontend component fails.

This is a platform-wide reliability system, not a page-specific patch.

PRIMARY GOAL

A user with valid access must always have a working fallback path to the product, content, or feature they are entitled to use.

This applies to all subscription activities and all gated or premium experiences across:
- nursing

- international nursing
- blog/lesson/study resources where gated
- practice questions

- downloads

NON-NEGOTIABLE PLATFORM RULE

Never let a valid entitlement result in a dead end.

If the primary experience fails, the user must automatically receive:
1. a safe fallback experience,
2. a backup content/version path,
3. clear messaging that access is protected,
4. logging + alerting for admin recovery.

SECTION 1 — PLATFORM-WIDE RELIABILITY ARCHITECTURE
==================================================

Build a centralized subscription resilience system with these core modules:

1. Entitlement Resolver
- single source of truth for access checks
- determines whether the user should have access
- handles subscriptions, one-time purchases, bundles, trial access, promotional access, admin overrides, legacy purchasers, free tier access, and grandfathered products
- must return a normalized access decision object, not ad hoc booleans

2. Access Delivery Orchestrator
- sits between entitlement checks and product rendering
- chooses the best available delivery mode for each feature:
  - primary experience
  - safe fallback mode
  - backup snapshot/version
  - substitute equivalent content
  - downloadable/static backup
- routes user automatically when primary path fails

3. Published Snapshot Service
- stores immutable last-known-good snapshots for premium content/products
- created at publish time and updated only when a new version passes validation
- allows serving verified backup copies instead of broken live payloads

4. Runtime Resilience Layer
- wraps all gated experiences with defensive validation, fallback routing, logging, and recovery UX
- no premium route should directly hard-crash without fallback handling

5. Incident Logging + Recovery Layer
- centralized structured logging
- admin dashboard for failures
- one-click recovery actions
- threshold alerts for production incidents

SECTION 6 — SUBSTITUTE CONTENT ENGINE
==================================================

If the exact premium experience is unavailable and backup rendering cannot safely proceed, automatically offer an equivalent substitute experience.

Build substitute matching rules using:
- same profession
- same tier
- same exam type
- same domain/category
- same country/region
- same language if possible
- same subscription plan eligibility

Examples:
- broken RN cardiovascular exam -> substitute RN cardiovascular backup practice set
- broken NP pharmacology CAT -> substitute NP pharmacology weighted practice session
- broken allied respiratory flashcard deck -> substitute validated respiratory therapy review deck
- broken premium lesson -> substitute equivalent published premium study guide

Rules:
- never substitute unrelated material unless clearly labeled
- prefer same-language substitute
- if exact language unavailable, use default language only if allowed and clearly disclose fallback
- log every substitution event

UI message:
“This item is temporarily unavailable. Your access is protected. We’ve opened the closest available backup resource for you.”
