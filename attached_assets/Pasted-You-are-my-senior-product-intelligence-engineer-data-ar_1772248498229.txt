You are my senior product intelligence engineer + data architect working inside NurseNest (Vite + React + Express + DB + multilingual SEO system).

Build a Data-Driven Content Gap + Blueprint Coverage Engine.

Objective:
Continuously identify weaknesses in:
- Blueprint category coverage
- Topic cluster depth
- Question bank calibration distribution
- Difficulty balance
- Language coverage
- SEO keyword targeting
- Conversion funnel coverage

The system must generate actionable insights, not just data.

====================================================
PART 1 — BLUEPRINT COVERAGE ANALYSIS
====================================================

For each exam (NCLEX-RN, NCLEX-PN, REx-PN, CNPLE, AANP, ANCC):

1) Store official blueprint categories + weight ranges.
2) For each category:
   - Count total questions.
   - Count calibrated questions by difficulty level.
   - Count case-based items.
   - Count NGN-style items.
   - Calculate percent of bank dedicated to that category.
   - Compare against blueprint weight target.

Output per category:
- status = underrepresented | balanced | overrepresented
- target_range
- current_percent
- recommended_questions_to_add

Display in admin as heatmap.

====================================================
PART 2 — DIFFICULTY DISTRIBUTION ENGINE
====================================================

For each exam:

Compute distribution of calibrated difficulty levels (1–5).

Target distribution example (RN):
- Level 1: 10%
- Level 2: 20%
- Level 3: 30%
- Level 4: 25%
- Level 5: 15%

If deviation >5%:
- flag imbalance.
- suggest number of items to add per level.

Also track discrimination index:
- flag items with low discrimination (<0.15).
- suggest revision or retirement.

====================================================
PART 3 — LANGUAGE COVERAGE MATRIX
====================================================

For each language and content type:

Compute:
- total pillar pages
- total cluster pages
- total glossary terms
- total lessons
- total questions translated
- translation completeness %
- average word count per pillar
- average word count per cluster

Display as 15×content matrix.

Flag:
- languages with <70% coverage in Tier 1 languages.
- thin translated pages (<1200 cluster, <2500 pillar).

====================================================
PART 4 — SEO KEYWORD GAP ENGINE
====================================================

For each language:

1) Pull keyword targets from seo_keyword_targets table.
2) Check if mapped_page_slug exists.
3) Check if mapped page meets wordcount + FAQ threshold.
4) Flag keywords with no optimized page.

Output:
- High-priority keyword gaps
- Suggested page type (pillar|cluster|comparison|blog)

====================================================
PART 5 — INTERNAL LINK DEPTH CHECK
====================================================

For each page:
- Count internal links.
- Count inbound links.
- Flag orphan pages.
- Flag pages with <5 internal links.
- Ensure each cluster links back to pillar.

====================================================
PART 6 — CONVERSION GAP ANALYSIS
====================================================

For each language:

Track:
- traffic
- diagnostic starts
- diagnostic completions
- study plan generation
- upgrade conversions
- revenue

Compute:
conversion_rate_by_language
revenue_per_visitor_by_language

Flag:
- high traffic + low conversion languages
- high diagnostic starts + low upgrade rate

Recommend:
- strengthen localized upgrade copy
- add localized trust signals
- improve diagnostic CTA placement

====================================================

====================================================

Every week generate:

1) Blueprint imbalance summary.
2) Difficulty imbalance summary.
3) Top 10 SEO keyword gaps.
4) Languages below coverage threshold.
5) Orphan pages list.
6) Conversion anomalies.

Store in:
content_intelligence_reports table.

Send admin notification summary.
