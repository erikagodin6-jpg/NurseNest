I need to redesign the NurseNest platform so the user experience is fully tier-native across the entire product.

Current problem:
Even if content is filtered by role, the platform still feels too generic. I do not want RPN/LVN, RN, and NP users to feel like they are all using the same interface with a few hidden sections. I want each tier to feel like its own dedicated exam-prep environment with its own labels, dashboard language, progression wording, exam naming, visual cues, study recommendations, and onboarding flow.

GOAL:
Create a fully role-native, tier-specific UX system for:
- FREE users
- RPN/LVN users
- RN users
- NP users
- Admin users

The experience should feel polished, intentional, premium, and psychologically matched to the learner’s scope, exam level, and career stage.

==================================================
PART 1 — CORE PRINCIPLE
==================================================

Implement a role-native UX architecture where the platform adapts dynamically based on the logged-in user’s role.

Canonical roles:
- free
- rpn
- rn
- np
- admin

For every logged-in user, the system must determine:
- role
- primaryExamTier
- allowedContentTiers
- dashboardVariant
- navigationVariant
- studyLanguageVariant
- brandingVariant
- examNamingVariant
- recommendationVariant

The UI should not merely hide unauthorized sections.
It should actively transform the experience so it feels custom built for that tier.

The platform should feel tailored, but still clearly part of the NurseNest brand.

4. Premium coherence
Paid users should feel like they entered a serious exam-prep environment designed for their level.

5. Polished microcopy
Avoid clunky generic labels.
Everything should sound intentional and premium.
