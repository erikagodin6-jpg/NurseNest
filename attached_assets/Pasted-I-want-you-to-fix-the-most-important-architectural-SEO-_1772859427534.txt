I want you to fix the most important architectural SEO mistake in my platform: too many application URLs, low-value pages, and fragmented dynamic routes are currently exposed to Google as if they are rankable content pages.

My site is a large educational platform with lessons, study guides, practice questions, exam pages, dashboards, filters, and dynamic app routes. Google Search Console shows a major impression drop and a very large number of discovered but not indexed URLs. I want a full architectural refactor so search engines see a clean, focused content library instead of a sprawling web app.

Your goal is to separate the site into two distinct layers:

1. A rankable SEO content layer
2. A non-indexable application/product layer

The SEO content layer should include only the strongest pages such as:
- lesson pages
- category/topic hubs
- high-value study guides and blog pages
- curated topic-based practice question pages
- major exam prep landing pages
- institutional or licensing pages

The application/product layer should include pages such as:
- individual question pages
- exam session pages
- CAT engine pages
- dashboards
- account pages
- admin pages
- preview pages
- filter/search result pages
- parameterized URLs
- paginated utility pages
- duplicate language or route variants
- temporary or low-value archive pages

Please perform the following across the codebase:

1. Audit every public URL type currently generated by the app.
Identify all page classes and route types, including dynamic routes, utility pages, search/filter states, preview pages, language variants, pagination, and content pages.

2. Classify every page type into one of these buckets:
- should be indexable
- should be noindex
- should be excluded from sitemap
- should redirect to a canonical URL
- should be blocked from crawling if appropriate

3. Implement a clean SEO architecture where only the content-library layer is indexable.
The website should appear to Google as a tightly organized educational resource, not as a fully crawlable application.

4. Apply proper canonical logic.
All true content pages that remain indexable must self-canonicalize correctly.
All duplicate route variants, alternate route structures, parameterized duplicates, and conflicting language versions must be consolidated.

5. Rebuild the sitemap so it includes only indexable canonical content pages.
Remove all low-value, duplicate, noindex, utility, or app-state URLs from the sitemap.
Include lastmod fields and split into sitemap index files if needed.

6. Implement robots and meta robots rules appropriately.
Use noindex for non-rankable application pages.
Prevent unnecessary crawl paths where they waste crawl budget.
Do not allow utility URLs to compete with educational pages.

7. Improve internal linking only among the pages that should rank.
Category hubs should link to child lessons and major topic clusters.
Lessons should link to related lessons, related study guides, and topic-based question pages.
Study guides should link back to lessons and hubs.
Exam landing pages should connect logically to supporting educational content.

8. Convert fragmented thin SEO candidates into stronger parent pages where appropriate.
For example, instead of indexing thousands of individual question pages, create and prioritize topic-based practice question pages with substantial explanatory content.

9. Audit templates for repetition and improve uniqueness for all indexable pages.
Ensure each indexable page has:
- a unique title tag
- unique meta description
- strong H1
- useful introduction
- clear heading hierarchy
- contextual internal links
- topic-specific content blocks
- meaningful standalone value

10. Ensure every indexable page loads quickly and is technically clean.
Optimize code splitting, image loading, JS execution, and above-the-fold rendering on pages that remain indexable so Google can crawl them efficiently.

11. Keep the product/app experience intact for users, but make sure it does not pollute the indexable SEO layer.

12. At the end, provide a detailed report showing:
- all URL/page types found
- which were kept indexable
- which were changed to noindex
- which were removed from the sitemap
- which were consolidated by canonical or redirect
- which route classes were causing indexation dilution
- how the new architecture separates rankable content from app mechanics

The final result should be that Google sees a high-quality, tightly curated educational content site with a clear topical hierarchy, while the application layer remains functional for users but does not dilute crawl budget or indexation quality.
