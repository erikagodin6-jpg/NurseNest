MASTER PRODUCTION FIX — NURSENEST NAVIGATION, ARCHITECTURE, ROUTING, LINKING, AND LESSON DISPLAY

Problem:
The published NurseNest experience is structurally inconsistent. Multiple sections behave like disconnected products, some routes redirect into different shells/subdomains, many links are broken or orphaned, and lesson content is being exposed in the wrong public structure.

This must be treated as one architecture incident, not a series of isolated bugs.

==================================================
SECTION 1 — CORE PRODUCT RULES
==================================================

1. NurseNest must feel like ONE ecosystem.
2. Allied Health must feel like a section of NurseNest, not a trap site.
3. New Grad / Nursing Certifications must be a separate section from Allied Health.
4. Public lesson pages must show curated, production-ready content only.
5. No link should render unless its destination exists and works on published.
6. The database is the source of content, NOT the source of public structure.

==================================================
SECTION 2 — AUDIT THE LIVE PUBLISHED SITE
==================================================

Audit the actual published site, not preview.

Document all of the following:
- main domain routes
- allied subdomain routes
- redirects between them
- pages using main NurseNest shell
- pages using disconnected/legacy shells
- header links
- footer links
- dropdown links
- hero CTA links
- lesson card links
- specialty/certification links
- locale behavior

For every link, record:
- visible label
- current href
- final destination
- whether it resolves
- whether it 404s
- whether it redirects to wrong shell/subdomain
- whether destination exists on published
- whether destination belongs in the correct section

- route Allied specialties to generic hero pages unless intentional
- mix multiple layout systems for these sections

==================================================
SECTION 9 — REPAIR LESSON PAGE TAXONOMY
==================================================

The current lessons implementation exposed the wrong public structure.

Fix the lesson pipeline so:
1. lessons are fetched from the database
2. lesson records are validated
3. only approved/public/production-ready lessons pass
4. lessons are mapped into approved systems/categories
5. only curated public systems render

Do NOT:
- render raw database dumps publicly
- create a new public “Database Lessons” block
- expose unfinished, internal, malformed, duplicate, or review-needed lessons

Hide from public display any lesson that is:
- draft
- incomplete
- low-quality
- duplicate
- unmapped
- malformed
- internal/test/seed content
- badly titled
- missing required metadata

If a lesson cannot be confidently mapped to an approved system, send it to admin review only.
