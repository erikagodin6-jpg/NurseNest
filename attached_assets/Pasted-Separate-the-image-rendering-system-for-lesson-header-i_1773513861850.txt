Separate the image rendering system for lesson header images versus lesson cards across the entire NurseNest platform.

PROBLEM
The platform is treating lesson header images and small lesson card thumbnails too similarly. This is causing educational images to be cropped badly, faded at the bottom, or displayed in a way that hides clinically important visual content. Lesson headers need one image strategy, and lesson cards need another.

GOAL
Create two clearly different image-rendering behaviors:
1. Lesson header / hero images
2. Lesson card / grid / list thumbnail images

Both should look polished and premium, but they must be optimized for different use cases.

APPLY TO
- RPN
- RN
- NP
- Allied Health
- Imaging
- New Grad
- all lesson-based pages and components

PART 1: LESSON HEADER / HERO IMAGE RULES

Lesson header images are large educational visuals shown on the lesson page itself.

These must:
- prioritize visibility of the actual educational image
- avoid excessive cropping
- avoid fading out the bottom of the image
- avoid decorative overlays that obscure the image
- preserve important anatomy/medical details
- feel premium and modern
- work well on desktop and mobile

HEADER IMAGE IMPLEMENTATION RULES
- do not use the same shallow thumbnail container used for cards
- avoid fixed short heights that cut off most of the image
- remove unnecessary bottom gradient fades, masks, opacity layers, or pseudo-element overlays
- if text overlays the image, keep the overlay subtle and make sure most of the image remains visible
- allow taller responsive containers where needed
- if object-cover is used, ensure focal positioning preserves the subject
- for some educational images, use object-contain with a tasteful background treatment if that shows the content better
- do not let overflow-hidden clip critical parts of medical diagrams or illustrations
- ensure the full or near-full composition is visible

HEADER IMAGE VISUAL GOAL
The lesson header should feel like a premium educational banner:
- clinically clear
- visually readable
- not awkwardly zoomed
- not faded out
- not cut in half
- not treated like a decorative stock image

PART 2: LESSON CARD / THUMBNAIL IMAGE RULES

Lesson cards are small previews used in:
- lesson grids
- search results
- carousels
- dashboards
- category pages
- recommendation sections

These should:
- be more compact
- remain visually consistent
- tolerate more cropping than header images
- preserve quick-recognition value
- still avoid ugly or misleading crops

CARD IMAGE IMPLEMENTATION RULES
- use a stable thumbnail aspect ratio
- allow object-cover if needed for consistency
- use smart focal positioning to keep the main subject visible
- no heavy fade overlays
- no dramatic masks
- no weird zooming
- no clipping that removes the key visual meaning
- maintain a polished card look

CARD IMAGE VISUAL GOAL
Small cards should look:
- clean
- consistent
- modern
- attractive in grids
- easy to scan quickly

But they do NOT need the same near-full-image treatment as lesson headers.

PART 3: CREATE TWO DISTINCT COMPONENT MODES

Refactor the image system so it explicitly supports separate modes, for example:
- variant="header"
- variant="card"

or separate components such as:
- LessonHeaderImage
- LessonCardImage

Do not rely on one shared generic image component with accidental styling inheritance if that is what is causing these problems.

PART 4: AUDIT THE CODEBASE

Search the entire project for:
- lesson image components
- hero image components
- card thumbnail components
- shared image wrappers
- overflow-hidden
- object-cover
- object-contain
- mask-image
- gradient overlays
- pseudo-elements
- fixed height image containers
- mobile-specific image styles

Find every place where lesson images are rendered and map them into either:
- header image behavior
- card image behavior

PART 5: REMOVE BAD DECORATIVE EFFECTS FROM HEADER IMAGES

For lesson header images specifically, remove or reduce:
- bottom fade overlays
- dark gradients that hide the lower half
- masked blending
- excessive text overlay layers
- decorative clipping that sacrifices image readability

If contrast is needed for title text, keep the treatment subtle and localized.

PART 6: RESPONSIVE BEHAVIOR

HEADER IMAGES
- on mobile, still show the image clearly
- do not crop the medically important focal point out of view
- ensure title and image spacing remains elegant
- avoid giant blank padding if object-contain is used

CARD IMAGES
- maintain uniform card grid appearance
- ensure images still look intentional at smaller sizes
- preserve subject visibility as much as possible

PART 7: PRESERVE NURSENEST BRANDING

Maintain NurseNest’s visual style:
- soft modern premium look
- polished rounded containers if already used
- clean and uncluttered layouts
- no amateurish icon-heavy feel
- no heavy dark overlays that compete with the pastel/clinical aesthetic

PART 8: DO NOT BREAK CONTENT OR SEO

Do not change:
- lesson slugs
- routing
- content structure
- analytics
- search indexing
- paywall logic
- admin controls

This is an image rendering refactor, not a content migration.

PART 9: VALIDATION CHECKLIST

Before finishing, verify:
1. lesson header images are clearly visible and no longer fade at the bottom
2. lesson header images are not cropped like thumbnails
3. lesson cards still look consistent in grids
4. mobile rendering is polished
5. card thumbnails do not inherit hero styles
6. hero images do not inherit card thumbnail cropping behavior
7. no lesson pages or card lists were broken

PART 10: RETURN A CLEAR SUMMARY

At the end, report:
- which components were separated or refactored
- which CSS classes/styles were changed
- what was causing the header image fade/cropping issue
- how header mode now differs from card mode
- any remaining edge cases needing manual tuning

IMPORTANT
Do not apply a one-off patch. Create a durable separation between lesson-header image rendering and lesson-card image rendering so future edits do not keep reintroducing the same problem.
