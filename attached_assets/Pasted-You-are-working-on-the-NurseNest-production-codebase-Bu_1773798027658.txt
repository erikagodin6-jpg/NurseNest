==================================================

- extend subscription by configurable duration

SECTION 5 — RELEASE CHECKLIST GATE
==================================================

Build a release gate that blocks risky deploys or publishes unless required checks pass.

Before deploy:
- critical path tests pass
- login works
- entitlement check works
- exam open works
- CAT start works
- flashcards load
- premium downloads work
- rollback target exists
- monitoring is healthy
- backup services are healthy

Before content publish:
- schema validation passes
- content health score above threshold
- backup artifacts generated
- fallback path available
- last-known-good version preserved
- translations valid where required

If checks fail:
- block deploy or publish
- show exact failing checks
- require explicit override only for authorized admin users
- log override action

Goal:
Do not allow risky changes into production casually.
