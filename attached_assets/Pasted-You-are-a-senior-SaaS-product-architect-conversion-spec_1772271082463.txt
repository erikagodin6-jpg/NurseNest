You are a senior SaaS product architect, conversion specialist, and full-stack engineer.

Goal:
Transform the flashcard system into a high-converting freemium SaaS model that:

- Allows 300 free flashcards per user
- Requires upgrade after 300 cards
- Is cheaper than Quizlet+
- Maximizes upgrade conversion
- Increases retention
- Is technically secure and scalable

--------------------------------------------------
PART 1 — PLAN STRUCTURE
--------------------------------------------------

Implement two plans:

FREE PLAN
- 300 total flashcards across all decks
- Can edit/view/delete existing cards
- Cannot create card #301
- Show progress counter (e.g. 247 / 300)
- Show soft upgrade prompts at:
    240 cards (80%)
    275 cards (90%)
- Hard stop modal at 300

PRO PLAN
- Unlimited flashcards
- AI flashcard generation
- Spaced repetition system
- Exam-mode testing
- Export to PDF
- Priority content indexing
- No ads (if ads ever exist)

--------------------------------------------------
PART 2 — PRICING STRATEGY
--------------------------------------------------

Pricing must be cheaper than Quizlet+.

Implement:

Option A:
$4.99/month

Option B:
$39/year (recommended, marked “Best Value”)

Add psychological pricing:

- Show crossed-out comparison:
   “Quizlet+ $7.99/month”
- Show:
   “Nearly 50% cheaper”

Add pricing comparison table.

--------------------------------------------------
PART 3 — DATABASE + ENFORCEMENT
--------------------------------------------------

Add to users table:
- plan_type (free|pro)
- flashcard_limit (default 300)
- stripe_customer_id
- stripe_subscription_id
- subscription_status
- plan_expires_at

Server-side enforcement only:
- On flashcard creation:
   Count total user cards.
   If free AND >= 300:
       return 403 with upgrade message.
- Do not rely on client.

Deletion must recalculate count dynamically.

--------------------------------------------------
PART 4 — STRIPE INTEGRATION
--------------------------------------------------

Implement:
- Stripe Checkout
- Webhook listener
- Subscription status sync
- Cancel at period end
- Failed payment handling

Upgrade flow:
- User clicks upgrade
- Redirect to Stripe Checkout
- On success:
   plan_type = pro
   flashcard_limit = null
   subscription_status = active
