Fix the test bank/practice question page layout so the content is not cut off at the bottom.

- The page should allow the full question, answer options, rationale, clinical pearl, and key takeaway sections to be fully visible and scrollable.
- This issue appears after the recent header/navigation changes, so check whether a height, overflow, sticky container, or viewport calculation was broken.

What to fix
- Remove any layout rule that causes the test bank content area to be vertically clipped.
- Ensure the full page can scroll naturally to the bottom.
- If the layout uses split panels, both panels must be able to display all content without truncation.
- If one or both panels are intended to have internal scrolling, make sure the internal scroll area is tall enough and fully usable on desktop, tablet, and mobile.
- Ensure sticky headers or sticky nav bars are not overlapping or reducing the usable content height incorrectly.
- Recalculate any `100vh`, `calc(100vh - ...)`, fixed height, min-height, max-height, or overflow settings that may now be wrong after header changes.
- Check for parent containers using `overflow: hidden`, fixed heights, or flex/grid alignment that prevents children from expanding.

Rules
- Do not redesign the test bank page.
- Do not change the question content, rationale content, or answer logic.
- Do not remove the side-by-side layout if it is supposed to stay.
- Do not touch pricing, lessons, or unrelated pages.
- Only fix the layout/height/scroll behavior so the full content is accessible.
- Keep the page clean and visually balanced.

Implementation guidance
- Audit the main test bank page container, question panel, rationale/review panel, and any shared layout wrappers.
- Check whether the page wrapper has:
  - `height: 100vh`
  - `overflow: hidden`
  - nested flex children with missing `min-height: 0`
  - sticky header offsets that are no longer correct
- If using a two-column flex/grid layout:
  - allow the outer page to grow naturally, or
  - explicitly define safe internal scroll regions with correct height calculations
- Make sure bottom padding exists so the final card/section is not flush against the viewport edge or hidden behind sticky UI.
- Verify the page at common laptop screen heights, especially smaller screens like MacBook Air.

Acceptance criteria

  - answer options

  - why other options are incorrect
