TASK: CANONICALIZE LESSON TITLES ACROSS THE ENTIRE NURSENEST PLATFORM

You are working inside the NurseNest production platform.

Your task is to standardize lesson naming so every medical topic has one canonical title only, even if duplicate or alternate names already exist across lessons, flashcards, rationales, search results, breadcrumbs, URLs, or metadata.

This task must improve:
- lesson consistency
- navigation
- search
- breadcrumbs
- SEO
- internal linking
- image matching
- rationale linking
- flashcard linking

IMPORTANT RULES

1. Work on the PRODUCTION / PUBLISHED database only.
2. Do NOT change tier assignments.
3. Do NOT delete content unless it is a true duplicate and safely merged.
4. Keep backend tier separation intact.
5. Users should see one clean medical topic title only.
6. Remove visible tier words such as RPN, RN, NP, NCLEX, LVN from user-facing titles.
7. Preserve all existing lesson content, flashcards, linked images, and exam question references during merges.

------------------------------------------------
PRIMARY GOAL

Each topic should have:

- 1 canonical lesson title
- 1 canonical display title
- 1 canonical slug
- mapped synonyms/aliases
- consistent breadcrumb label
- consistent flashcard/rationale references
- consistent internal search behavior

Example:

All of these should map to one canonical topic:
- Pulmonary Embolism
- PE
- Acute Pulmonary Embolism
- Pulmonary embolism overview
- RN Pulmonary Embolism
- NP Pulmonary Embolism

Canonical output:
- Title: Pulmonary Embolism
- Slug: pulmonary-embolism

------------------------------------------------
STEP 1 — SCAN FOR DUPLICATES AND VARIANTS

Scan all lesson-related tables and content sources for duplicate or near-duplicate topics.

Check:
- lessons
- lesson_title
- lesson_display_title
- lesson_slug
- lesson_header
- flashcards
- rationale lesson links
- test bank references
- CAT exam references
- image tags
- SEO metadata
- sitemap entries
- breadcrumbs
- search index
- related lesson cards
- recommendation widgets

Detect duplicates caused by:
- abbreviations
- alternate capitalization
- singular vs plural
- longer descriptive titles
- tier prefixes
- exam prefixes
- “overview”, “management”, “pathophysiology”, “treatment” suffixes
- spelling inconsistencies
- punctuation variation

------------------------------------------------
STEP 2 — CREATE A CANONICAL TITLE RULESET

Use these title rules:

1. Title should be the standard medical condition or concept name only.
2. Prefer the most recognizable exam-friendly medical term.
3. Avoid extra words unless medically necessary.
4. Remove tier labels.
5. Remove filler words such as:
   - overview
   - management
   - lesson
   - guide
   - explained
   - review
   - fundamentals
   - introduction to
   - understanding
6. Keep medically important qualifiers only when needed.
7. Keep titles short, clean, and consistent.

TITLE RULES
- ideal length: 2–4 words
- max length: 45 characters
- title case formatting
- no tier words
- no exam words
- no marketing words

------------------------------------------------
STEP 5 — MERGE DUPLICATE LESSON RECORDS SAFELY

When multiple lessons refer to the same topic:

1. Select a primary canonical lesson record.
2. Preserve the strongest content-rich record as the parent.
3. Merge useful content from duplicates into the canonical lesson only if non-duplicative.
4. Reassign linked flashcards, rationales, images, question references, breadcrumbs, and SEO references to the canonical lesson ID.
5. Retire duplicate records safely.

Do NOT hard delete unless safe.
Preferred approach:
- mark duplicates as merged/inactive
- add redirect or alias mapping
- preserve audit trail

------------------------------------------------
STEP 6 — STANDARDIZE SLUGS

Each canonical lesson should have one clean slug.

Slug rules:
- lowercase
- hyphen-separated
- no tier terms
- no unnecessary words
- use canonical topic only

Examples:
Pulmonary Embolism → pulmonary-embolism
Cardiac Tamponade → cardiac-tamponade
Peripheral Artery Disease → peripheral-artery-disease
Second-Degree AV Block Type I → second-degree-av-block-type-1
Supraventricular Tachycardia → supraventricular-tachycardia

If old slugs exist, create redirects or alias resolution so old links do not break.

STEP 9 — PROTECT TIER-SPECIFIC CONTENT WITHOUT SHOWING TIER WORDS

Do not collapse RN and NP content into one lesson if your architecture intentionally stores separate tier lesson bodies.

Instead:
- keep separate backend records if needed
- use the same canonical user-facing title
- keep tier logic hidden in the backend
- ensure user only sees the topic title

- pulmonary-embolism (RN record)
- pulmonary-embolism (NP record with tier key)
Frontend:
- Pulmonary Embolism

The displayed title should be identical, while the fetched content depends on the user’s tier.

------------------------------------------------
STEP 10 — HANDLE LONG TITLES

Any title longer than 45 characters should be shortened to the canonical medical topic.

Examples:
"Understanding the Pathophysiology of Cardiac Tamponade" → "Cardiac Tamponade"
"Advanced Pharmacologic Treatment of Heart Failure" → "Heart Failure Treatment" or "Heart Failure"
"Pathophysiology and Management of Acute Pulmonary Embolism" → "Pulmonary Embolism"

Prefer the disease/topic alone unless a second word is truly necessary.
