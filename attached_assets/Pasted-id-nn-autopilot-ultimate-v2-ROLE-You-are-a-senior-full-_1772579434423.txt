3) ENGINE: BLOG CLUSTER GENERATOR (300+ ARTICLES/YEAR)
============================================================

Admin tab: Blog Cluster Engine

Purpose:
Generate pillar + supporting articles in exam-specific silos with schema + internal links + conversion CTAs.

Silo rules:
- Nursing: /rex-pn, /nclex-pn, /nclex-rn
- NP Canada: /np-canada only
- NP US: /np-aanp-fnp, /np-ancc-fnp, /np-agnp, /np-pmhnp
- Allied: /allied/[profession]

Pillar requirements:
- 2500–4000 words
- H1/H2/H3 hierarchy
- “How to use this guide” section
- FAQ 8–12 questions + FAQ schema
- Breadcrumb schema
- Internal links: at least 8 (practice page, mock exam, pricing, relevant cluster posts)
- Provide diagram requests (to Visual Factory)

Supporting article requirements:
- 1500–2500 words
- Internal links: at least 5
- FAQ 6–10 + schema
- CTA blocks:
  - Start Free Trial (where applicable)
  - Take a Mock Exam
  - View Study Plan
  - Upgrade for full rationales/diagrams

Scheduling default:
- 3 articles/week rotating nursing/NP/allied
- 1 pillar/month per major track

Validation:
- duplicate check (title + content similarity)
- “medical accuracy red flags” check (basic rule-based + allow review queue)
- plagiarism check heuristic (avoid verbatim copying; must be original)

Publishing:
- Create DRAFT entries in publishing_queue
- If auto-publish enabled and validation passes, schedule to publish
- Otherwise require admin approval

4) ENGINE: PRACTICE QUESTION SEO PAGES (MASS INDEXING)
============================================================

Admin tab: Practice SEO Engine

Purpose:
Create SEO landing pages targeting long-tail “practice questions” keywords that convert.

URL patterns:
- /practice/rex-pn/[topic]-practice-questions
- /practice/nclex-pn/[topic]-practice-questions
- /practice/nclex-rn/[topic]-practice-questions
- /practice/np-canada/[topic]-practice-questions
- /practice/np-aanp-fnp/[topic]-practice-questions
- /practice/allied/[profession]/[topic]-practice-questions

Page template:
1) Intro (400–700 words) with blueprint relevance
2) 5–10 sample questions (rendered from QBank or generated set)
3) Gating rules:
   - anonymous: show 2 questions only
   - logged-in free: show 5 questions with partial explanations
   - trial completion unlocks full explanations for trial set only
   - paid: full
4) CTAs: Start Trial / Take Mock / Unlock Full Bank
5) FAQ 8+ + schema + breadcrumbs
6) Internal links:
   - pillar guide
   - mock exam page
   - qbank page
   - pricing page
   - one related blog

Scheduling:
- 10 practice pages/week minimum

Validation:
- ensure correct silo and track_key
- ensure internal links exist
- ensure no cross-country contamination (US vs CAN terms/units)

7) ENGINE: QUESTION FACTORY (DAILY QBANK GROWTH)
============================================================

Admin tab: Question Factory

Purpose:
Generate batches of new questions with strict validation and ingest.

Batch size:
25/50/100

Per-track rules:
- Nursing includes NGN types as appropriate
- NP is MCQ-only, certification-specific
- Allied uses profession-valid formats only (no NGN gimmicks)

Validation hard-fail:
- strict JSON parse
- count exact
- domain weighting ±3% (or profession rules)
- prohibited formats blocked
- unit correctness (CAN vs US)
- scope enforcement (PN vs RN vs NP)
- duplication checks (minhash + similarity)
- rationale min length (config; default 250–400 for batch; deep upgrade later)

Workflow:
- Generate → Validate → Draft Set → Admin approve → Ingest → Publish

Metadata logging:
runId, promptVersion, model, timestamp, costs, pass/fail summary
