Problem 2: Homepage hero is auto-scrolling down to the image area
The homepage hero section is automatically pulling the page downward toward the analytics/images area, and this is bad UX. I want users to stay at the top of the hero naturally when the page loads so they can read the main headline and copy first.

What must be fixed:
1. Find the exact cause of the automatic downward scroll/jump on page load.
2. Remove any logic that forces the viewport to move to the image area, analytics cards, or lower hero content on initial render.
3. Check for and remove or disable any of the following if they are causing this:
   - `scrollIntoView()`
   - `window.scrollTo()`
   - autofocus on an element lower on the page
   - focus being programmatically moved to an element in the hero image section
   - hash/anchor navigation on load
   - layout effects or effects tied to refs
   - animation libraries that are moving the viewport
   - carousel/image logic that steals focus on mount
   - intersection observer side effects
   - hydration mismatch causing reflow and jump
4. Keep the images/analytics preview loading normally, but only as passive content.
5. It is completely fine if images lazy load when the user scrolls to that area.
6. Do not auto-scroll, auto-focus, or auto-jump to the image block for any reason on initial page load.
7. The page should open with the viewport anchored at the top of the hero headline and supporting text.
8. Make sure this remains true on:
   - desktop
   - tablet
   - mobile
9. If there is code restoring prior scroll position incorrectly, fix that too.
10. If there is a component mounting twice in development and triggering scroll behavior, fix the behavior safely rather than masking it.

Implementation checklist:
- Audit the homepage hero component, parent layout, and any navbar or route transition hooks.
- Search the codebase for:
  - `scrollIntoView`
  - `scrollTo`
  - `focus()`
  - `autoFocus`
  - hash fragments
  - route transition effects
  - hero image refs
  - animation on mount
- Check whether an image, card, or lower CTA is receiving focus on load.
- Check whether the page is trying to highlight a selected element below the fold.
- Check for browser scroll restoration logic and disable unwanted restoration for this page if needed.

Expected result for hero section:
When a user lands on the homepage, they should remain at the top of the hero and be able to read the headline and text without the page pulling them downward. The lower images/cards can still load, but they must not move the viewport.
