STEP 2 — DEFINE ALLOWED QUESTION FORMATS PER PROFESSION
-----------------------------------------------------

For each profession, enforce allowed types:

Common Types:
- MCQ_SINGLE (single best answer)
- MCQ_SATA (select all that apply)
- CASE_BASED_CLUSTER (multiple questions tied to 1 scenario)
- NUMERIC_ENTRY (dosage/calculations)
- MATCHING
- MATRIX_SINGLE
- MATRIX_MULTI
- SHORT_CASE_ANALYSIS (psychotherapy/social work)
- PRIORITIZATION
- ORDERING

Enforce realistic usage:

MLT:
- Case-based lab interpretation
- Calculation (dilutions, units)
- Data tables (reference ranges)
- Matching lab findings
- No NGN bowtie unless supported by real exam format

Pharmacy Tech:
- High emphasis on calculations
- Prescription interpretation
- Drug class identification
- Matching generic/brand
- Limited multi-response unless realistic

Paramedic:
- Scenario-based
- ECG interpretation (if supported)
- Prioritization
- Protocol-based MCQs
- Calculation for medication dosing

RRT:
- ABG interpretation
- Vent settings
- Case-based respiratory scenarios
- Numeric calculations
- Flow/volume loops (if graphic supported)

Psychotherapist:
- Case vignettes
- Ethical/legal scenario analysis
- Best intervention questions
- No math
- No drag-and-drop unless official exam supports it

Social Work:
- Policy, ethics, crisis intervention
- Cultural competence scenarios
- Case analysis
- Prioritization
- No unnecessary matrix formats

Addictions Worker:
- Stages of change
- Harm reduction
- Crisis intervention
- Motivational interviewing scenarios
- Case-based reasoning

-----------------------------------------------------
STEP 3 — MOCK EXAM ASSEMBLY ENGINE RULES
-----------------------------------------------------

When generating a mock exam:

1) Respect domain weight ranges.
2) Enforce minimum questionTypesRequiredMin.
3) Prevent overuse of exotic formats unless official exam includes them.
4) Ensure:
   - No more than 10% trick/negative phrasing questions.
   - No unrealistic distractors.
   - No two nearly identical stems.
   - No reuse within same exam.

5) If blueprint cannot be satisfied:
   - Throw an admin validation error.
   - Do not publish exam.
   - Log missing domain or format.

-----------------------------------------------------
STEP 4 — FORMAT VALIDATION RULES
-----------------------------------------------------

Before publishing any mock exam:

Validate:
- Total question count within allowed range.
- Domain distribution within weight tolerance ±2%.
- Required calculation questions present (if applicable).
- Required case clusters present (if applicable).
- Question formatting matches allowed types only.
- Scoring model matches blueprint.

Block publishing if:
- A prohibited format is included.
- Required case-based questions missing.
- Calculation questions missing when required.
- More than 20% of questions from one domain unless blueprint allows it.

-----------------------------------------------------
STEP 5 — SCORING ENGINE
-----------------------------------------------------

Implement scoring rules per profession:

If scoringModel = "dichotomous":
  - 1 point correct, 0 incorrect.

If "partialCredit":
  - SATA / matrix allow proportional scoring.
  - Deduct for incorrect selections if configured.

If "scaled":
  - Convert raw score to scaled score using linear transformation.
  - Provide percentile estimation (internal only, not guarantee).

Display:
- Domain breakdown
- Strength/weakness by competency
- Recommendation for remediation
