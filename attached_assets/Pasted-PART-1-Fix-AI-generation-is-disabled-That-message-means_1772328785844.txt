PART 1 — Fix: “AI generation is disabled”
That message means your backend has an environment-level kill switch active.
This is almost always controlled by something like:
AI_ENABLED=false
DISABLE_AI=true
or an Admin safety toggle flag in your database
What’s likely happening
Your Product Generator is calling something like:
POST /api/ai/generate-product
And the server is checking:
if (!process.env.AI_ENABLED) {
   return res.status(403).json({ error: "AI generation is disabled" })
}
So even though the UI exists, the backend blocks it.
Prompt to Fix AI Toggle (Send to Replit AI)
Use this:
Enable AI generation in production and development.
Remove or modify any global AI kill switch that blocks product generation.
Replace hard-disable logic with role-based enforcement:
Admin users can always generate AI content.
Free-tier users respect rate limits.
If AI safety mode exists in Admin > AI Safety, change it from global disable to:
Rate limit
Max tokens
Daily quota
Add visible toggle indicator in Admin panel that shows:
AI Enabled: true/false
Current daily usage
Model being used
Ensure product generator calls the correct AI endpoint and does not silently fail.
Return descriptive errors instead of “AI generation is disabled”.
If AI is expensive and you’re worried about cost:
Instead of disabling it completely, implement:
3 generations per session for free
Unlimited for admin
Daily cap
Never globally disable it again — it kills workflow.
