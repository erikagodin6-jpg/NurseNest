TECHNICAL TASKS

1. Audit nav source of truth
Find where navigation is defined:
- static config
- route-based arrays
- CMS/menu config
- layout wrappers
- page-level overrides
- header component props
- user-role-aware nav builders

Identify whether multiple conflicting nav systems exist.

2. Standardize navigation architecture
Create one clear source-of-truth system for navigation, such as:
- global ecosystem nav
- section-specific nav variants
- route-aware but tightly controlled dropdown definitions

Recommended model:
- global top strip
- section shell nav
- optional contextual subnav where appropriate

Do not let page components assemble their own ad hoc nav items.

3. Route-to-nav mapping
Create strict mapping from route groups to allowed nav shells.

For example:
- /newgrad/* -> New Grad nav shell only
- /allied/* -> Allied nav shell only
- /pricing/allied or allied pricing routes -> Allied nav shell
- nursing lesson/practice/tier routes -> Nursing nav shell
- generic marketing pages -> Main/default marketing nav shell

If using subdomains or section-specific prefixes, ensure they resolve consistently.

4. Fix dropdown contamination
Audit all dropdown menus for incorrect child items.
Specifically verify:
- New Grad dropdown
- Learning dropdown
- Resources dropdown
- Career Guides dropdown
- Allied dropdown if present
- mobile drawer equivalents

Ensure no unrelated section links appear because of:
- concatenated arrays
- spread operators combining configs
- fallback children
- route partial matches
- old experiment flags

5. Stabilize header across subpages
Right now some subpages appear to swap nav/header unexpectedly.

Required behavior:
- pages within the same section should maintain a stable header/nav model
- moving from a section home page to a section subpage should not unexpectedly change top nav identity
- a user should not see one nav on the section home page and a different unrelated nav on the child page unless intentionally designed

Audit:
- section layouts
- nested layouts
- shared wrappers
- route transitions
- hydration differences
- client-side vs server-side route recognition
- conditional rendering based on pathname fragments

6. Fix mobile navigation too
This bug often exists in both desktop and mobile separately.

Verify:
- mobile menu shows correct section links
- New Grad mobile dropdown does not include Allied Health
- allied mobile nav does not include nursing-tier tabs
- mobile back/close states work correctly
- no stale menu state persists when moving between sections
