You are working on the NurseNest production codebase.

Implement PHASE 4: SCALE, DATA SAFETY, OPERATIONAL CONTROL, AND REVENUE PROTECTION.

This phase focuses on preventing large-scale failures, protecting revenue, and ensuring the platform remains stable as content, users, and complexity grow.

==================================================
PRIMARY GOAL
============

Ensure NurseNest remains:

* stable under scale
* safe under schema evolution
* recoverable under failure
* observable under real-world usage
* protected from revenue loss

==================================================
SECTION 1 — DATA MIGRATION + ROLLBACK SAFETY (CRITICAL)
=======================================================

Build a safe migration system for all schema and data changes.

Requirements:

* versioned migrations (up + down)
* dry-run mode before applying
* migration validation checks
* automatic rollback if migration fails
* migration audit logs

Add:

* backward compatibility layer so old content still renders
* schemaVersion awareness across all content

Prevent:

* breaking existing exams, lessons, or flashcards after schema changes

==================================================
SECTION 2 — TENANT / SCOPE ISOLATION
====================================

Isolate failures by content scope.

Scopes:

* profession (RPN, RN, NP, Allied)
* exam type (CAT vs standard)
* language
* region

Requirements:

* failure in one scope must NOT affect others
* fallback routing must be scoped
* quarantine must be scoped
* kill switches can target specific scope

==================================================
SECTION 3 — PROGRESSIVE DEGRADATION SYSTEM
==========================================

Instead of jumping directly to fallback, degrade in layers:

Level 1:

* disable animations
* disable heavy UI

Level 2:

* disable advanced components
* simplify layout

Level 3:

* switch to safe renderer

Level 4:

* switch to static backup

Level 5:

* substitute content

Automatically escalate based on:

* error rate
* render failures
* latency thresholds

==================================================
SECTION 4 — SHADOW BACKUP PLATFORM (LITE MODE)
==============================================

Build a minimal backup delivery system:

NurseNest Lite:

* minimal API
* minimal renderer
* prebuilt payloads
* no heavy dependencies

If main system fails:
→ route traffic to Lite mode

Requirements:

* compatible with entitlement system
* supports core study flows
* always available

==================================================
SECTION 5 — OPERATIONAL RUNBOOK SYSTEM
======================================

Create predefined incident playbooks.

Include runbooks for:

* exam not loading
* CAT failure
* subscriber access issues
* billing sync issues
* slow site / outage
* broken content

Each runbook must include:

* symptoms
* likely causes
* steps to diagnose
* actions to take
* which kill switches or flags to use
* communication template to send

Accessible in admin panel.

==================================================
SECTION 6 — USER BEHAVIOR TELEMETRY
===================================

Track behavioral signals beyond errors:

* retry frequency
* fallback usage
* drop-off points
* repeated failures
* most problematic exams/content
* session abandonment points

Use this data to:

* identify weak areas
* prioritize fixes
* detect silent failures

==================================================
SECTION 7 — GRACEFUL TIMEOUT SYSTEM
===================================

Add strict timeouts for:

* exam load
* CAT start
* entitlement check
* content fetch

If timeout exceeded:
→ trigger fallback automatically

Prevent:

* infinite loading states
* frozen UI
