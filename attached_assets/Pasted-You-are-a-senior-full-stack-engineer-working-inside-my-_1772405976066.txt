B) New API endpoints under /api/generator-v2

1) POST /api/generator-v2/generations

  template, exam, region, targetCount, chunkSize, promptBase, settings
}

2) POST /api/generator-v2/generations/:id/run
Starts/resumes generation:
- Runs content generation if template includes content
- Runs QBank generation worker until completion OR until user stops
- Emits events

3) POST /api/generator-v2/generations/:id/pause

4) GET /api/generator-v2/generations/:id/status
Returns {status, created_count, target_count, last_error, distributions, recent_events}

5) GET /api/generator-v2/generations/:id/questions?offset&limit
Returns saved questions

6) POST /api/generator-v2/generations/:id/compile

Returns a downloadable PDF OR stores a “draft product” record depending on existing site patterns.

C) Worker logic (SERVER)
Implement a robust generation worker with these behaviors:

- Chunk request format (JSON only):
{
  "items":[ ... ],
  "meta": { "startId": 46, "count": 15, "chunk": 4 }
}

- Chunk prompt must include:
  - Next idx range
  - Remaining needs to hit distributions:
    * total target
    * 70% MCQ / 30% SATA
    * difficulty: 30% moderate / 50% hard / 20% very_challenging
  - Systems coverage (rotate through systems; prevent clumping)
  - Anti-duplication: provide last 20 stem hashes/snippets to avoid repeats
  - Strict rule: output EXACTLY meta.count items

- Validation rules:
  - MCQ: exactly 4 choices labeled A-D, exactly 1 correct answer
  - SATA: >=5 choices labeled A-E (up to H allowed), 2-5 correct answers
  - correct answers must match labels
  - stem must be non-empty and not duplicated (hash)
  - rationale fields must be present
  - Canadian/SI lab values when labs included

- Partial accept:
  - Save valid items immediately
  - Collect invalid ids + reasons
  - Request replacements:
    { "replacements":[ { "id": 49, ...item... }, ... ] }
  - Only replace invalid ids, do not rewrite valid items.

- Resume:
  - Use created_count + max idx in DB to resume
  - prompt_state tracks distributions already achieved

- Safety:
  - If model output is not valid JSON, automatically retry with smaller chunk size and stricter “JSON only” wrapper
  - If repeated failures occur, pause job with actionable last_error
