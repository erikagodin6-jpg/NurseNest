FLASHCARD REQUIREMENT
For every newly created question, automatically create at least one linked flashcard for the test bank/flashcard system.

Flashcard rules:
- front should test the key concept in a concise way
- back should provide the correct answer plus a brief high-yield explanation
- if the question is scenario-based, the flashcard should distill the teachable point
- link the flashcard to the relevant lesson when possible
- assign the flashcard to the same tier and category
- ensure flashcards are randomized in the user experience as already designed

Example flashcard transformation:
Question concept: loop diuretic adverse effect
Front: What electrolyte imbalance is most associated with loop diuretics?
Back: Hypokalemia. Loop diuretics increase potassium excretion, which can lead to weakness and dysrhythmias.

If a question contains multiple teachable facts, optionally create:
- one primary flashcard
- one secondary reinforcement flashcard
but avoid redundancy

RATIONALE REQUIREMENT
Each question must have a strong rationale that includes:
1. why the correct answer is correct
2. why the other options are incorrect when useful
3. a short clinical pearl
4. brief lesson linkage when appropriate

IMAGE REQUIREMENT
While generating questions and rationales, attach an appropriate existing NurseNest image to the rationale if an image already exists in the site image library and is relevant.

Use images only when they add value, such as:
- ECG strips
- anatomy diagrams
- medication mechanism visuals
- disease pathophysiology diagrams
- assessment graphics
- obstetric diagrams
- pediatric growth/assessment visuals
- wound/skin staging visuals
- neuro diagrams
- endocrine feedback diagrams
- lab interpretation visuals

Do not attach random decorative images.
Do not attach images that do not match the concept.
Do not create broken image references.

Image logic:
- search the existing internal image/image asset library for a concept match
- if a strong match exists, assign the imageReference to the question rationale and flashcard support view if applicable
- if no appropriate image exists, leave imageReference null rather than forcing a bad match

Examples:
- MI rhythm question -> ECG image if available
- pressure injury staging -> staging image if available
- DKA pathophysiology -> endocrine/metabolic diagram if available
- fetal position question -> obstetric diagram if available
- cranial nerve question -> neuro anatomy image if available
