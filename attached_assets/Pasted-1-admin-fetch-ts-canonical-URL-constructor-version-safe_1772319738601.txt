1) admin-fetch.ts canonical URL-constructor version (safest)

If you haven’t already, your final adminFetch must do all 5:

new URL(url, window.location.origin) for relative URLs

inject creds into query for GET/HEAD

inject creds into body for non-GET

never send body on GET

accept body as object or string

If you want to sanity check quickly: search inside admin-fetch.ts for these red flags:

❌ url + "?" + ... string concatenation (replace with URL object)

❌ if (method === "GET") out.body = ... (must never happen)

❌ JSON.stringify(init.body) without first parsing strings (causes double stringify)

4) Shop: download & preview gotchas (critical)
A) Admin download must be blob(), not json()

If your shop download button currently does something like res.json(), it will break.

Use:

const res = await adminFetch(`/api/admin/shop/products/${id}/download`);
if (!res.ok) throw new Error("Download failed");
const blob = await res.blob();

const url = URL.createObjectURL(blob);
const a = document.createElement("a");
a.href = url;

// Try to respect server filename if set
const cd = res.headers.get("Content-Disposition") || "";
const match = cd.match(/filename\*?=(?:UTF-8''|")?([^";]+)"?/i);
a.download = match ? decodeURIComponent(match[1]) : `product-${id}.pdf`;

a.click();
URL.revokeObjectURL(url);
B) If you use admin creds in query for GET, don’t use it on POST

If you had anything like:

POST /api/admin/...?...username=...
replace with adminFetch body auth.
