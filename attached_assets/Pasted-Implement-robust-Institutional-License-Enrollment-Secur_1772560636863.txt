2) CODE REDEMPTION RULES
=========================================================

When a user enters an institution invite code:

Hard requirements:
- User must have verified email if requireEmailVerified = true
- Seat limit not exceeded
- Code not expired

Then enforce eligibility based on enrollmentMode:

A) DOMAIN_LOCK
- user.email domain must be in allowedEmailDomains
- If passes, auto-assign seat (student role)
- Log event in audit log

B) ROSTER_ALLOWLIST
- user.email must exactly match an active allowlist entry
- If match, auto-assign seat
- If not match, block with message:
  "Your email is not on the approved roster. Contact your program coordinator."

C) APPROVAL_REQUIRED
- user.email domain must be allowed (if allowedEmailDomains exists)
- Create institutionSeatRequest with status pending
- Show message:
  "Request submitted. Your instructor will approve access shortly."
- Institution admin / faculty_pro can approve requests
- Only after approval create institutionSeat (student role)
