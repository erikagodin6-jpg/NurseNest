Build an advanced theme engine for NurseNest so the logo, top bars, navigation, menus, buttons, cards, and key UI accents automatically adapt to the user’s selected theme in a polished, premium, production-ready way.

Main goal
Create a centralized theme system that supports:
- predefined NurseNest themes
- automatic logo color adaptation
- accessible contrast handling
- light and dark theme support
- saved user theme preferences
- optional custom theme creation later

Important constraints
1. Do not redesign the NurseNest logo artwork.
2. Do not alter the spacing, proportions, or alignment of the logo.
3. The logo must remain SVG-based and crisp at all sizes.
4. The site must keep its current structure and responsiveness.
5. Do not hardcode random colors across components. Everything must come from centralized theme tokens.
6. The system must work across the main site, allied health pages, new grad pages, dashboards, pricing pages, auth pages, and admin areas where appropriate.

Architecture requirements

1. Create a centralized theme token system
Implement one source of truth for all theme values.

Create theme tokens for:
- brand primary
- brand secondary
- accent
- surface background
- elevated surface
- border
- muted text
- heading text
- body text
- logo leaf color
- logo wordmark color
- top announcement bar background
- top announcement bar text
- navbar background
- navbar border
- menu hover background
- menu hover text
- menu active state
- button primary background
- button primary text
- button secondary background
- button secondary text
- card background
- card border
- input background
- input border
- focus ring
- success
- warning
- error

Use CSS variables or a theme object that maps cleanly into the current frontend stack.

Example naming style:
--color-brand-primary
--color-brand-secondary
--color-accent
--color-surface
--color-surface-elevated
--color-border
--color-text-primary
--color-text-secondary
--color-logo-leaf
--color-logo-wordmark
--color-topbar-bg
--color-topbar-text
--color-navbar-bg
--color-navbar-border
--color-menu-hover-bg
--color-menu-hover-text
--color-menu-active
--color-btn-primary-bg
--color-btn-primary-text
--color-focus-ring

2. Add multiple predefined branded themes
Implement at least these theme presets:
- pastel-lilac
- lavender-dream
- clinical-light
- soft-sage
- dusk
- dark-mode

Each preset should feel on-brand for NurseNest and should not look childish, overly neon, or generic.
Aim for soft, modern, feminine-clinical, premium educational platform styling.

3. Make the logo theme-aware
Use the SVG logo as a reusable Logo component.

Requirements:
- leaf icon color should be theme-driven
- wordmark color should adapt to background and contrast needs
- preserve exact logo artwork
- preserve exact spacing between leaf and brand name
- preserve header alignment and sizing

Implementation guidance:
- convert the leaf fill to currentColor or a CSS variable
- keep the wordmark fill controlled separately via a theme token
- do not rasterize the SVG
- do not regenerate the logo art

Example concept:
leaf uses var(--color-logo-leaf)
wordmark uses var(--color-logo-wordmark)

4. Apply the theme system across all major UI
Refactor shared layout and design components so they consume theme tokens instead of hardcoded colors.

Apply to:
- announcement bars at the top
- main header
- navigation background and borders
- desktop dropdown menus
- mobile menu
- footer
- tabs
- buttons
- pills/badges
- cards
- search inputs
- progress bars
- selected states
- hover states
- analytics widgets
- study dashboard highlights
- pricing badges
- auth forms

5. Add accessibility-aware contrast logic
Build a utility that checks whether text and icons have sufficient contrast against their background.

Requirements:
- ensure navigation text remains readable on all themes
- ensure logo wordmark remains readable in dark and light modes
- ensure button text passes contrast checks
- ensure hover and active states remain visible
- avoid low-contrast pastel-on-pastel combinations

If a theme token combination fails contrast requirements, automatically choose the safer text token.

6. Persist user theme choice
Save the user’s theme selection locally and, if the app already has user preferences/profile settings, also save it to the user profile so it persists across devices for signed-in users.

Requirements:
- load saved theme immediately on app startup
- avoid flash of incorrect theme
- apply theme before main render when possible

7. Add a theme switcher UI
Create or improve the user-facing theme picker.

Requirements:
- show theme preview swatches
- clearly label each theme
- instantly preview changes on selection
- include dark mode
- keep it elegant and on-brand
- place it where users can find it easily without cluttering the navbar

Optional enhancement:
Include “Use system theme” for automatic dark/light preference.

8. Prepare for future custom themes
Build the architecture so a custom theme builder can be added later without a full rewrite.

Prepare support for:
- user-selected primary color
- user-selected accent color
- optional saved custom themes
- admin-controlled locked theme presets

Do not fully build the custom theme editor unless it is easy within the current structure, but set up the data model so it can be added cleanly.

9. Avoid regressions
Do not break:
- navbar height
- logo height
- logo click-through to homepage
- spacing in menus
- dropdown alignment
- mobile responsiveness
- sticky headers
- auth layouts
- existing role-based dashboards

10. Clean up old styling
Remove or refactor scattered hardcoded colors in shared components.
Consolidate duplicate theme logic.
Prefer shared utility functions and shared theme tokens over page-specific hacks.

11. Validate thoroughly
Test across:
- desktop and mobile
- main marketing pages
- nursing tier pages
- allied health pages
- new grad pages
- dashboards
- pricing pages
- account settings
- auth pages
- admin pages if theme applies there

Validate:
- logo color updates correctly
- top bars and menus update correctly
- no unreadable text
- no layout shift
- no broken imports
- no production-only asset failures
- no hydration mismatch or theme flicker

12. Deliverable expectation
At the end, provide:
- a summary of the theme architecture
- list of theme presets created
- list of shared components refactored
- any remaining hardcoded color areas that still need cleanup
- confirmation that the logo, bars, and menus now respond to theme selection

Design direction
The final result should feel like a polished SaaS education platform:
- soft but professional
- premium and cohesive
- modern-clinical
- elegant in both light and dark modes
- consistent with NurseNest branding

Do not produce a quick patch.
Implement this as a scalable theming system that can support long-term growth of NurseNest.
