Upgrade NurseNest flashcards into a true adaptive learning engine.

The current flashcards should not behave like static shuffled cards only. They should become an intelligent study system that adapts to each user’s weaknesses, confidence, and performance over time.

IMPORTANT
Do not remove the existing flashcard UI.
Add adaptive logic underneath it so the experience becomes smarter and more personalized.

CORE GOAL

Flashcards should behave like:
- untimed exam-style question cards
- with rationale
- with incorrect option explanations
- with lesson links
- with adaptive repetition and weakness targeting

The system should continuously decide:
- what the user needs next
- what topics need more reinforcement
- what cards should be repeated sooner
- what concepts are already mastered

--------------------------------------------------
PART 2 — CONFIDENCE-BASED LEARNING
--------------------------------------------------

After each card, ask the user a confidence question:

How sure were you?

Options:
- Confident
- Somewhat unsure
- Guess
or
- Got it
- Unsure
- Need more review

Store this value.

Logic:
- if user gets card correct but selected “Guess” or “Unsure”, do not treat it as mastered
- if user gets card correct and selected “Confident”, reduce repetition frequency
- if user gets card incorrect, increase repetition priority regardless of confidence

This makes the system smarter than simple right/wrong logic.

--------------------------------------------------
PART 3 — ADAPTIVE CARD PRIORITIZATION
--------------------------------------------------

Every card should receive a dynamic priority score.

Example scoring model:

priorityScore =
  weakTopicWeight +
  incorrectHistoryWeight +
  lowConfidenceWeight +
  daysSinceSeenWeight +
  flaggedWeight +
  blueprintImportanceWeight +
  examUrgencyWeight

Possible weighting:

if card.wasIncorrectRecently:
  +5

if card.confidence == "Guess":
  +4

if card.flagged:
  +3

if card.topic in weakTopics:
  +4

if card.notSeenInLongTime:
  +2

if card.mastered:
  -5

if card.correctStreak >= 3 and confidence high:
  -4

Then sort cards by priority and shuffle intelligently within tiers.

--------------------------------------------------
PART 4 — SPACED REPETITION RULES
--------------------------------------------------

Implement spaced repetition intervals.

If a card is:
- incorrect → show again soon
- low confidence → show again fairly soon
- correct and confident → delay review longer
- repeatedly mastered → push far into future

Suggested review timing:

incorrect:
- same session or next day

correct but unsure:
- 2 to 4 days

correct and confident:
- 7 to 14 days

mastered repeatedly:
- 21 to 45 days

Store nextReviewAt for each card.

--------------------------------------------------
PART 5 — WEAK AREA DETECTION
--------------------------------------------------

The system must identify weak areas automatically.

Weak areas can be based on:
- low accuracy
- repeated misses
- low confidence
- high response time
- frequent flagging
- lack of recent review

Example weak area detection:

if topicAccuracy < 70%
  weakArea = true

if subtopicAccuracy < 65%
  weakArea = true

if 3 recent cards incorrect in same concept
  weakArea = true

Display weak areas on user dashboard.

Examples:
- Hyperkalemia Management
- ECG Interpretation
- Pediatric Fever
- Renal Pharmacology
- Sepsis Recognition

--------------------------------------------------
PART 6 — SMART SESSION TYPES
--------------------------------------------------

Build multiple adaptive session types.

1. Recommended Session
Mix of weak topics + due cards + exam-high-yield cards

2. Weak Areas Session
Only cards from weak systems/subtopics

3. Due for Review Session
Only spaced repetition cards due today

4. Flagged Cards Session
Only flagged items

5. Rapid Review Session
High-yield compact review

6. Mixed Adaptive Session
Best overall personalized mix

7. Pre-Exam Boost Session
High priority + weak area + blueprint-heavy cards

PART 9 — MASTERY MODEL
--------------------------------------------------

Each card should have a mastery state.

Possible values:
- new
- learning
- improving
- nearlyMastered
- mastered

Example logic:

new:
never seen

learning:
seen 1 to 2 times, accuracy inconsistent

improving:
seen multiple times, getting better

nearlyMastered:
2 to 3 correct with moderate/high confidence

mastered:
3+ correct with high confidence and spaced repetition success

Display this on the dashboard and optionally on cards.

--------------------------------------------------
PART 12 — RANDOMIZATION WITH INTELLIGENCE
--------------------------------------------------

Do not simply randomize all cards equally.

Use adaptive shuffle:

Step 1:
group cards by priority band
- urgent review
- weak area
- due soon
- mixed reinforcement
- mastered/low priority

Step 2:
shuffle within each band

Step 3:
assemble session from weighted mix

Example 20-card session:
- 6 weak area cards
- 4 due review cards
- 4 recent incorrect cards
- 4 mixed high-yield cards
- 2 reinforcement/mastery cards

This keeps sessions varied but purposeful.
