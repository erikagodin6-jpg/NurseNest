Update the NurseNest frontend so the very top announcement/utility bar always matches the currently selected custom theme.

Goal:
When the user changes theme (Lavender, Slate, Mint, Clinical, Coral, etc.), the very top bar must automatically update to that same theme instead of staying blue/purple.

Requirements:
1. Find the topmost header/announcement/utility bar component above the main navigation.
2. Remove any hardcoded blue/purple background, gradient, border, icon color, and text color values.
3. Make this bar use the same active theme source as the rest of the site:
   - the same theme context, theme store, CSS variables, or data-theme attribute already used by the page
   - do not create a separate theme system
4. Bind the top bar styles to theme tokens, for example:
   - background: var(--theme-primary) or the current theme gradient token
   - text: var(--theme-primary-foreground)
   - border/separator/icon hover states should also use theme-aware variables
5. If the site uses gradients per theme, the top bar should use that theme’s gradient.
6. If the site uses solid colors per theme, the top bar should use the matching primary color.
7. Keep contrast accessible:
   - automatically use the correct foreground token for dark vs light themes
   - links, icons, and small text must remain readable
8. Preserve existing layout, spacing, links, and responsive behavior.
9. Ensure this works instantly when the theme changes without requiring a page refresh.
10. Ensure the theme persists on reload exactly the same way as the rest of the site.

Implementation notes:
- Reuse existing theme tokens only.
- If the top bar currently has inline styles, replace them with theme-driven classes or CSS variables.
- If needed, add missing variables such as:
  --topbar-bg
  --topbar-fg
  --topbar-border
  --topbar-hover
  but derive them from the active theme, not from fixed colors.
- Audit both desktop and mobile headers so the top strip matches theme in all breakpoints.
- Check all themes in the theme picker and confirm the top bar changes with each one.

Deliverables:
1. Updated component code
2. Updated CSS/Tailwind classes/theme tokens
3. Any theme mapping changes required
4. Final QA checklist showing the top bar now matches every custom theme
