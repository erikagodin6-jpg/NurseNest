=========================================================
1) DEFAULT ENROLLMENT MODEL
=========================================================

Set enrollmentMode default to "DOMAIN_LOCK" for all new institutions.

Institution can optionally switch to:
- "APPROVAL_REQUIRED"
- "ROSTER_ALLOWLIST"

=========================================================
2) DATA MODEL
=========================================================

Update institutions table:

- enrollmentMode ("DOMAIN_LOCK" | "APPROVAL_REQUIRED" | "ROSTER_ALLOWLIST") default "DOMAIN_LOCK"
- allowedEmailDomains string[] NOT NULL (must be set during institution onboarding)
- requireEmailVerified boolean default true
- allowPersonalEmailFallback boolean default false
- fallbackApprovalRequired boolean default true

Add tables:

institutionSeatRequests
- id
- institutionId
- userId
- requestedAt
- status ("pending" | "approved" | "rejected")
- reason nullable
- decidedAt nullable
- decidedByUserId nullable

institutionRosterAllowlist (only used if enrollmentMode = ROSTER_ALLOWLIST)
- id
- institutionId
- email (normalized lowercase)
- status ("active" | "revoked")
- addedAt
- addedByUserId

institutionAuditLog
- id
- institutionId
- actorUserId
- actionType
- metadata JSON
- createdAt

=========================================================
3) REDEMPTION FLOW (CODE + ELIGIBILITY)
=========================================================

When user enters invite code:

Hard requirements:
- code valid and not expired
- seat cap not exceeded
- user is authenticated
- if requireEmailVerified = true: user email must be verified

Eligibility rules by mode:

A) DOMAIN_LOCK (DEFAULT)
- extract domain from user email
- if domain matches allowedEmailDomains:
   - auto-assign seat (student role)
   - accessEnd computed by licenseModel (COHORT -> semesterEndDate, INDIVIDUAL -> examDate or defaultDurationDays)
   - log: "seat_granted_domain_lock"
- else:
   - if allowPersonalEmailFallback = false:
       block with message:
       "This license is restricted to your school email. Please sign in with your student email."
   - if allowPersonalEmailFallback = true:
       create institutionSeatRequest pending if fallbackApprovalRequired = true
       else block

B) APPROVAL_REQUIRED
- if allowedEmailDomains set: require domain match OR create pending request depending on policy
- always create institutionSeatRequest pending
- show message:
  "Request submitted. Your instructor will approve access."

C) ROSTER_ALLOWLIST
- normalize user email to lowercase
- allow only if email exists active in allowlist
- else block with message:
  "Your email is not on the approved roster. Contact your program coordinator."

Prevent abuse:
- one seat per user per institution
- rate limit code attempts per IP + per user
- audit log for all accept/deny actions
