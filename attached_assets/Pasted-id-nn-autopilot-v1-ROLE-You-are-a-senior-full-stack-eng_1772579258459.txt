========================================================
B) AUTOMATION ENGINE 1: BLOG CLUSTER GENERATOR (300+/year)
========================================================
Goal:
Generate pillar + cluster articles for each track with internal linking, schema, and conversion CTAs.

Inputs:
- Keyword queue (from Keyword Discovery)
- Track (nursing/NP/allied)
- Region (US/CAN)
- Tone (clinical, friendly, exam-focused)
- Word count (1500–2500 supporting, 2500–4000 pillar)

Outputs:
- SEO title + meta description
- Slug and URL placement (proper silo)
- Content with H1/H2/H3
- Internal links (5 minimum)
- FAQ section (8–12 questions)
- JSON-LD schema (FAQ + Breadcrumb + Course/Product where relevant)
- Suggested diagram requests (to Visual Factory)

Scheduling defaults:
- 3 posts/week total across nursing/NP/allied with rotation
- 1 pillar/month per major track

Publishing rules:
- Draft → QA gate → queued → publish
- Auto-publish only if validation passes and admin enables

========================================================
C) AUTOMATION ENGINE 2: PRACTICE QUESTION SEO PAGES (mass indexing)
========================================================
Goal:
Create SEO-targeted practice pages that rank for long-tail “practice questions” queries and convert to trials/subscriptions.

URL patterns:
- /practice/[track]/[topic]-practice-questions
Examples:
- /practice/rex-pn/endocrine-practice-questions
- /practice/nclex-rn/pharmacology-practice-questions
- /practice/allied/mlt/hematology-practice-questions
- /practice/np-canada/hypertension-practice-questions

Page sections:
1) 400–700 word intro (exam relevance + blueprint tie-in)
2) 5–10 sample questions (with limited access controls)
3) Locked explanations until signup/trial completion (configurable)
4) CTA blocks: “Start Free Trial”, “Take a Mock Exam”, “View Study Plan”
5) FAQ (8+) + schema
6) Internal links to: pillar guide, mock exams, qbank, pricing

Gating:
- Anonymous: show stems only or first 2 answers
- Logged-in free: show full questions but limited explanations
- Trial completed: unlock trial explanations
- Paid: full access

========================================================
F) QUESTION FACTORY (daily): batch generation + validation + ingest
========================================================
Goal:
Automatically generate and ingest questions for:
- Nursing (NGN where appropriate)
- NP (MCQ only)
- Allied (profession-accurate formats only)

Admin route:
- /admin/question-factory

Batch sizes:
- 25/50/100

Hard validation:
- Strict JSON parse
- Domain distribution ±3%
- Required type mix per exam/profession
- Rationale min word count (config)
- Units and scope validation (US vs CAN; PN vs RN; NP)
- Duplicate/near-duplicate detection
- Prohibited formats blocked (NP/allied)

Workflow:
- Generate → Validate → Draft Set → Admin review → Publish
- Store run metadata: runId, promptVersion, timestamp, cost estimate, model

========================================================
G) AUTO CONTENT EXPANSION ENGINE (the multiplier)
========================================================
Goal:
When any page begins to perform (ranking/traffic/conversions), automatically expand it into a hub.

Trigger rules:
- Page enters top 20 for target query OR
- Traffic > threshold OR
- Trial conversions > threshold

Expansion outputs:
1) 3 supporting blog posts targeting adjacent long-tail
2) 1 practice question SEO page
3) 1 visual infographic + 3 Pinterest pins
4) 20-question mini quiz set linked to that topic
5) Internal linking updates (hub-spoke)

Must remain in same silo (don’t cross US/Canada NP content).

- Auto-publish OFF by default

5) Logging
