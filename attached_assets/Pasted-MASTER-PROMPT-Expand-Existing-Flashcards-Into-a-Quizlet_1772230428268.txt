MASTER PROMPT: Expand Existing Flashcards Into a Quizlet-Plus Experience WITH AI Accuracy Verification + Creation Limits (Non-Tiered by Scope, Revenue-Protective)

You are my senior full-stack engineer working inside my existing NurseNest codebase (Vite + React frontend, Express backend, existing DB supports flashcards/sets/users). Implement the features below as an EXPANSION of my current flashcard system, not a rebuild.

NON-NEGOTIABLE REQUIREMENTS
1) Non-tiered by scope: Do NOT apply RPN/RN/NP scope logic to user-generated flashcards. Users can create cards on any body system/topic.
2) Revenue-protective: Do NOT replace or undermine existing paid tier features (scope-tiered lessons, exam simulator, AI tutor, etc.). This feature set is for user-generated cards and study tools. Apply gating only where specified (card creation limits + verification volume + advanced exports).
3) Safety-first + trust: AI verification is an accuracy filter. Unsafe cards cannot be publicly searchable/shareable.
4) No long-form teaching: AI output must be concise. No full lessons, no exam coaching.
5) SEO indexing policy: Only AI-Verified PUBLIC sets are indexable and discoverable. Private/unverified/flagged content must not be indexed.

NEW REQUIREMENT — FREE CARD CREATION LIMIT (Upgrade Trigger)
Implement a hard cap on how many flashcards a FREE account can create before requiring upgrade.

Choose a limit that aligns with common freemium study tools where core usage is generous but serious users must upgrade:
- DEFAULT FREE LIMIT: 300 total user-created flashcards per account (across all sets).
- Paid subscribers: unlimited flashcards.

Rationale: Many platforms monetize via limiting premium modes/AI rather than card count, but for NurseNest we need a clear upgrade trigger while still allowing meaningful free use. 300 cards supports several courses worth of studying but pushes heavy users (nursing exam prep) to subscribe.

This limit must be:
- enforced server-side (never trust the client)
- configurable via environment variable, e.g., FREE_FLASHCARD_LIMIT=300
- displayed clearly in UI with a progress indicator (e.g., “214 / 300 cards used”)
- accompanied by an upgrade CTA when the user approaches/ reaches the limit (e.g., at 80% and 100%)
- not applied to “starred/saved” cards from other users (only counts cards the user CREATED)

GOALS

B) Add AI medical accuracy verification layer that prevents misinformation and builds trust.

SECTION 3 — AI ACCURACY VERIFICATION (Must-Have)
Implement an AI verification workflow for user-created flashcards.

When a user creates or edits a flashcard:
1) Backend endpoint verifies content (front/back text + optional image metadata + tags).
2) AI returns ONLY STRUCTURED JSON in this schema:
{
  "status": "VERIFIED"|"PARTIALLY_ACCURATE"|"INACCURATE"|"UNSAFE",
  "confidence": 0-100,
  "issues": [
    { "field":"front"|"back"|"both",
      "type":"FACT_ERROR"|"OUTDATED"|"AMBIGUOUS"|"MISSING_CONTEXT"|"UNSAFE"|"IMAGE_ACCURACY",
      "snippet":"...",
      "explanation_short":"(2-4 sentences max)",
      "suggested_fix":"(short correction, not a lesson)"
    }
  ],
  "corrected_version": { "front":"...", "back":"..." },
  "safety_flags": [
    { "category":"DOSING"|"CONTRAINDICATION"|"EMERGENCY_ACTION"|"CRITICAL_VALUE"|"ACID_BASE"|"OTHER",
      "severity":"HIGH"|"MED"|"LOW",
      "note_short":"..."
    }
  ],
  "publish_eligibility":"PUBLIC_OK"|"PRIVATE_ONLY"|"ADMIN_REVIEW_REQUIRED",
  "disclaimer_short":"One sentence safety disclaimer."
}

Rules:
- If status is UNSAFE or publish_eligibility is ADMIN_REVIEW_REQUIRED => card cannot be public; route to admin queue.
- Store verification results with model name/version and timestamps.
- Show badges:
  "AI Verified", "AI Flagged (Private)", "Not Verified"
- Only VERIFIED + PUBLIC_OK cards may be included in PUBLIC sets that are indexed.

Verification gating (revenue-protective but keep core usable):
- Free: limited verifications per day (configurable 5–10/day). Show status + minimal issue summary only.
- Paid: unlimited verification, full concise fix suggestions, and bulk set verification.

IMAGE VERIFICATION:
- Users can upload images for flashcards (front/back). For paid users, run additional AI image-accuracy check (lightweight). If fails => PRIVATE_ONLY.
- Do NOT embed text in generated images.
