You are implementing strict taxonomy protection for the NurseNest content generation system.

The content generator must NEVER create new categories, topics, menu items, navigation groups, or taxonomy structures automatically.

All generated content must map only to the existing canonical taxonomy used by the production site.

This rule is critical for preventing content structure chaos, SEO dilution, and navigation duplication.

------------------------------------------------
PRIMARY GOAL
------------------------------------------------

Protect the existing content structure while allowing the generator to safely publish new questions, flashcards, and lessons.

Generated content must only attach to existing:

tiers
categories
topics
body systems
exam blueprint sections
navigation groups

------------------------------------------------
STRICT TAXONOMY RULE
------------------------------------------------

The generator is NOT allowed to create:

new categories  
new topics  
new body systems  
new navigation menu items  
new site sections  
new URL slugs  
new content groupings  

If the generator encounters a topic that does not exactly match an existing taxonomy node, it must map it to the closest existing node.

Under no circumstance should the generator automatically insert new taxonomy records.

------------------------------------------------
TOPIC NORMALIZATION SYSTEM
------------------------------------------------

Implement a topic normalization layer before saving generated content.

Example mappings:

"AKI" → Acute Kidney Injury  
"OCD" → Obsessive Compulsive Disorder  
"Heart attack" → Acute Coronary Syndrome  
"High blood pressure" → Hypertension  
"Pregnancy hypertension" → Preeclampsia  

Normalization rules must map variations to the canonical topic slug.

If a synonym appears that does not exist in the taxonomy:

1. attempt synonym matching
2. attempt fuzzy matching
3. fallback to the parent body system category

Example:

If topic "Renal failure management" appears
→ map to Acute Kidney Injury

------------------------------------------------
FAIL-SAFE RULE
------------------------------------------------

If a generated topic cannot be mapped to an existing taxonomy node:

DO NOT CREATE A NEW ONE.

Instead:

fallback to the parent body system category.

Example:

unknown cardiology topic
→ assign to Cardiovascular general category.

Log the event for admin review.

------------------------------------------------

Create a taxonomy review queue for unmapped topics.

confidence score
timestamp

Admins can then manually approve new taxonomy entries later if needed.
