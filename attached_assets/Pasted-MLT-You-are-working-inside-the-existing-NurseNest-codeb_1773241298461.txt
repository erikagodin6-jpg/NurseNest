Build a true adaptive testing engine for the USA MLT / MLS exam-prep section and a separate adaptive practice engine for both the Canada and USA tracks.

-------------------------------------

CORE ENGINE REQUIREMENTS

Build an adaptive engine that selects questions based on:

- prior response correctness
- question difficulty
- content area balance
- cognitive level balance
- exposure control
- weak area prioritization
- recent question history
- minimum blueprint/category coverage

Do not make it random.
Do not just increase difficulty after every correct answer without content balancing.

-------------------------------------

QUESTION DIFFICULTY MODEL

Use difficulty levels:
- easy
- medium
- hard
- very_hard

Internally map these to numeric values for adaptive logic, for example:
easy = 1
medium = 2
hard = 3
very_hard = 4

Create a user ability estimate that updates after each response.

Minimum requirements:
- initialize ability estimate at medium
- move estimate up after correct answers
- move estimate down after incorrect answers
- weight movement more strongly when the question discrimination is high
- weight movement less when the user is guessing repeatedly or response times are abnormally low
- cap movement so the exam does not swing unrealistically after one question

-------------------------------------

QUESTION SELECTION LOGIC

The next question should be chosen using a weighted scoring formula based on:

1. closeness to current ability estimate
2. not previously used in the session
3. target blueprint coverage not yet met
4. target discipline coverage not yet met
5. user weak area priority
6. exposure control so the same highly-ranked item is not always used
7. exam mode rules
8. image question spacing so image-heavy questions do not cluster excessively

Design a function that scores candidate questions and chooses the best available one.

Pseudo priorities:
- realistic CAT exam mode prioritizes psychometric realism and balanced domain coverage
- adaptive practice mode prioritizes remediation and weak area improvement
