Goal

Current problem
- Clicking the MLT test bank button routes to an error page instead of a valid test bank.
- This is likely a broken route, bad slug, missing content mapping, empty dataset, or a page/render failure.

What to do
1. Trace the full MLT test bank button flow:
   - button component
   - href/onClick handler
   - route path
   - slug/ID passed
   - page loader
   - API/data fetch
   - test bank/session initialization
2. Identify the exact failure point.
3. Fix the source issue rather than redirecting around it.
4. Verify the button opens a valid MLT test bank experience end to end.

Things to inspect carefully
- The MLT card/button link target
- Whether the MLT test bank route exists
- Whether the slug/path matches the actual MLT content record
- Whether MLT is incorrectly pointing to a generic or deleted route
- Whether the page exists but the content relation is null
- Whether the content is unpublished
- Whether the test bank has zero questions and crashes on load

- Whether the “Something went wrong” page is coming from a frontend error boundary
- Whether there is a mismatch between allied health program names and stored slugs/IDs
