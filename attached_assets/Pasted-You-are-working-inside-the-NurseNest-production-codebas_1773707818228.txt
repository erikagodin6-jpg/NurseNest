You are working inside the NurseNest production codebase.

Build a complete AI-powered Content Integrity and Auto-Repair System for the production database and published site.

This system must continuously monitor, detect, repair, and prevent content gaps across all current and future tiers without creating database chaos, duplicate menus, duplicate records, broken taxonomy, or disconnected published content.

This is a platform-level quality control and automation feature.
It must integrate directly into the existing NurseNest admin, production database, publishing pipeline, question banks, flashcard system, lessons, blogs, CAT exams, and analytics pages.

DO NOT create a separate standalone app.
DO NOT create a disconnected “AI tool.”

PRIMARY GOALS

1. Automatically detect missing or incomplete content across all content types.
2. Automatically generate and repair missing supporting content where allowed.

SECTION 2 — AUTO-REPAIR RULES
==================================================

Build an AI repair pipeline that can automatically fix eligible content issues.

AUTO-REPAIR ELIGIBLE ITEMS

QUESTIONS
If missing rationale:
Generate a high-quality rationale with:
• why correct answer is correct
• why distractors are incorrect
• concise clinical reasoning
• exam strategy takeaway
• pathophysiology where relevant
• nursing implications where relevant
• scope-appropriate language depending on tier

If missing metadata:
Infer and populate:
• tier
• category
• body system
• concept tags
• exam blueprint tags
• difficulty estimate
• topic/subtopic
• clinical domain

If missing flashcard linkage:
Generate and link a flashcard based on the question.

If formatting is broken:
Normalize formatting and sanitize content safely.

If answer choices are malformed:
Repair safely only if recoverable from source structure.
Otherwise send to manual review.

FLASHCARDS
If missing flashcard for a published question:
Generate one automatically.

If flashcard exists but is weak:
Improve structure while preserving source linkage.

Flashcard format:
Front:
• concise testable concept prompt

Back:
• correct concept
• concise explanation
• key clinical relevance
• exam pearl or memory aid

LESSONS / BLOGS / STUDY GUIDES
If missing metadata:
Generate SEO title, meta description, H1, schema-friendly summary, and internal links.

If content is thin:
Flag for expansion queue.
Optionally generate draft expansion sections but do not overwrite strong human-authored content blindly.

If medically reviewed section missing:
Insert:
“Medically reviewed by a Registered Nurse”
using the existing site’s reviewer pattern and data model.

If references missing where required:
Generate a references draft field or queue for editor review depending on confidence.
