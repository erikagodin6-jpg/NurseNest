====================================================
PART 1 — GLOBAL MINIMUM WORD COUNT ENFORCEMENT
====================================================

Implement word count validation at publish level for all content types.

SEO PILLAR PAGES:
- Minimum 2,500 words
- Recommended target: 3,000–4,000
- Minimum 12 FAQs
- Minimum 10 internal links
- Block indexable=true if below threshold.

CLUSTER PAGES:
- Minimum 1,200 words
- Minimum 6 FAQs
- Minimum 6 internal links
- Block indexable=true if below threshold.

BLOG POSTS:
- Minimum 1,000 words (unless marked short_update=true)
- Minimum 3 internal links.

LESSONS:
- Minimum 1,500 words (core lessons)
- Advanced/NP lessons: minimum 2,000 words
- Must include:
   objectives
   red flags
   mini quiz
   FAQ block (min 5)

GLOSSARY:
- Minimum 150 words per term
- Must include related terms.

QUESTION RATIONALES:
- MCQ: minimum 120 words
- SATA: minimum 150 words
- Case-based: minimum 200 words
- Case summary: minimum 300 words

If below threshold:
- Publishing blocked.
- QC error displayed.

PART 2 — AI PRODUCTION CONTROL SYSTEM
====================================================

- Explicit manual trigger only.
- Show estimated token usage before running.
- Monthly token budget limit.
- Hard stop if budget exceeded.
- Queue-based job system.
- Never auto-generate entire libraries without confirmation.

When AI generates:
- Automatically run QC:
   - word count check
   - duplicate similarity scan
   - internal link suggestion
   - blueprint tag check

   - mark as draft.
