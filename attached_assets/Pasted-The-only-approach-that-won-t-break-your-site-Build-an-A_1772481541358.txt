Paste-ready “Master Prompt” for Replit AI (tailored to your reality)
Copy/paste this exactly:
You are a senior full-stack engineer. Add an Automation Center to my existing education web app (NurseNest-style) WITHOUT breaking current paid-wall logic, Stripe checkout, or existing question bank rendering.

GOAL
Implement a safe automation framework that can generate and schedule drafts for question banks, mock exams, SEO pages, and study plans—then route everything through validation + admin review before publishing.

NON-NEGOTIABLES
1) Draft-first: all generated assets must be saved as DRAFT. Nothing auto-publishes unless (a) it passes validation AND (b) auto-publish is enabled for that automation.
2) Do not change existing paid-wall logic, pricing, Stripe, or current question display. Add new endpoints and admin pages only.
3) Add global kill switch: pause all jobs instantly.
4) Add cost caps: max jobs/day, max items/run, max tokens/run (approx), retry limits.
5) Add audit logs: every run logs inputs, outputs, validation results, status, timestamps.
6) Add RBAC: admin can approve/publish; editor can edit drafts; viewer read-only.
7) Add idempotency: prevent duplicates when jobs rerun.

ARCHITECTURE
- Create Job Queue + Scheduler:
  - Prefer BullMQ + Redis if available; if not, implement a DB-backed queue.
  - Cron runner enqueues scheduled jobs every 5 minutes.
- Create tables:
  Automations, AutomationRuns, Jobs, DraftAssets, PublishedAssets, ValidationReports, PromptTemplates, TopicTaxonomy
- Implement Admin UI:
  - Automation list (enable/disable, schedule, scope, run-now)
  - Draft Review Queue (edit + approve/publish)
  - Logs/Validation reports
  - Metrics overview

VALIDATION (must exist before publishing)
- Schema validation: JSON output matches required fields
- Duplicate detection: stem similarity check within same topic/tier
- Format rules: one best answer, plausible distractors, no “all of the above”, no cueing
- Country/tier tagging: each item must store country + tier metadata
- Content safety: must be educational, no personalized medical advice

IMPLEMENT FIRST (end-to-end)
A) Daily Question Pack automation:
- Input: country, tier, exam, topic, count, rationale_min_words
- Output: DraftAssets (question items) + ValidationReport
- Admin can approve/publish batch
B) Weekly Mock Exam Builder:
- Assembles exam from bank + injects a % new items
C) Daily SEO Blog Draft automation:
- Drafts only. Includes internal link suggestions and CTA blocks

PROMPT ENGINE
- Store prompt templates in DB with placeholders:
  {{country}} {{tier}} {{exam}} {{topic}} {{count}} {{rationale_min_words}}
- Admin can edit prompt templates.

Now build the DB schema + queue + scheduler + admin UI + the 3 automations above. Keep changes incremental and avoid touching existing payment/tier gating.

The guardrails that will save you from “AI messed up my site”
In your Admin Automation Center, every automation must have:
Max items/run
Max runs/day
Auto-publish toggle (default OFF)
Rationale min words
Difficulty distribution
Topic scope (checkbox list)
Country/tier scope
Kill switch (global)
And in validation:
Reject if duplicate similarity > threshold
Reject if missing fields
Reject if rationale too short
Reject if unsafe language (“you should take…”, “treat by…”)
