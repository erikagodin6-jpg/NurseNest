Redesign the NurseNest Flashcards landing page so it becomes a premium study launcher with built-in paywall conversion logic.

IMPORTANT
The flashcards are behind a paywall for full access, but the page should still feel like a real study tool, not a generic pricing page.

The goal is to maximize:
- trust
- product understanding
- free-to-paid conversion
- study session starts
- premium perceived value

Do not build a landing page that is only marketing text.
Build a page that previews the flashcard experience and encourages users to start a free preview or unlock full access.

CORE POSITIONING

NurseNest flashcards are:
- exam-style clinical questions
- tier specific (RPN, RN, NP)
- untimed
- rationale-based
- adaptive to weak areas
- connected to lessons

This should be obvious within the first screen.

PAGE STRATEGY

This page must act as:
1. product explanation
2. study launcher
3. premium preview
4. paywall conversion page

VISUAL STYLE

Use premium NurseNest styling:
- pastel lavender
- mint
- blush
- soft clinical blue
- rounded cards
- soft shadows
- elegant spacing
- modern clinical learning platform feel

The page should feel calm, premium, and trustworthy.

PAYWALL UX REQUIREMENTS

Build the paywall so it converts well.

Do NOT:
- hard lock everything before the user understands the product
- send users straight to pricing without previewing value
- hide what makes the product different

DO:
- allow free preview/sample cards
- show premium features clearly
- let users experience the interface before upgrading
- upsell after value is demonstrated

GATING LOGIC

If user is free:
- allow limited preview cards
- limit advanced modes
- show upgrade prompts after sample usage
- allow topic browsing but not full launch

If user is paid:
- full flashcard launcher available
- full topic access
- full adaptive modes
- full rationale system
- save progress normally

If user is not logged in:
- allow limited preview
- encourage sign-up before full launch
- preserve selected tier/topic if possible

- minimal clutter
