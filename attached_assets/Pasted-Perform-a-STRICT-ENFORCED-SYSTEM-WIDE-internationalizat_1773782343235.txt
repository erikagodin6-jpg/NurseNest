Perform a **STRICT, ENFORCED, SYSTEM-WIDE internationalization (i18n) overhaul** across the entire NurseNest platform.

This is a **non-optional, blocking requirement**. The system must NOT ship, render, or publish content that is not fully localized.

This applies to:

* all pages (public + gated)
* all lessons
* all question banks
* all CAT exams
* all flashcards
* all blogs
* all career guides
* all new grad content
* all allied health content
* all dashboards
* all admin panels
* all modals, tooltips, forms, and notifications
* all SEO metadata
* all dynamic AI-generated content
* all current and future languages

---

## CRITICAL ENFORCEMENT RULE

If any content is missing translation:

→ It MUST NOT render
→ It MUST NOT publish
→ It MUST surface an error or fallback UI (not English leakage)

---

## CORE PROBLEM

Mixed-language rendering is occurring due to:

* hardcoded English strings
* missing translation keys
* incomplete locale files
* AI-generated content not localized
* DB content not language-aware
* silent fallback behavior

---

## HARD REQUIREMENTS (NON-NEGOTIABLE)

1. ZERO HARDCODED TEXT POLICY

* ANY user-facing string found in code is a failure
* Build a scanner to detect:

  * raw strings in JSX/TSX
  * strings in arrays/configs/constants
* Block build if violations exist

2. TRANSLATION KEY ENFORCEMENT

* ALL text MUST use translation keys
* Missing key = runtime error (NOT silent fallback)
* Log and track all missing keys

3. FULL DATABASE MULTI-LANGUAGE STRUCTURE

* ALL content types must support:
  {
  en: {...},
  fr: {...},
  es: {...}
  }
* OR a translation mapping layer

Applies to:

* lessons
* questions
* rationales (CRITICAL)
* CAT exam nodes
* flashcards
* blog content

4. AI GENERATION ENFORCEMENT

* AI content MUST:
  EITHER
  → be generated in the selected language
  OR
  → be immediately translated before storage

* It is NOT acceptable to:
  generate in English and display in French

* Add validation:
  → detect language of generated content
  → reject if mismatch

5. LANGUAGE VALIDATION LAYER
   Implement automatic detection:

* Run language detection on:

  * questions
  * rationales
  * lessons
  * UI strings
* If mismatch:
  → flag content
  → block rendering OR mark for admin correction

6. NO MIXED LANGUAGE RENDERING (STRICT)

* A single page must NEVER contain multiple languages
* If mismatch detected:
  → fail component render OR show fallback state

7. ADMIN TRANSLATION DASHBOARD (REQUIRED)
   Admin must see:

* missing translations per content item
* % translation completion
* language coverage gaps
* flagged mixed-language content

Must include:

* filters by language
* filters by content type
* quick edit capability

8. FALLBACK LOGIC (CONTROLLED)

* Fallback to English ONLY IF:
  → explicitly allowed
  → logged
  → visible in admin

* NEVER silently fallback

9. BUILD-TIME VALIDATION

* Add checks that:

  * scan for missing translation keys
  * scan for hardcoded text
* FAIL BUILD if:

  * untranslated keys exist
  * locale files are incomplete

10. RUNTIME VALIDATION

* Detect:

  * missing keys
  * wrong-language content
* Log to:

  * console
  * monitoring system

11. CONSISTENT TERMINOLOGY ENGINE

* Central dictionary for:

  * nursing terms
  * exam terminology
* Enforce consistency across:

  * lessons
  * questions
  * rationales
  * AI outputs

12. CONFIG-DRIVEN CONTENT FIX

* ANY config arrays (cards, features, chips):
  → must use translation keys
  → NOT raw text
