CRITICAL IMPLEMENTATION RULE FOR NURSENEST AI OPS

Replit AI must ONLY be used during the implementation/build phase to help write code.

Replit AI must NEVER be used as the runtime AI engine for the website after the system is built.

All runtime AI generation (questions, flashcards, lessons, blog drafts, admin copilot actions, SEO fixes, etc.) must run through a configurable AI provider router built inside the NurseNest backend.

The runtime system must NOT call any Replit AI or Replit Agent APIs.

Instead, implement a provider abstraction layer called:

AIProviderRouter

This router must support the following providers:

1. Ollama (local/self-hosted)
2. vLLM (OpenAI-compatible inference server)
3. LM Studio (local server)
4. OpenAI-compatible providers (optional)
5. Anthropic-compatible providers (optional)

The router must allow switching providers without modifying business logic.

The default provider must be a low-cost local/self-hosted model.

Premium providers must be disabled by default.

LOCAL MODEL FIRST POLICY

The runtime system must follow this policy:

1. Attempt generation using the configured local provider
2. Validate output using schema validation
3. If validation fails:
      retry with the same provider
4. If retry fails and fallback is enabled:
      use fallback provider
5. If fallback disabled:
      mark job as NEEDS_REVIEW

Do not automatically call premium providers unless explicitly enabled in admin settings.
