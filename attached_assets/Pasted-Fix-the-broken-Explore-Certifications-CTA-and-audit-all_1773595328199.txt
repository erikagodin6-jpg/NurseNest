Fix the broken “Explore Certifications” CTA and audit all New Grad certification routing so users are not redirected to the wrong page. Right now when a user is on the New Grad Certifications page and clicks “Explore Certifications,” it sends them to the main exam-prep homepage instead of the intended New Grad certifications content flow. This is a routing, navigation, and CTA integrity bug and needs a full audit.

PRIORITY

Treat this as a high-priority UX and conversion bug.

Current problem:
- User is on the New Grad Certifications page
- They click “Explore Certifications”
- They are sent to the main exam prep homepage instead of deeper certification content
- This breaks user intent, feels sloppy, and interrupts the New Grad journey

OBJECTIVE

Make sure the New Grad Certifications page keeps users inside the New Grad certification experience.

The “Explore Certifications” button should never redirect users to the unrelated main hero/home page unless that is explicitly intended, which it is not here.

Fix the root cause and audit all similar CTA and internal-link issues in the New Grad section.

NON-NEGOTIABLE RULES

- Do not break the existing nursing tier menus that already work.
- Do not redirect New Grad certification CTAs to the main exam-prep homepage unless explicitly designed as a fallback.
- Do not leave any New Grad CTA pointing to a generic catch-all page when a section-specific destination exists.
- Do not patch only one button if the same problem exists elsewhere.
- Do not change the content theme of the New Grad page; fix the route and flow.
- Do not leave users in a mixed navigation state between New Grad and main exam-prep pages.

BUG TO FIX

Source page:
New Grad Hub > Certifications

Current CTA:
“Explore Certifications”

Current incorrect result:
- lands on the main page with:
  “Pass Your Nursing or Allied Health Exam With Confidence”
- shows the general exam-prep site shell and main homepage hero
- breaks the New Grad certifications journey

Expected result:
The “Explore Certifications” CTA should take the user to one of the following, depending on current architecture:
- a proper New Grad certifications index page
- an anchor/jump link further down the same certifications page
- a filtered certifications listing inside New Grad
- a structured certification category grid
- or the first intended certification resource section within New Grad

It should not route to the main homepage.

Inspect:
- href / to value
- route helper
- button component props
- page-level override
- CTA config source

Check if it currently points to:
- "/"
- main marketing page
- generic exam-prep landing page
- wrong slug
- missing route that falls back to home
- stale redirect from previous implementation

Based on the New Grad Certifications page structure, the button should likely do one of these:
- scroll to the certification cards/list lower on the same page
- route to a dedicated New Grad certifications overview page
- route to certification-specific subpages
- open a filtered certification hub such as ACLS / BLS / PALS / TNCC / NRP / CEN / CCRN content

Choose the correct destination based on the actual information architecture already being built for New Grad.

Preferred behavior:
If the page already contains certification sections below the hero, make the button scroll to that section.
If there is a dedicated certifications index route, send it there.
If individual certification pages already exist, send users into the certification catalog page first.

- button href hardcoded to home
- missing target route causing redirect to homepage
- route mismatch between slug and actual page
- layout wrapper using wrong route helper
- button component defaulting to root when link is undefined
- CMS field empty so component falls back to homepage
- client-side router failing and sending user to the marketing root
- page generated from template but CTA config not updated

Check these specifically.
