Fix the **tier isolation problem** on the NurseNest production site.

Problem:
The **RN page is currently displaying RPN and NP lessons**, which should never happen. Each tier must only display its own curriculum. This issue must be fixed at the **data filtering / routing / query level**, not just hidden in the UI.

Non-negotiable:
All fixes must apply to the **live production site and production database**, not only the Replit preview or development environment.

Primary goal:
Ensure that **each tier page only loads and displays content that belongs to that tier**.

Affected tiers include (at minimum):

• RPN / LVN
• RN / NCLEX
• NP Advanced

These tiers must remain **completely isolated in content queries**.

Step 1 — Audit the lesson data structure

Inspect the lesson database schema and confirm there is a **tier field or category identifier** such as:

tier
examTier
program
level

Verify that every lesson record correctly identifies its tier.

Examples:

RPN lessons → tier = "rpn"
RN lessons → tier = "rn"
NP lessons → tier = "np"

If the field does not exist or is inconsistent, create a **strict tier field** and normalize existing data.

Step 2 — Fix the lesson queries

Audit the code responsible for loading lessons on:

• tier landing pages
• lesson libraries
• lesson search results
• related lessons sections
• lesson navigation menus

Ensure all queries filter by tier.

Correct behavior example:

RN page should query:

WHERE tier = "rn"

NOT:

SELECT * FROM lessons

OR

SELECT lessons WHERE published = true

Every lesson fetch must explicitly filter by tier.

Step 3 — Fix shared components

Audit shared UI components that display lessons:

• lesson cards
• lesson grids
• lesson carousels
• featured lessons
• “related lessons” modules
• sidebar lesson lists

Ensure these components receive the **current tier as a required parameter** and filter results accordingly.

Shared components must **never default to showing all lessons across tiers**.

Step 4 — Fix routing logic

Verify routing for:

/rpn
/rn
/np

Each route must set a **tier context** so the application knows which content to load.

Example logic:

route "/rn" → tierContext = "rn"
route "/rpn" → tierContext = "rpn"
route "/np" → tierContext = "np"

All lesson queries on the page should reference that tier context.

Step 5 — Fix lesson linking

Ensure:

• RN lessons link only to RN content
• RPN lessons link only to RPN content
• NP lessons link only to NP content

Audit internal links that might be pulling lessons by **topic name instead of tier + topic**.

Step 6 — Fix admin dashboard

Update the admin content manager so lessons cannot accidentally appear in multiple tiers.

Admin panel should show:

Lesson title
Tier
Category
Published status

Add validation rules:

A lesson **must belong to exactly one tier** unless explicitly flagged as cross-tier.

Step 7 — Prevent future cross-tier leakage

Implement safeguards:

• tier validation when saving lessons
• tier validation in queries
• automated check that prevents mixed-tier lesson lists

Step 8 — Verify on the live site

After fixing:

Check the following pages on the **live production site**:

RN lessons page
RPN lessons page
NP lessons page

Confirm:

• RN page only shows RN lessons
• RPN page only shows RPN lessons
• NP page only shows NP lessons

Step 9 — Reporting

When complete, output a report showing:

• number of lessons per tier
• any lessons missing a tier tag
• any lessons that previously appeared on incorrect pages
• confirmation that tier filtering is now enforced in queries
