Audit and fix all multilingual sitemap files under /sitemaps/ because many language sitemaps are showing “Couldn’t fetch” and “Unknown” in Google Search Console.

Problem
Some language sitemap URLs are working, but many are not. Examples currently failing include:
- sitemap-lang-id.xml
- sitemap-lang-tr.xml
- sitemap-lang-pt.xml
- sitemap-lang-fa.xml
- sitemap-lang-ja.xml
- sitemap-lang-ur.xml
- sitemap-lang-ht.xml
- sitemap-lang-vi.xml

Meanwhile some others like:
- sitemap-lang-th.xml
- sitemap-lang-de.xml
are working correctly.

Goal
Make every submitted language sitemap URL return a valid XML sitemap that Google can fetch successfully.

What to check
1. Verify each sitemap file actually exists at its public deployed URL.
2. Verify each sitemap route returns:
   - HTTP 200
   - valid XML
   - correct sitemap structure
   - correct Content-Type such as application/xml or text/xml
3. Check whether failed sitemap routes are returning:
   - 404 pages
   - HTML fallback pages
   - app shell index.html
   - malformed XML
   - server errors
   - empty output
4. Compare a working sitemap (such as th or de) against a failing sitemap and identify the exact difference in generation or deployment.
5. Check whether locale-specific slug generation is failing for certain languages.
6. Check whether the sitemap builder is skipping languages with no URLs instead of generating a valid sitemap.
7. Check whether filenames, route registration, or deployment output paths are inconsistent.

Required fix
- Every language sitemap URL must return a valid XML sitemap.
- If a locale has no pages, still return a valid minimal sitemap or remove that sitemap from submission generation.
- Do not leave broken sitemap URLs in the sitemap index or Search Console workflow.
- Ensure all language sitemap files are publicly accessible after deployment.

Validation requirements
For every language sitemap:
- open directly in browser
- confirm valid XML
- confirm HTTP 200
- confirm Google can fetch it
- confirm it is not returning HTML
- confirm it includes the correct localized URLs only

Architecture rules
- Do not break the currently working sitemap files
- Do not change the main navigation
- Do not remove valid localized pages
- Do not generate fake URLs just to fill the sitemap

Also check
- sitemap index file references
- robots.txt references
- locale route generation
- deployment/public output path for /sitemaps/
- server headers for xml files

Acceptance criteria
- All live sitemap-lang-*.xml files either work correctly or are removed from submission/index generation
- Google Search Console recognizes them as real sitemaps instead of “Unknown”
- No “Couldn’t fetch” remains for submitted language sitemap URLs
