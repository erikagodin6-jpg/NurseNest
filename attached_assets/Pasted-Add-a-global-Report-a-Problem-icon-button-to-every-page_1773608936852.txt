Add a global “Report a Problem” icon/button to every page of the published NurseNest site.

Goal
- Users should be able to quickly report a broken link, content issue, wrong answer, typo, layout bug, empty lesson, exam issue, or other page-specific problem from any page on the site.
- This must appear on every page across:
  - main site
  - RPN
  - RN
  - NP
  - allied health
  - new grad
  - lessons
  - flashcards
  - test banks
  - CAT/exams
  - store/product pages
  - blog/content pages

Core UX requirements
- Add a small, clean, non-intrusive floating icon/button that stays visible but does not block content.
- Best placement:
  - bottom right on desktop
  - bottom right or slightly raised on mobile so it does not overlap sticky controls/tab bars
- Use a simple bug/flag/chat-warning style icon with label:
  - “Report a Problem”
- It should match the NurseNest aesthetic:
  - soft, clean, modern, rounded
  - consistent with existing theme colors
  - professional, not harsh or clunky

Behavior
- When clicked, open a modal, drawer, or popover form.
- The form should automatically capture page context so the user does not need to explain where they are.

Automatically capture and include
- current page URL
- page title
- tier/program/site section
- lesson/test bank/exam/content ID if available
- user ID if logged in
- timestamp
- device type if available
- browser info if available
- locale/language if available

Form fields
Required:
- problem type dropdown
- short description textarea

Helpful optional fields:
- email or account email autofill if available
- screenshot upload if supported
- severity dropdown
- can we contact you checkbox

Problem type dropdown options
- Broken link / 404
- Empty lesson / missing content
- Wrong answer / rationale issue
- Typo / grammar
- Page layout problem
- Exam won’t start
- Flashcards missing / broken
- Payment / subscription issue
- Translation issue
- Other

Desired UX details
- Keep the form short and easy.
- Allow users to submit in 2–3 taps/clicks.
- Show success confirmation:
  - “Thanks — your report has been sent.”
- Do not navigate users away from the page after submission.
- Preserve what they were doing.

Admin / backend requirements
- Save every report to a database table or admin-visible issue collection.
- Reports should be viewable in the admin dashboard.
- Include status fields such as:
  - new
  - in review
  - fixed
  - dismissed
- Allow filtering by:
  - page type
  - tier
  - problem type
  - date
- Allow admin notes on each report.

Notification requirements
- Send the report to the configured admin support email or internal notification channel if that already exists.
- If no notification system exists, at minimum store the reports reliably in the admin dashboard/database.

Technical requirements
- Implement this from the shared layout/app shell level so it appears globally across all pages.
- Do not manually add it page by page if a shared component can handle it.
- Do not break current navigation, sticky footers, exam UI, or mobile bottom bars.
- Make sure it does not overlap critical buttons like:
  - next question
  - reveal rationale
  - add to cart
  - checkout
  - sticky nav
  - mobile menu

Page-specific intelligence
If the user is on:
- a lesson page: attach lesson metadata
- a flashcard page: attach deck/card metadata
- a test bank question page: attach question/session/bank metadata
- a CAT exam page: attach exam/session/question metadata
- a product/store page: attach product ID/slug
- a blog page: attach article slug
- an allied health page: attach allied program slug

Nice-to-have enhancements
- prefill the issue category intelligently based on page context if possible
- include a “take screenshot” helper if browser-supported
- include a small “Report content issue” inline link on rationale/lesson pages if easy, but only after the global button works

Accessibility
- keyboard accessible
- screen-reader label
- sufficient contrast
- large enough tap target on mobile

Analytics
- track:
  - icon clicked
  - report submitted
  - problem type selected
  - page section
This should feed analytics if your current analytics system supports custom events.

Do not
- do not create a separate ugly standalone page unless needed as fallback
- do not require the user to log in just to report a problem if guest submission is possible
- do not make the modal long or intimidating
- do not hide useful page metadata from admins

Acceptance criteria
- A visible “Report a Problem” icon/button appears on every page
- Clicking it opens a working form
- The form auto-attaches page context
- Reports are stored and accessible in admin
- The button does not interfere with page functionality
- Works on desktop and mobile
- Looks cohesive with NurseNest branding
