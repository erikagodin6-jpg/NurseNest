Implement an Admin-only “Social Scheduler” feature in my NurseNest Next.js + Tailwind app that automatically posts original nursing exam-style questions to a Facebook Page.

Goal
- Post 1–3 times per day to a Facebook Page I own
- Drive traffic back to my site (Question of the Day landing pages)
- Track performance and avoid duplicates

Non-negotiables
- All questions must be original (no copying or reworded clones from commercial banks)
- Do not claim “official NCLEX scoring” or that this replicates the real exam
- Maintain NurseNest pastel clinical aesthetic in the admin UI
- Mobile-friendly admin UI
- Secrets must never be exposed to the browser

Platforms
- Facebook Page posting via Meta Graph API:
  - Publish to Page feed using /{page-id}/feed
  - Handle errors and rate limiting gracefully

Permissions/Access (implementation notes)
- Use a Page Access Token and ensure required permissions for posting are handled (e.g., pages_manage_posts and pages_read_engagement) as applicable.
- Document the setup steps in comments.

Admin routes
- /admin/social (overview dashboard)
- /admin/social/queue (create/edit scheduled posts)
- /admin/social/logs (posting history)

Database tables
1) social_accounts
- id, platform ("facebook"), page_id, page_name
- access_token_encrypted, token_expires_at
- created_at, updated_at

2) social_post_queue
- id, platform, account_id
- jurisdiction ("CA"|"US"|"NEUTRAL")
- tier ("PRACTICAL_NURSE"|"RN"|"NP")
- post_type ("question_of_day"|"blog_share"|"announcement")
- scheduled_for (timestamp)
- status ("queued"|"posted"|"failed"|"skipped")
- post_text (string)
- link_url (string)
- image_url (nullable)
- dedupe_hash (unique)
- created_at, updated_at

3) social_post_logs
- id, queue_id
- attempted_at
- status ("success"|"fail")
- response_payload (json)
- error_message (text nullable)

Scheduler runner
- Create a server-side endpoint: POST /api/admin/social/run
- Protect it with a secret token header or query param:
  - env SOCIAL_CRON_SECRET
- The runner should:
  - find all queued posts where scheduled_for <= now
  - post them in order
  - update statuses and log responses
  - retry transient errors with exponential backoff
  - if rate-limited, stop and reschedule remaining items

Tracking
- Automatically append UTMs to link_url:
  utm_source=facebook&utm_medium=social&utm_campaign=qotd&utm_content={tier}_{jurisdiction}

Return
- Exact file-by-file code changes
- Migrations/schema
- Admin UI components
- API routes + scheduler logic
- Clear env vars required:
  FACEBOOK_PAGE_ID
  FACEBOOK_PAGE_ACCESS_TOKEN
SOCIAL_CRON_SECRET
