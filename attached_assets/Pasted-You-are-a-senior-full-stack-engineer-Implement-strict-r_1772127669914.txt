You are a senior full-stack engineer. Implement strict region-aware content separation for NurseNest with two domains:
- nursenest.ca (Canada)
- nursenest.us (USA)

GOAL
Stop cross-domain “wording updates” from overwriting each other. Region-sensitive exam wording and lab units must be locked. However, Canadian students may take American NP-related exams/content, so NP tier content can be shared across both domains.

NON-NEGOTIABLE RULES
1) Region is determined ONLY by domain/hostname server-side:
   - .ca => region = "CA"
   - .us => region = "US"
   Do not allow region selection in the client.

2) Every piece of content must be scoped by BOTH:
   - tier: "PRENURSING" | "RPN" | "RN" | "NP"
   - audience/visibility: which region(s) may view it

3) Hard-lock behavior:
   - RPN tier: show ONLY CA version on .ca and ONLY US version on .us (no cross-over)
   - RN tier: show ONLY CA version on .ca and ONLY US version on .us (no cross-over)
   - NP tier: MAY be shared across CA and US (allowed on both)
   - PRENURSING: default shared across both unless explicitly region-specific

4) Exam references/wording must be region-specific for RPN/RN tiers:
   - CA site uses Canadian terminology/spelling/guidelines where relevant
   - US site uses American terminology/spelling/guidelines where relevant

5) Lab values MUST be region-specific unless explicitly marked “shared”:
   - CA: Canadian units/ranges conventions
   - US: American units/ranges conventions
   Implement separate JSON or DB tables, not “AI rewritten text”.

DATA MODEL (Implement this exactly)
Add fields to content entities (lessons, flashcards, exams, question banks, OSCEs, lab references, and any “templates”):
- region_scope: "CA_ONLY" | "US_ONLY" | "BOTH"
- tier: "PRENURSING" | "RPN" | "RN" | "NP"
- version_key: string (e.g. "cardiac_assessment_intro") to link CA/US variants of the same page
- updated_by_ai: boolean
- protected_fields: string[] (fields AI must not alter; include lab units/ranges and exam names)

ENFORCEMENT (Server-side ONLY)
Create a single function used by ALL content queries:
getEffectiveRegion(hostname) => "CA" | "US"
getAllowedRegionScopes(tier, region):
  if tier in ("RPN","RN"):
    return region == "CA" ? ["CA_ONLY"] : ["US_ONLY"]
  if tier == "NP":
    return ["BOTH", region == "CA" ? "CA_ONLY" : "US_ONLY"]  (allow shared + region-specific NP)
  if tier == "PRENURSING":
    return ["BOTH", region == "CA" ? "CA_ONLY" : "US_ONLY"]

All reads MUST filter by:
- tier
- region_scope IN allowedRegionScopes

ADMIN UX REQUIREMENTS (Prevent accidental cross-editing)
1) In the admin editor, show the current DOMAIN region banner (CA/US) and the tier being edited.
2) When creating new content:
   - Default region_scope is derived from tier and current domain:
     - if tier in (RPN,RN): default to CA_ONLY on .ca, US_ONLY on .us (cannot choose BOTH)
     - if tier == NP: default BOTH but allow CA_ONLY / US_ONLY
     - if tier == PRENURSING: default BOTH but allow CA_ONLY / US_ONLY
3) Disable changing region_scope after publish for RPN/RN items (to prevent leaking).
4) Provide “Create US Variant / Create CA Variant” button using version_key to duplicate and localize without overwriting the original.

AI SAFETY GUARDRAILS (Hard Stop)
Implement a middleware/validator in any “AI update content” endpoint:
- Determine region from hostname.
- For RPN/RN:
   - Reject updates if the item’s region_scope does not match the domain’s allowed scope.
   - Reject updates that attempt to modify protected_fields (exam names, lab values, units, ranges).
- Log rejected attempts with reason.

FILE/FOLDER SEPARATION (If file-based content exists)
Separate region-sensitive constants:
- /constants/ca/exams.ts
- /constants/us/exams.ts
- /constants/ca/labs.json
- /constants/us/labs.json
Never import a mixed file.

TESTS (Must add)
Add automated tests to confirm:
1) RPN/RN content never appears cross-domain (CA item never on .us; US item never on .ca).
2) NP BOTH content appears on both domains.
3) Lab values loaded are region-correct.
4) Admin create defaults are correct per tier/domain.
5) AI update endpoint refuses cross-region edits and protected field edits.

DELIVERABLES
- Code changes implementing region detection middleware, unified query filtering, admin enforcement, variant creation, and AI update validation.
- A short README explaining how region_scope + tier works, and how to add a CA/US variant properly.

Do not simplify this by using client-side conditional wording. The fix must be structural and server-enforced.
