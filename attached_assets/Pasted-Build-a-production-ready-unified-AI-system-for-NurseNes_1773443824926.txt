Build a production-ready unified AI system for NurseNest called the NurseNest AI Operations Engine.

PRIMARY GOAL
Create a scalable, low-cost, admin-controlled AI system that can:
1. Generate premium-quality nursing, allied health, specialty, and new-grad educational content
2. Generate exam-style questions, realistic CAT/adaptive-style question pools, flashcards, lessons, rationales, clinical pearls, SEO articles, and image suggestions
3. Help make small website/content/SEO/admin fixes from the admin dashboard using a low-cost AI copilot
4. Protect content quality, medical safety, tier accuracy, specialty accuracy, review requirements, and production database integrity
5. Reduce reliance on expensive premium API calls by prioritizing local or low-cost AI providers first

NON-NEGOTIABLE OUTCOMES
- Do not guarantee medical accuracy
- Do build the system to maximize accuracy, verifiability, consistency, and reviewability
- Do not fabricate citations or sources
- Do not mix tiers or scopes
- Do not auto-publish unreviewed content
- Do not silently write to the wrong database
- Do not allow small admin AI fixes to directly break production
- Do create a hybrid, structured, review-based AI pipeline suitable for NurseNest

SECTION 5 — QUESTION GENERATION STANDARDS
==================================================

Questions must feel like premium exam-prep content.

Every generated question must be:
- clinically realistic
- educationally useful
- grammatically polished
- professionally written
- scope-appropriate
- tier-appropriate
- specialty-appropriate
- varied in style
- non-repetitive
- free of obvious distractors
- free of trivia-only bloat
- aligned to real clinical reasoning where applicable

SUPPORTED QUESTION FORMATS
Support a balanced mix of:
- single-best-answer
- scenario-based
- prioritization
- delegation
- patient teaching
- assessment-first
- safety/risk reduction
- therapeutic communication
- next-best-action
- select-all-that-apply where appropriate
- cue recognition
- condition differentiation
- emergency/escalation
- clinical judgment
- unfolding case items where applicable
- NGN-style / adaptive-style items where supported by track/page/exam type

QUESTION RULES
- Avoid repetitive wording patterns
- Avoid obvious answer cues
- Avoid implausible distractors
- Avoid repetitive correct-answer positions
- Avoid trivial recall overload
- Keep stems readable and appropriately sized
- Use realistic patient scenarios
- Difficult questions should reflect reasoning complexity, not obscure trivia
- Easier questions should still be useful and clinically sound

QUESTION LENGTH CONTROLS
Create configurable validators for:
- short factual questions
- standard case questions
- advanced case questions

Reject:
- bloated stems
- underdeveloped stems
- unrealistic overlong narratives
- overly vague one-liners that teach nothing

RATIONALE RULES
Each question must support:
- rationale for correct answer
- explanation for incorrect answers except where a CAT/adaptive mode explicitly excludes this
- concise educational teaching value
- clinical pearl when appropriate
- linked learning objective
- optional linked lesson slug
- image suggestion metadata when appropriate

Rationales must:
- explain why the correct answer is correct
- explain why the others are wrong unless excluded by mode
- remain concise and high-yield
- not become textbook chapters
- not be generic filler
- not contradict the stem
