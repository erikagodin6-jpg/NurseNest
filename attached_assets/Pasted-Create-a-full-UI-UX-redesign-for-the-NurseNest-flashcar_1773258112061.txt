1. SINGLE QUESTION PAGE LAYOUT

Create a premium question page layout with this structure:

A. Top study header
At the top of the page include:
- exam mode or study mode label
- topic/category badge
- question number out of total
- subtle progress bar
- optional timer area if supported
- bookmark/flag icon
- report issue button if it exists
- clean, balanced spacing

The header should feel compact but premium. It should not dominate the screen. It should stay readable on mobile.

B. Main question content area
Below the header create a main content card:
- large question stem area
- optional clinical vignette/case intro block if the question has one
- supportive formatting for paragraphs, lists, lab values, tables, or lead-in sentences
- excellent line height and padding for readability
- if a question has an associated image, chart, ECG, diagram, or illustration, place it in a dedicated media section inside the question card, not awkwardly floating

The question stem should feel easy to read and not intimidating.

C. Answer choices
Below the question stem show answer choices as polished interactive answer cards:
- each answer choice should be its own card/button row
- clearly separated with proper spacing
- hover, focus, selected, correct, and incorrect states should all be distinct
- include the answer letter/label in a clean pill or circle
- avoid overly bright colors
- use elegant visual states that still communicate clearly

Support:
- single best answer
- select all that apply
- ordered response if your system has it
- bowtie or advanced question types if applicable
- image-based answer choices if they exist

Selected state should look intentional.
Correct/incorrect states after submission should feel premium and educational, not childish or harsh.

D. Right-side panel on desktop or stacked panel on mobile
On desktop:
- create a right-side support column or panel area for question tools
- include topic tags, difficulty, notes, saved status, quick resources, or related concepts if supported
- keep it clean and secondary, never distracting

On mobile:
- stack this content beneath the main question content or move essential actions into a sticky bottom action bar

E. Bottom question action bar
At the bottom of the question view include polished actions:
- submit answer
- next question
- previous question
- save/bookmark
- flag
- reveal rationale after answering if needed
- optional “end quiz” or “back to bank” if relevant

On desktop this can sit below the answer card list.
On mobile use a sticky bottom action bar if that improves usability.

2. RATIONALE PANEL LAYOUT

This is a major priority.
The rationale should be transformed into a premium educational explanation panel that feels rich, organized, and genuinely useful.

After answering, show a structured rationale card or expandable panel with this layout:

A. Result summary header
At the top of the rationale panel show:
- Correct or Incorrect badge
- “Correct answer: X”
- short one-line explanation summary
- optional score/progress update
- visually clear but elegant

B. Structured rationale sections
Break the rationale into clearly separated sections:
- Why this is correct
- Why the other options are wrong
- Key clinical takeaway
- Exam pearl / memory tip
- Related concept if available

Each section should have:
- a clean heading
- soft container/card subsection
- strong spacing
- readable typography
- optional icon if tasteful, but keep it minimal

Do not dump all rationale text into one big paragraph.

C. Rationale image/media block
Very important:
If the question has one or more associated images already stored on the site, show them directly inside the rationale panel in a premium media section.

Implement the rationale media section like this:
- media section title such as “Related visual” or “Clinical image reference”
- image displayed inside a clean bordered/rounded media card
- image caption directly underneath
- image description/explanatory text beneath or beside the image
- support multiple images in a responsive stacked layout or grid
- clicking an image can open a larger modal/lightbox if that already fits your design system
- maintain consistent aspect ratio handling
- images must lazy load
- broken images must gracefully fall back without destroying the layout

The image should feel like part of the teaching experience, not randomly attached.

D. “Why the others are wrong” layout
If the data supports it, render each wrong answer explanation in separate mini-cards:
- Option A
- Option B
- Option C
- Option D

Each mini-card should show:
- answer option label
- whether it was selected or not
- concise explanation of why it is incorrect
- visual emphasis only where needed

This section should be highly scannable.

E. Related image logic
Wire rationale rendering so that if content includes any of these fields, they display automatically:
- imageUrl
- imageAlt
- imageCaption
- imageDescription
- rationaleMedia[]
- rationaleImages[]
- relatedVisuals[]
- explanationMedia[]

If current schema does not support this properly, extend the schema and content mapping so each question and flashcard can attach one or many rationale media items.

Each media item should support:
- url
- alt
- caption
- description
- sort order
- optional type (image/chart/diagram)

F. Empty state
If no rationale image exists, the rationale section should still look complete.
Do not leave a weird empty media container.
Gracefully omit the media block if empty.

Redesign the flashcard view so it feels premium and clean, like a paid interactive study deck.

Front side:

- optional topic badge

Back side:
- answer/definition/explanation

- associated image block if one exists

- optional exam pearl or quick memory tip

C. Flashcard controls
Include polished controls for:
- previous
- next
- flip/reveal
- mark difficult
- save

- restart deck if applicable

Controls should be aligned, modern, and not oversized.

D. Flashcard image support
If a flashcard has an associated image:
- show it in a dedicated visual panel on the back side or beneath the primary content
- display caption/description
- keep layout balanced
- ensure image scales well on mobile
- never allow image overflow or stretching
