TYPE-SPECIFIC PAYLOAD SPECS (MINIMUM VIABLE)
Define and enforce these structures:

A) DRAG_DROP_CLOZE
itemPayload:
- textTemplate: string with placeholders like {{blank1}} {{blank2}} ...
- draggableOptions: { id, label }[] (4–10)
- blanks: { id: "blank1", accepts: optionId[] }[]
correctResponse:
- placements: { blankId, optionId }[]
scoringRule:
- perBlank: 1 point each; allow partial credit

B) DRAG_DROP_RATIONALE
itemPayload:
- baseSentenceTemplate: string with {{cause}} {{effect1}} {{effect2?}}
- draggableCauses: { id, label }[]
- draggableEffects: { id, label }[]
- effectsCount: 1 or 2
correctResponse:
- causeId
- effectIds: string[] (len 1–2)
scoringRule:
- require correct cause AND all required effects for full credit; partial credit allowed by effect

C) DROPDOWN_CLOZE
itemPayload:
- paragraphs: { id, textTemplate }[] (1–5 sentences)
- dropdowns: { id, placeholderKey, options: { id, label }[] }[] (each 3–5 options)
correctResponse:
- selections: { dropdownId, optionId }[]
scoringRule:
- 1 point per dropdown; partial credit

D) DROPDOWN_RATIONALE
itemPayload:
- sentenceTemplate: string with {{cause}} {{effect1}} {{effect2?}}
- causeOptions: { id, label }[] (3–5)
- effectOptions: { id, label }[] (3–5)
- effectsCount: 1 or 2
correctResponse:
- causeId
- effectIds: string[] (len 1–2)
scoringRule:
- MUST choose correct cause and effect(s) for credit; optionally allow partial credit only if configured, default OFF

E) DROPDOWN_TABLE
itemPayload:
- table: { columns: string[], rows: { rowId, label, cells: { cellId, dropdownOptions: {id,label}[] }[] }[] }
correctResponse:
- selections: { cellId, optionId }[]
scoringRule:
- 1 point per cell; partial credit

F) MATRIX_SINGLE (one per row)
itemPayload:
- columns: { id, label }[] (2–10)
- rows: { id, label, options: { columnId }[] }[] (4–10 rows; options derive from columns)
- requireAllRowsAnswered: true
correctResponse:
- selections: { rowId, columnId }[] (exactly 1 per row)
scoringRule:
- 1 point per row

G) MATRIX_MULTI (multi allowed)
itemPayload:
- columns: { id, label }[] (2–10)
- rows: { id, label }[] (4–7)
- selectionRule:
   - perRowMin: 0..n
   - perRowMax: 0..n
   - perColumnMin: 1 (ENFORCE if required by item)
correctResponse:
- selections: { rowId, columnId }[] (multiple per row allowed)
scoringRule:
- row/column based partial credit; enforce selectionRule validation before submit

H) MULTI_RESPONSE_GROUPING
itemPayload:
- groups: { id, label, options: { id, label }[] }[] (2–5 groups; each 2–4 options)
- requireAtLeastOnePerGroup: true
correctResponse:
- selections: { groupId, optionIds: string[] }[]
scoringRule:
- score per group; partial credit by correct options minus incorrect if configured

I) TREND (wrapper)
itemPayload:
- timepoints: { tId, label, nurseNotes?, vitals?, labs?, meds?, orders? }[] (2+)
- embeddedItem:
   - questionType: one of the above
   - stem: optional
   - itemPayload
   - correctResponse
   - scoringRule
correctResponse:
- embeddedCorrectResponse
scoringRule:
- embedded scoring; plus optional “interpretation” points (MVP: embedded only)

J) HIGHLIGHT_TEXT
itemPayload:
- passage: string
- highlightSpans: { spanId, start, end, label? }[] (pre-tokenized selectable segments; up to 10 selected)
- maxSelections: number (<=10)
correctResponse:
- selectedSpanIds: string[]
scoringRule:
- full credit only if all correct and no extras by default; allow partial credit if configured

K) BOWTIE
itemPayload:
- conditionOptions: { id, label }[]
- actionOptions: { id, label }[]
- monitorOptions: { id, label }[]
- slots:
   - conditionSlotId: "condition"
   - actionSlotIds: ["action1","action2"]
   - monitorSlotIds: ["monitor1","monitor2"]
correctResponse:
- conditionId
- actionIds: [string,string]
- monitorIds: [string,string]
scoringRule:
- default: 1 point per slot; allow partial credit
