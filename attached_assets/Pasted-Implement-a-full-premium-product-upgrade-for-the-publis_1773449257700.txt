==================================================

Implement a secure 24-hour free trial flow for exactly one selected tier.

Do not rely only on IP address.

Trial requirements:
• require account creation
• require verified email
• require valid Stripe payment method before trial starts
• do not charge until trial ends unless existing billing rules require otherwise
• allow one trial only per qualified user
• store all trial logic server-side
• never trust client-only state

Data model:
Add fields to existing user/subscription tables or create a dedicated trial_entitlements table tracking:
• userId
• selectedTier
• trialStartedAt
• trialEndsAt
• trialStatus (active, expired, canceled, blocked, converted)
• verifiedEmailAt
• stripeCustomerId
• paymentFingerprint or equivalent non-sensitive dedupe field if available
• deviceFingerprintHash
• signupIp
• lastSeenIp
• abuseFlags JSON
• questionsViewedDuringTrial
• flashcardsViewedDuringTrial
• lessonsViewedDuringTrial
• mockExamPreviewUsed

Trial restrictions:
During trial:
• disable PDF export
• disable print mode for premium content
• disable downloads/export buttons
• enforce usage caps configurable by environment variables
• allow limited questions
• allow limited flashcards
• allow limited lessons
• allow mock exam preview only

Suggested configurable defaults:
• 40 to 75 questions
• 20 to 30 flashcards
• 2 to 3 lessons

UI:

PART 5 — ANTI-ABUSE LAYER FOR TRIALS
==================================================

Block abusive repeat trials using layered checks:

• same verified email already used a trial
• same Stripe payment fingerprint already used a trial

• same IP creating suspicious trial volume in a configurable time window

Add:
• trial audit logs
• abuse notes
• trial activation logs
• admin-visible reasons for blocked trials

Add rate limiting to:
• signup
• login
• email verification
• trial activation
• premium content fetch endpoints

Use feature flags and configuration variables for fraud thresholds.
