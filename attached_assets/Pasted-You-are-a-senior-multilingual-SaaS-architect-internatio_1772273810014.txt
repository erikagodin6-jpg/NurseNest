PHASE 1 — CANONICAL MULTILINGUAL URL ARCHITECTURE
========================================================
Implement URL-based language routing using first path segment:

/en/...
/fr/...
/es/...
etc.

Rules:
1) Language is derived ONLY from the URL segment once present.
2) If user visits a URL without a language segment, redirect to best match:
   - Saved user preference (if exists)
   - Browser language (if supported)
   - Default English (/en/)
3) Language switcher must update the URL segment and trigger full re-render.
4) No ?lang= query parameters for canonical pages.

Deliverables:

- Redirect logic plan

========================================================
PHASE 2 — SINGLE TRANSLATION PIPELINE (NO MORE HARD-CODED STRINGS)
========================================================
Eliminate hardcoded strings that bypass translation (especially static TS lesson titles/content).

Implement one resolver used across the entire app:

resolveI18n(i18nObjectOrKey, lang, fallback="en") -> string/object

Resolution rules:
- If translation exists in requested lang, use it.
- Else use English.
- Never mix languages mid-page: if a structured object (lesson body) is missing in requested language, fallback the entire object to English.
- Log missing translation fields for coverage tracking.

Apply resolver to:
- Lesson list titles and cards
- Lesson detail sections (overview, patho, S&S, diagnostics, management, nursing actions, etc.)
- Activities, quizzes, interactive tools
- Flashcards and decks
- Exam question stem/choices/rationales/exam traps
- UI microcopy relevant to learning flows
- Meta title/meta description/OG/Twitter
- FAQ schema and structured data

Deliverables:
- A list of files/components to refactor (with patterns to replace)
- A shared i18n utility module
- A policy for structured content fallback

PHASE 5 — INTERNATIONAL SEO: HREFLANG + CANONICAL + META + SCHEMA
========================================================
Implement correct SEO signals for every language URL:

A) Canonical
- Each language page must self-canonicalize to its own language URL.
- Never canonicalize a French page to the English page.

B) hreflang
- Add hreflang alternates for every language version of the page.
- Include x-default pointing to /en/.
- Ensure reciprocal correctness.

Example structure:
<link rel="alternate" hreflang="en-ca" href=".../en/...">
<link rel="alternate" hreflang="fr-ca" href=".../fr/...">
<link rel="alternate" hreflang="es" href=".../es/...">
<link rel="alternate" hreflang="x-default" href=".../en/...">

C) Meta tags
- Translate meta title and description via meta_i18n.
- Translate OG + Twitter tags.
- Ensure language attribute in HTML is set correctly.

D) Schema
- FAQ schema must be language-specific.
- JSON-LD must include correct inLanguage.

Deliverables:
- hreflang generator implementation
- canonical rules implementation
- meta generator implementation
- schema translation implementation
