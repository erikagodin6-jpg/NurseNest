Refactor the exam system across NurseNest so each tier (RPN, RN, NP, and other relevant exam-prep tiers) has a clear, realistic exam experience with three distinct products:

1. CAT Exam = realistic adaptive exam simulation
2. Practice Exams = customizable exam builder
3. Test Bank = open-ended study/practice mode

This should align with real nursing exam expectations and clearly separate exam simulation from study tools.

PRIMARY PRODUCT DEFINITIONS

A. CAT Exam
- Realistic adaptive exam simulation
- Minimum 85 questions
- Maximum 150 questions
- Stops only when rules are met after 85 questions or when the user reaches 150
- Adaptive difficulty based on performance
- Blueprint/category-aware selection
- No rationale during active exam
- Review with rationale only after completion

B. Practice Exams
- User chooses exam length
- Can optionally choose topics, systems, difficulty, timed/untimed if supported
- No rationale during active exam
- Full rationale only after final submission and review

C. Test Bank
- Open-ended study and exam-style question practice
- Not a strict exam session
- Can allow rationale in learning/review contexts
- Serves as the flexible study/practice product inside tiers

CAT EXAM REQUIREMENTS

5. Build a realistic CAT exam mode that behaves as close as possible to a real nursing CAT exam:
- Minimum question count = 85
- Maximum question count = 150
- Must never stop before 85 questions
- After 85 questions, evaluate stopping rules after each question or at reasonable checkpoints
- End when sufficient evidence is reached
- If evidence remains inconclusive, continue until 150
- Force end at 150 if still inconclusive

6. CAT decision logic must consider:
- overall ability/confidence estimate
- competency threshold
- blueprint/category coverage
- sufficient evidence across categories/domains
- not just raw score or simple percent correct

7. Blueprint/category-aware CAT logic:
- pull items across required blueprint areas
- avoid over-sampling only one or two topics
- track whether enough evidence exists across core categories
- continue beyond 85 if category coverage is insufficient
- support tier-specific blueprint logic for RPN, RN, NP and future tiers

8. Adaptive difficulty logic:
- increase difficulty when user performs well
- decrease difficulty when user struggles
- avoid repetitive topic clustering
- maintain category balancing while adapting
- use available metadata such as difficulty, topic, system, category, subcategory, tier, question type

9. CAT exam integrity:
During active CAT mode, do NOT show:
- rationale
- correct answer reveal
- distractor analysis
- clinical pearls
- lesson links
- immediate score reveal
- tutor hints

Allow only normal exam interactions such as:
- answer selection
- next question
- flag question if supported
- pause/resume if supported
- progress indicator that does not reveal answers

10. CAT completion screen should show:
- total questions answered
- estimated readiness / performance label
- blueprint/category breakdown
- strengths
- weak areas
- button to enter review mode

11. CAT review mode should unlock only after completion:
- correct answer
- rationale
- distractor analysis
- clinical pearls
- lesson links if available
- filter for missed questions
- flagged question review
- category-by-category review

18. Preserve or create Test Bank as the flexible study/practice experience inside each tier.
Test Bank should be:

- not positioned as a formal exam session

19. Test Bank can include:
- exam-style questions
- rationale in learning/review contexts
- question-by-question study modes
- weak area review
- topic filtering
- flagged question review
- adaptive study if already implemented outside true CAT mode

20. Do not blur Test Bank with CAT Exam or Practice Exams.
The user should immediately understand:
- CAT Exam = real exam simulation
- Practice Exams = custom test builder
- Test Bank = flexible practice/study
