PROMPT FOR AI
Build an interactive web-based OSCE practice game designed for nursing and health-care students. The application should simulate a patient room scenario where the learner must complete clinical tasks by clicking visual hotspots and selecting actions in the correct sequence.
Core requirements:
The game must display a realistic patient room scene as the primary interface. The room should contain a patient, bedside equipment, monitors, medication cart, documentation tools, and common clinical objects (IV pole, oxygen setup, call bell, chart, PPE, etc.). The scene should feel clinically authentic rather than cartoonish.
Users must be able to click directly on objects within the image. Each clickable object should function as a hotspot. When clicked, the hotspot should trigger context-specific actions, choices, or assessments. Example: clicking the patient opens assessment options; clicking the IV pump opens infusion controls; clicking the medication cart opens medication administration steps.
Implement task-based logic:
The learner must complete OSCE-style objectives (for example: initial assessment, vital signs, medication administration, safety checks, patient education, documentation). Actions must be order-dependent. Incorrect sequencing should generate realistic feedback and prevent passing the scenario.
Include choosable text interactions:
When appropriate, display decision panels with selectable clinical responses. Example: “What is your next priority?” or “Which intervention is most appropriate?” Choices should influence scoring and progression.
Hotspot behavior:
Hotspots must visually highlight on hover to indicate interactivity. Clicking a hotspot should not navigate away from the scene but instead open overlays, modals, or side panels.
Scoring and evaluation system:
Track correct actions, incorrect actions, missed safety steps, and sequencing errors. Provide OSCE-style feedback at the end including performance summary, critical errors, and rationale.
Multiple scenarios:
Structure the code so multiple patient cases can be added. Each scenario should be modular, data-driven, and configurable without rewriting core logic.
User experience requirements:
Use a clean, modern UI suitable for educational use. Avoid clutter. Ensure responsiveness for desktop and tablet. Use subtle animations and visual cues to reinforce interactions.
Technical requirements:
Use a standard web stack compatible with Replit (React + TypeScript preferred). Separate logic for scenes, hotspots, scoring, and scenarios. Use component-based architecture. Do not hardcode a single case.
Accessibility considerations:
Support keyboard interaction alternatives for hotspots and readable UI text.
Output expectation:
Generate the full project including components, state management, hotspot system, scenario engine, scoring logic, and placeholder assets. Include clear comments explaining how to add new OSCE cases and room images.
