The NurseNest flashcard system must be redesigned. Currently the flashcards are identical across tiers (RPN, RN, NP). This is incorrect.

Flashcards must be tier-specific and function like untimed practice exam questions with full rationales.

The goal is a study experience similar to UWorld-style flashcards where:
- each card is essentially an exam-style question
- the learner has unlimited time
- rationales are always available
- cards are tier specific (RPN vs RN vs NP)

FLASHCARD SYSTEM ARCHITECTURE

Flashcards must not be generic cards shared between tiers.

Each flashcard must include a required field:

examTier

Valid values:
- RPN
- RN
- NP
- RRT (if applicable)

Only flashcards matching the user’s selected exam tier should load.

Example logic:

if userTier === "RPN":
    show flashcards where examTier == "RPN"

if userTier === "RN":
    show flashcards where examTier == "RN"

if userTier === "NP":
    show flashcards where examTier == "NP"

No cross-tier flashcards should appear unless explicitly designed as universal content.

FLASHCARDS SHOULD BE EXAM QUESTIONS

Flashcards should not be simple definition cards.

Each flashcard should be structured like an exam question:

questionStem
answerChoices
correctAnswer
rationale
incorrectOptionRationales
clinicalPearl
relatedLesson
topic
subtopic
examTier
difficulty

Example schema:

{
  examTier: "RN",
  topic: "Cardiovascular",
  subtopic: "Hyperkalemia",
  questionStem: "...",
  options: {
     A: "...",
     B: "...",
     C: "...",
     D: "..."
  },
  correctAnswer: "C",
  rationale: "...",
  incorrectOptionRationales: {
     A: "...",
     B: "...",
     D: "..."
  },
  clinicalPearl: "...",
  relatedLesson: "hyperkalemia-management"
}

FLASHCARD MODES

Flashcards should have two modes:

LEARN MODE
- answer is visible after user selects option
- rationale immediately appears
- image displayed if available
- lesson link displayed

TEST MODE
- question behaves like exam question
- answer hidden until selected
- rationale appears after answering
- unlimited time
- no score pressure

Both modes should exist within the flashcard system.

FLASHCARDS SHOULD BE TIER APPROPRIATE

RPN cards
- focus on fundamental nursing knowledge
- prioritization
- safety
- basic pharmacology
- common conditions

RN cards
- deeper pathophysiology
- more complex prioritization
- interpretation
- advanced pharmacology
- multi-step clinical reasoning

NP cards
- diagnosis
- treatment decisions
- differential diagnosis
- diagnostic testing
- prescribing decisions

Example:

Topic: Hyperkalemia

RPN flashcard
- identify signs
- basic interventions

RN flashcard
- priority medication
- ECG interpretation

NP flashcard
- determine treatment plan
- select correct pharmacologic therapy

Each tier must have its own flashcard bank.

FLASHCARDS SHOULD COME FROM QUESTION BANK

The flashcard pool should pull from the existing question bank.

Logic:

flashcardPool = questionBank.filter(
    question.examTier === userTier
)

This ensures:
- unlimited flashcards
- continuous expansion
- aligned with exam blueprint

FLASHCARD COUNTS PER TIER

Target minimum flashcard banks:

RPN
1500+ flashcards

RN
2000+ flashcards

NP
2000+ flashcards

Flashcards should mirror exam blueprint categories.

FLASHCARD UI

Flashcards should look like compact exam questions.

Layout:

left panel
question
answer choices

right panel
rationale
image
incorrect explanations
clinical pearl
lesson link

All must fit within one viewport.

Do not create giant vertical cards.

IMAGE RULES

Images must match the question.

Never display a COPD image for asthma.

Image mapping priority:

questionImage
subtopicImage
exactDiagnosisImage
no image

Never use broad fallback images.

LESSON LINKING

Every flashcard must link to the relevant lesson.

Example:

Review Lesson: Hyperkalemia Management

The lesson should open in the learning section.

REMOVE GENERIC FLASHCARDS

Remove or deprecate:
- term definition cards
- generic memorization cards

All flashcards must teach through clinical questions.

ADMIN PANEL

Update admin tools to support:

examTier selector
topic selector
difficulty
question → flashcard conversion

Admins should be able to convert question bank items into flashcards.

ACCEPTANCE CRITERIA

Flashcards must:

- differ by exam tier
- behave like untimed exam questions
- include rationale and incorrect explanations
- link to lessons
- use correct images
- fit within one viewport
- pull from the question bank

Once implemented summarize:

1. new flashcard schema
2. tier filtering logic
3. how flashcards differ across RPN RN NP
4. how question bank integration works
