You are a senior full-stack engineer and product automation architect. Implement an “Automation Center” for my education site that generates, validates, schedules, publishes, and monitors content (question banks, flashcards, study plans, blog posts, comparison pages, and email sequences) with strict guardrails and admin review.

NON-NEGOTIABLE REQUIREMENTS
1) Never change existing paid-wall logic, Stripe logic, or existing question bank delivery format unless explicitly requested. Add automations in a new module and integrate via APIs.
2) All generated content must be stored as DRAFT first, then optionally auto-promoted to PUBLISHED only if it passes validation. Admin must be able to approve/reject/edit.
3) Add role-based permissions: Admin can run/approve; Editor can edit drafts; Viewer can only view.
4) Add global rate limits, cost caps, and job throttles. Include a “kill switch” to pause all automations instantly.
5) Add “Country Mode” and “Tier Mode” support for every generation pipeline (Canada vs USA; RPN/PN/RN/NP; and allied health programs). The system must store country/tier metadata with each asset and render appropriate units/values.
6) Add audit logs for every automation run: inputs, model used, tokens/cost estimate, validations, output ids, publish status.
7) Use a job queue for all automation tasks (e.g., BullMQ/Redis or a simple DB-backed queue). Jobs must be retryable with exponential backoff and idempotent.
8) Add a Scheduler supporting: one-off runs, recurring schedules (daily/weekly), and conditional schedules (e.g., run only if X new items not yet generated).
9) Add content validation: uniqueness checks, format checks, prohibited content checks, tier/country lab value mapping checks, and QA rules for rationales.
10) Add observability: dashboard metrics (jobs run, pass rate, publish rate, errors), notifications (email/webhook) on failures.

AUTOMATION OUTPUT TYPES
- Question items (multiple question types supported)
- Case studies / NGN-style items where applicable
- Flashcards (basic + cloze)
- Study plans (calendar + milestones)
- Blog posts (SEO)
- Comparison pages (SEO)
- Product pages / bundle pages (store)
- Email sequences (onboarding + upsell)
- “Daily question” feature to drive retention

DELIVERY

  a) Automations list with toggles, schedules, and last-run status
  b) Run Now button

  d) Logs/metrics page
  e) Validation report page
- Create APIs to:
  a) create jobs
