You are implementing a full **E-E-A-T authority framework and SEO trust architecture** for a healthcare education website.

The site is a nursing education platform similar to NurseNest and contains approximately **319 public pages** including study guides, certification prep, and practice questions.

Because the site contains healthcare education content, it falls under **Google YMYL (Your Money or Your Life)** standards and must demonstrate strong **Experience, Expertise, Authoritativeness, and Trustworthiness (E-E-A-T)**.

Your task is to **implement the following systems across the entire website** so that search engines recognize the site as a high-quality medical education resource.

Do not remove pages. Improve and strengthen them.

---

AUTHOR AND MEDICAL REVIEW SYSTEM

Create a global **medical reviewer component** that automatically appears on every educational page.

This section should include:

Reviewed by: Registered Nurse
Credentials: RN
Role: Nursing educator and clinical reviewer

Include:

• reviewer name
• credentials
• professional role
• short bio
• optional LinkedIn profile link

Create a dedicated page:

/medical-review-team

This page must contain:

• reviewer credentials
• explanation of the medical review process
• commitment to evidence-based nursing education

Add internal links to this page from the footer and from all article reviewer sections.

---

CONTENT DEPTH STANDARD

Ensure every clinical topic page follows a structured educational format.

Automatically expand or enforce the following sections where relevant:

Introduction
Definition
Pathophysiology
Signs and symptoms
Diagnostic tests and labs
Medications and pharmacology considerations
Nursing interventions
Patient monitoring
Complications
Clinical pearls or exam tips
Practice questions with rationales

Content must include **clinical reasoning**, not only definitions.

---

TOPIC CLUSTER ARCHITECTURE

Implement topic clusters to demonstrate subject authority.

Create hub pages for major nursing categories such as:

Cardiology
Respiratory
Endocrine
Neurology
Electrolytes
Pharmacology

Each hub page must:

• link to all related topic pages
• include short summaries of each topic
• contain structured navigation

Each topic page must also link back to its hub page.

Ensure every article contains **3–8 contextual internal links** to related educational topics.

---

MEDICAL REFERENCE SYSTEM

Add a references section to all clinical articles.

Where applicable, cite authoritative sources such as:

World Health Organization
Centers for Disease Control and Prevention
National Institutes of Health
peer-reviewed medical literature

References must appear near the end of each article.

---

LAST REVIEWED DATES

Add a visible line near the top or bottom of each page:

Last medically reviewed: [current month and year]

Ensure this date updates when content is modified.

---

FAQ SECTIONS

Where appropriate, add an FAQ section at the end of educational pages.

Questions should address common nursing exam or clinical queries.

Examples:

What causes hyperkalemia?
How is hyperkalemia treated?
What labs confirm hyperkalemia?

Implement **FAQ schema markup** for these sections.

---

STRUCTURED DATA IMPLEMENTATION

Add structured data where appropriate:

Article schema for study guides
FAQ schema for question sections
Breadcrumb schema for navigation

Ensure schema follows schema.org standards.

---

INTERNAL LINKING SYSTEM

Ensure every page has internal links to:

• its parent topic hub
• related clinical topics
• relevant practice question pages

No page should be isolated.

All pages must be reachable within **three clicks from the homepage**.

---

TECHNICAL SEO AND INDEXING

Perform a full crawl and indexing audit.

Tasks:

Generate a complete XML sitemap containing every public page.

Ensure the sitemap is accessible at:

/sitemap.xml

Verify robots.txt allows search engine crawling.

Confirm no pages contain a "noindex" directive unless intentionally hidden.

Ensure canonical tags point to the correct URL.

Identify orphan pages that lack internal links.

Ensure each page has:

• a unique SEO title
• a unique meta description

Confirm server responses:

• valid pages return HTTP 200
• missing pages return HTTP 404

---

JAVASCRIPT SEO COMPATIBILITY

Ensure the website's React or JavaScript framework does not block search engine crawling.

Critical requirements:

• primary page content must be visible in the initial HTML render
• avoid relying solely on client-side rendering
• ensure meta tags and structured data are present in the server response

If necessary, implement server-side rendering or pre-rendering for content pages.

---

PERFORMANCE AND USER EXPERIENCE

Ensure each page:

• loads quickly
• renders correctly on mobile devices
• contains readable formatting
• avoids large layout shifts

Fix any issues that reduce page performance or usability.

---

FINAL OUTPUT REPORT

Generate a report containing:

Pages fully compliant with E-E-A-T standards
Pages requiring content expansion
Pages missing internal links
Pages missing metadata
Pages blocked from indexing
Orphan pages
Pages missing structured data
Technical SEO issues

Provide recommended improvements for any page that does not meet the quality standard.
