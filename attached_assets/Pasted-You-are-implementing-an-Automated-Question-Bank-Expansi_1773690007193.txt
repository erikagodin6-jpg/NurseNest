You are implementing an **Automated Question Bank Expansion and Quality Control System** for the NurseNest platform (https://www.nursenest.ca).

The objective is to allow NurseNest to **continuously grow its question banks automatically while preventing duplicates, maintaining exam blueprint coverage, and preserving question quality**.

This system must work across all tiers:

RPN (REx-PN)
RN (NCLEX-RN)
NP
Allied Health professions

The system must automatically detect gaps in topic coverage and generate additional high-quality exam questions where needed.

Do NOT break the current practice question engine or lesson architecture.

---

CRITICAL RULES

Do NOT delete existing questions.
Do NOT create duplicate questions.
Do NOT generate extremely similar question stems.
Do NOT break multilingual support.
Do NOT break exam simulation functionality.
Do NOT modify navigation structure.

All generated questions must pass a **quality validation step before publishing**.

---

STEP 1 — BUILD QUESTION BANK MONITORING

Create a monitoring system that continuously scans the database and tracks:

Total questions per tier
Questions per topic category
Questions per difficulty level
Questions per lesson

The system must identify areas where the number of questions falls below target thresholds.

---

STEP 2 — DEFINE QUESTION BANK TARGETS

Use the following target ranges.

RPN / REx-PN Tier

Minimum 1,800 questions
Target 2,000–2,500 questions

RN / NCLEX-RN Tier

Minimum 3,000 questions
Target 3,500–4,500 questions

NP Tier

Minimum 1,500 questions
Target 2,000–2,500 questions

Allied Health professions

Minimum 300 questions per profession
Target 400–600 questions

---

STEP 3 — IDENTIFY TOPIC GAPS

When scanning the question bank, detect underrepresented topics.

Example categories:

Electrolytes
Cardiac nursing
Respiratory disorders
Pharmacology
Maternal newborn
Pediatrics
Mental health
Leadership and delegation
Critical care
Emergency nursing

If a category falls below the minimum question threshold, mark it for expansion.

---

STEP 4 — AUTOMATIC QUESTION GENERATION

Generate new questions only for topics that require expansion.

Each generated question must include:

Clinical scenario
Four answer options
One correct answer
Detailed explanation
Rationale for incorrect answers
Lesson reference

Questions must simulate **real exam format used in NCLEX or REx-PN style exams**.

---

STEP 5 — DUPLICATE DETECTION SYSTEM

Before adding any generated question to the database, run a duplicate detection check.

Compare:

Question stems
Answer patterns
Clinical scenario structure

If similarity exceeds the allowed threshold, reject the question.

---

STEP 6 — QUESTION DIFFICULTY BALANCING

Maintain balanced question difficulty across the entire bank.

Target distribution:

Easy – 30%
Moderate – 45%
Hard – 25%

Automatically tag generated questions with difficulty labels.

---

STEP 7 — LESSON LINKING

Every generated question must be linked to a corresponding lesson.

Example:

Electrolyte imbalance question → Electrolyte Disorders lesson

This ensures that users can immediately review the underlying concept.

---

STEP 8 — QUALITY VALIDATION

Before publishing a generated question, run validation checks:

Ensure the correct answer is accurate.
Ensure distractor answers are plausible.
Ensure explanations are medically accurate.
Ensure grammar and clarity meet educational standards.

Questions that fail validation must not be published.

---

STEP 9 — MULTILANGUAGE QUESTION SUPPORT

After a question passes validation, automatically generate translations for all supported languages.

Translate:

Question stem
Answer options
Explanations

Ensure translations maintain medical terminology accuracy.

---

STEP 10 — UPDATE HERO QUESTION COUNTS

Whenever new questions are added, automatically update the displayed counts on hero pages.

Examples:

RPN Hero

2,000+ Practice Questions

RN Hero

4,000+ NCLEX-Style Questions

NP Hero

2,000+ Advanced Clinical Questions

Counts must update dynamically without manual editing.

---

STEP 11 — MAINTAIN EXAM SIMULATION QUALITY

Ensure simulated exams draw questions randomly across all categories and difficulty levels.

Example exam sizes:

RPN simulation – 85–100 questions
RN simulation – 100–120 questions
NP simulation – 75–100 questions

Avoid repeating the same questions within the same simulated exam.

---

STEP 12 — CONTINUOUS IMPROVEMENT SYSTEM

Allow the system to periodically analyze:

Questions users frequently miss
Topics where students struggle
Questions with poor explanations

Use this data to generate improved questions and explanations.

---

FINAL RESULT

After implementation the NurseNest platform will have a **self-expanding exam preparation system** that:

Maintains exam blueprint coverage
Prevents duplicate questions
Automatically expands question banks
Maintains balanced difficulty levels
Supports all languages
Updates hero statistics automatically

This system allows NurseNest to scale into a **large professional exam preparation platform comparable to major nursing exam prep providers**.
