0) Fix the Home hooks error properly (don’t guess)

If the app compiles but you’re seeing a “React hooks error” in Home, it’s almost never the Badge import itself. Missing imports usually throw a reference error, not a hooks rule error.

The two common causes after adding a “dual-path” section:

A hook is being called conditionally (inside an if, inside a ternary, inside a map callback, etc.)

You accidentally created a nested component/function that uses hooks and is invoked conditionally

Quick checklist in Home.tsx

Ensure all hooks (useState, useEffect, useMemo, useTranslation, etc.) are at the top level of the component, before any returns/conditionals.

If you added something like:

if (something) {
  const { t } = useTranslation()
}

move it to the top.

If you wrote:

const DualPath = () => {
  const [x, setX] = useState(...)
  ...
}
if (show) return <DualPath />

That’s OK. But if you call it like DualPath() anywhere, not OK.

Fix the Badge import (still do it)

If you are using shadcn/ui style imports, add:

import { Badge } from "@/components/ui/badge";

If your project uses a different path, match your existing import patterns.

✅ After these two checks, the hooks error should disappear. If not, paste the exact hooks error text (it usually names the file/line and “Rendered fewer hooks than expected” / “Invalid hook call”) and I’ll tell you the exact edit.
