Implement locale-prefixed URLs, hreflang tags, and localized sitemap for my Vite + React + Express app using Wouter routing. Current i18n works via t() and language stored in localStorage; keep that as fallback but make URL the source of truth for locale.

Locales (keep existing set): en, fr, es, fil, hi, zh, ar, ko, pt, pa, vi, ht, ur, ja, fa

GOALS (must complete all):
1) Locale-prefixed URLs:
All public routes must support:
  /:locale/...
Examples:
  /en/lessons
  /fr/lessons
  /es/anatomy
Locale must be a required first path segment for indexable pages.

2) Backward compatibility + redirects (critical):
Legacy routes without locale (e.g., /lessons, /anatomy, /blog, /faq, /mock-exams, /dashboard, /flashcards, etc.) must 301 redirect to the default locale (en):
  /lessons -> /en/lessons
Also redirect "/" -> "/en" (or "/en/" depending on routing)
Redirect must be done in Express (server-side), not client-only, to preserve SEO.

3) Locale resolution rules:
- URL locale is authoritative.
- If URL lacks locale: server redirects to /en/...
- If locale is invalid: redirect to /en/...
- Persist user selection: when user changes language, update localStorage AND navigate to the same path under the new locale.
- Keep English fallback behavior for missing keys.

4) Wouter routing changes:
Update route definitions to include optional/required locale segment and propagate locale into the app context so t() uses it.
- Implement a LocaleProvider that reads locale from the URL param.
- Ensure all internal links generate locale-prefixed hrefs (e.g., use a helper like hrefWithLocale(locale, path)).
- Ensure dynamic routes (lesson pages, blog posts, etc.) work identically across locales.

5) RTL support:
Maintain/confirm RTL for ar and fa (and ur if already supported).
Set dir="rtl" at document level for RTL locales.

6) hreflang + canonical (SEO critical):
On every indexable page, inject:
- canonical pointing to the current locale URL (self-referencing canonical)
- hreflang alternates for ALL locales, pointing to the equivalent path in each locale
- x-default pointing to the English version
Implementation approach:
- Compute the “path without locale” (e.g., /lessons, /blog/post-slug) then generate alternates by prefixing each locale.
- Ensure domain is https://www.nursenest.ca for all generated links.
- Ensure the head tags update on client navigation (SPA). Use react-helmet-async or an equivalent that works with Vite + React.

7) Localized sitemap (domain safety critical):
Generate sitemap XML that includes ONLY https://www.nursenest.ca URLs and includes localized URLs for all locales.
- Include all public, indexable routes (and dynamic content like blog posts and lessons if they exist in data).
- Exclude private/paywalled/noindex routes from sitemap.
- Provide either:
  a) a single sitemap.xml with all localized URLs, OR
  b) sitemap_index.xml + per-locale sitemaps (preferred if large)
Serve sitemap(s) from Express endpoints: /sitemap.xml (and /sitemap_index.xml if used).
No replit.app URLs anywhere.

Do NOT break existing business logic.
