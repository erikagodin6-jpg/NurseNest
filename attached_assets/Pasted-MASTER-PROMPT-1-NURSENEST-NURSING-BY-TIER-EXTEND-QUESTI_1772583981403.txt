MASTER PROMPT 1 — NURSENEST NURSING (BY TIER) — EXTEND QUESTION BANK GENERATOR WITHOUT TOUCHING CORE

You are a senior full-stack engineer working on the NurseNest platform.

TASK
Extend the existing Question Bank Generator to support advanced, exam-accurate question generation for NurseNest Nursing TIERS, WITHOUT modifying or refactoring the current generator’s core logic, layouts, PDF compilation, or existing prompt templates.

ABSOLUTE CONSTRAINTS (NON-NEGOTIABLE)
Do NOT edit:

* the existing generator function(s)
* PDF/layout rendering logic
* existing templates, styles, branding, or page order
* existing endpoints and routes (unless adding NEW routes)
* database schema fields currently used by the generator (you may ADD new optional fields)

IMPLEMENTATION APPROACH
Create a non-destructive “Enhancer Layer” that runs AFTER the legacy generator output is produced.

Flow:
LegacyGenerator() → NursingTierEnhancer() → FinalQuestionSet()

Create new module:

* /server/services/nursingTierEnhancer.ts

Add feature flag:

* ENABLE_TIER_ENHANCER=true|false
  If false: output must match legacy behavior exactly.

---

A) NURSING TIERS + EXAMS (SUPPORTED MODES)

Support these tier profiles:

1. RPN Tier (Canada)

* Primary exam: REx-PN
* Country mode default: CA
* Scope: practical nursing, foundational med-surg, LTC/community, maternity basics, pedi basics, mental health, safety/ethics, meds fundamentals, assessment.

2. RN Tier (Canada + US)

* Exams: NCLEX-RN (US), Canadian RN practice (content style can be NCLEX-RN aligned)
* Country modes: CA or US toggle
* Scope: broader med-surg depth, leadership/management, complex patients, prioritization, delegation.

3. NP Tier (US + Canada focused content)

* Exams: AANP/ANCC-style (do not claim perfect alignment; generate common NP-style formats)
* Country modes: CA or US toggle
* Scope: advanced assessment, diagnosis/differential, management, prescribing considerations, guidelines-based reasoning.

B) ENHANCED QUESTION TYPES (BY TIER)

The enhancer must attach structured support for multiple question formats while keeping legacy MCQ compatibility.

Global supported types:

* MCQ (single best answer)
* SATA
* Ordered response
* Fill-in / numeric
* Matrix/grid (select options per row)
* Case-based sets

NGN-focused types for RN tier and above (and optionally RPN):

* Bowtie
* Trend/clinical judgment case
* Highlight table/case decision

Tier enforcement:

* RPN: mostly MCQ + SATA + basic case sets; optional NGN lite
* RN: MCQ + SATA + full NGN suite (bowtie/trends/matrix)
* NP: vignette-heavy, diagnosis/management, best next step; fewer NGN mechanics, more clinical reasoning

C) RATIONALES (TIER-SPECIFIC DEPTH)

Expand rationales WITHOUT replacing legacy rationales.

Function:
expandRationale(question, tier)

Minimum word counts:

* RPN: 350+ words
* RN: 500+ words
* NP: 650+ words

Rationale sections (all tiers):

1. What the question is testing
2. Why the correct answer is correct
3. Why each distractor is wrong
4. Nursing/clinical implications
5. Exam strategy tip

D) BLUEPRINT TAGGING + METADATA (NON-BREAKING)

Add optional metadata to each question using a JSON field:
question_metadata (JSON)

Required keys:

* tier: "RPN"|"RN"|"NP"
* exam: "REx-PN"|"NCLEX-RN"|"AANP/ANCC"
* country_mode: "CA"|"US"
* domain: e.g., "Cardio", "Resp", "Pharm", "Mental Health"
* competency: e.g., "Assessment", "Prioritization", "Safety", "Teaching"
* difficulty: "easy"|"moderate"|"hard"|"very_hard"
* question_type: "mcq"|"sata"|"bowtie"|...
* learning_objective: short string
* exam_trap_notes: short string

Do not require the UI to use these fields immediately. They must be optional.

E) COUNTRY MODE RULES (CA VS US)

Add country_mode support in the enhancer:

* CA: Canadian terminology, REx-PN tone, metric units, mmol/L when relevant
* US: NCLEX wording conventions, mg/dL when relevant

The enhancer must not rewrite the entire stem—only adjust terminology in the ADDED sections and metadata unless an “ALLOW_STEM_NORMALIZATION” flag is enabled.

Flags:

* ENABLE_TIER_ENHANCER=true
* ALLOW_STEM_NORMALIZATION=false by default
