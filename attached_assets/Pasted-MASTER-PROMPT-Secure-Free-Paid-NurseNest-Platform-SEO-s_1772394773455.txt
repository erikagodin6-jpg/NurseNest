MASTER PROMPT — Secure Free + Paid NurseNest Platform (SEO-safe) with Premium Smart Prep, QBank/Exam Factory, Store, Unlocks, Analytics, Friends/Comparison, and Marketing Automation

Goal:
Implement a secure hybrid free+paid architecture so Google traffic can discover and preview content, but cannot access premium features or paid content without valid entitlements. Also implement the Premium Smart Prep feature set (analytics, percent-correct, percentile, readiness score, flashcards, friends comparison) and the QBank/Exam creation+publishing pipeline, with automated marketing blocks on the hero page and paid plans page.

NON-NEGOTIABLE SECURITY RULES (must be enforced server-side):
1) Never rely on frontend hiding. All premium content must be gated on the server.
2) Do NOT ship premium question banks (full JSON) in page HTML, client bundles, or public static files.
3) Do NOT expose raw file URLs. All downloads must be authenticated or signed (short-lived).
4) Prevent query-param bypasses (e.g., ?full=1). Server must validate entitlements on every request.
5) Add rate limits and anti-scraping protections for question endpoints and downloads.
6) Use noindex/robots rules to prevent indexing of /app/ premium routes.

Deliverables:
- Surgical diffs only (no full-file rewrites).
- A security audit checklist confirming no premium data is exposed publicly.
- UI proof flows for: free user, logged-out user, premium user, purchased single product user.

PART C — SECURE DOWNLOADS (NO PUBLIC FILES)
========================================================
Downloads must go through server:
- GET /api/download/:productId?variant=...
Server must:
1) Verify auth session/JWT
2) Check entitlement:
   - purchase exists OR unlock exists OR premium unlimited rule
3) If valid:
   - Return a short-lived signed URL (5–10 minutes) OR stream the file
4) If invalid:
   - Return 403

Watermarking (optional, recommended for premium):
- Watermark PDFs with user identifier (email hash) and purchase/unlock id.

========================================================

Features:

- distribution mix (MCQ/SATA/PRIORITY/DELEGATION)

- Canadian context toggle
- Editions: Questions-only+AnswerKey (default ON), Study Edition (optional)

Generation engine:
- Chunked generation (50 per batch) until requestedCount reached.
- Strict JSON only.
- Validate each chunk and final set:
  Types: MCQ/SATA/PRIORITY/DELEGATION
  SATA: 6–8 options, 2–5 correct
  MCQ/PRIORITY/DELEGATION: 4 options, 1 correct
  Categories limited to your 4 domains
  Topic tag required

Input: requestedCount, existingQuestions, basePrompt, topicMix, typeMix, difficulty
Output: merged validated set exactly requestedCount + audit.

Audit panel must exist and be persistent:
requestedCount, generatedCount, countMatch, byType, byCategory, byTopic, attempts, errors.

PART G — PREMIUM SMART PREP FEATURES (GATED)
========================================================
Premium-only features:
1) Per-question “% of users got this correct” + difficulty badge
2) Percentile ranking on exams
3) Readiness score (based on exams + domain balance + trend)
4) Smart flashcards (unlimited + AI-generated)
5) Study groups + friend comparison dashboards
6) Shareable stats cards

Free tier:
- limited flashcards (e.g., 20)
- no peer stats, no percentile, no readiness score, no friends

Important peer-stats threshold gating (avoid fake stats early):
- Do NOT show per-question percentCorrect until:
   totalAttempts >= 30 AND uniqueUserCount >= 15
- Do NOT show exam percentiles until:
   totalUsersWithQualifiedExamData >= 30
- If below threshold, show:
   “Peer data will appear once more students answer.”

Post-answer only:
- Do not show peer stats before the user answers the question.

Tracking:
For each question:
- totalAttempts, totalCorrect, uniqueUserCount, percentCorrect, difficulty
For each user:
- firstQuestionAnsweredAt, totalQuestionsAnswered, totalExamsCompleted, studyMinutes
Compute experience bracket:
- 0–7d, 8–30d, 31–90d, 90+d
Comparison “students practicing as long as you” uses this bracket.
