You are my senior full-stack engineer working in my existing NurseNest codebase (Vite + React + Express + DB). Build an “Exam Questions Scheduler + Generator” system that works similarly to my existing blog scheduling function, but for exam questions.

GOALS
1) For each tier and for each question type, generate and/or add **100 new questions** (bulk creation).
2) Ensure **ALL question types** that exist on each exam are covered for that tier (no missing types).
3) Let me **view scheduled questions** (calendar + list) inside my admin dashboard.
4) Let me **add, edit, bulk import, and schedule** more questions exactly like the blog posting scheduler (draft → scheduled → published).
5) Admin must bypass any copy/paste/screenshot restrictions that apply to regular users.

TIERS + EXAMS
- Canada:
  - RPN tier: REx-PN style
  - RN tier: NCLEX-RN style (Canada uses NCLEX-RN)
  - NP tier: Canadian NP exam style (and any US NP exams I support, see below)
- USA:
  - PN/LVN tier: NCLEX-PN style
  - RN tier: NCLEX-RN style
  - NP tier: US NP exams I support (configureable list, e.g., AANP, ANCC; do not hardcode—make it admin-configurable)

QUESTION TYPES (MUST COVER ALL THAT APPLY PER EXAM)
Implement a “Question Type Registry” so the system knows which question types exist per exam and can enforce coverage.
At minimum support (and allow adding more later):
- Single best answer MCQ (4–5 options)
- SATA / multiple response (select all that apply)
- Ordered response / prioritization (drag-drop order)
- Case study / vignette sets (multi-question, shared stem)
- Matrix / grid (if applicable)
- Fill-in / short answer (only if exam supports it)
- Hotspot / image-based (optional; support attaching an image later even if none now)
- Exhibit / chart / lab interpretation (attach a table/exhibit to the stem)
- Bow-tie / triage-style (if applicable to the exam format I choose)
Make the registry flexible: per EXAM you can enable/disable types and optionally set weighting targets.

HARD REQUIREMENTS (IMPORTANT)
- Bulk generation: “Create 100 per tier per question type” must create 100 *for each (tier × question_type)* that is enabled for that tier’s exam mapping.
- Coverage validation: Admin UI must show a coverage checklist and block “Generate” if any enabled question type has zero target or is excluded accidentally.
- Scheduling: Each question can be:
  - Draft (not visible)
  - Scheduled (has publish_at datetime, not visible until time)
  - Published (visible)
  - Archived (hidden but retained)
- Scheduler runner: a server job (cron-like) that promotes scheduled → published at the correct time. Must be timezone-aware (use America/Toronto as default, but admin can choose).
- Admin dashboard views:
  - “Queue” view: upcoming scheduled items, filters by tier, exam, question type, date range, status
  - “Calendar” view: month/week calendar showing publish events
  - “Bulk generator” view: select tiers, exams, enabled types, set targets (default 100 each), set publish cadence (e.g., 10/day), schedule window, and auto-assign publish_at times.
  - “Question editor” view: edit stem/options/correct answers/rationales/tags/difficulty and preview as learner sees it.
- Similar UX to blog scheduler: include “Add to schedule”, “Duplicate”, “Bulk edit”, “Reschedule”, “Pause schedule”.
- Permissions:
  - Only admins can generate/schedule/edit.
  - Admin can preview all tiers and gated content.
- SEO/site behavior: scheduled questions should not be publicly indexable before publish (no leaking), and question pages should respect my existing paywall/tier rules after publish.

QUESTION DATA MODEL
Create or extend DB tables with at least:
- questions
  - id, tier, exam, question_type, status, publish_at, created_at, updated_at
  - stem (rich text), options (json), correct_answer (json), rationale (rich text)
  - difficulty (1–5), tags (array), body_system, topic, subtopic
  - case_id (nullable) for case sets, exhibit_id (nullable)
- question_cases (optional) for shared stems
- exhibits (optional): tables, labs, images metadata
- question_type_registry
  - exam, question_type, is_enabled, default_target_count, display_name, validation_rules (json)
- question_schedule_log (optional) for audit trail
Ensure indexes for publish_at, status, tier/exam/type.

- Canadian vs US wording + units + ranges where relevant (tie to my existing “Canada vs USA” separation rules).

Admin must be able to:

- Set difficulty distribution targets
- Choose “generate as drafts only” or “generate + schedule automatically”
