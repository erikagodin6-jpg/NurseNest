==================================================

- AI generation
- bulk question generation
- PDF/image generation
- analytics aggregation
- background validation jobs
- scheduled audits
- synthetic tests
- large exam assembly

Implement a permanent separation of concerns:

SECTION 2 — MEMORY STABILITY AND LEAK PREVENTION
==================================================

Perform a deep audit for memory pressure and memory leaks.

Inspect and fix:
- large exam payload creation
- oversized JSON responses
- overfetching from the database
- unbounded caches
- global arrays/maps/objects that grow over time
- repeated timers, listeners, subscriptions
- retained request/response objects
- memory not returning to baseline after requests
- duplicate in-memory transformations
- repeated polling / retry loops
- frontend refetch loops
- synthetic tests repeatedly hammering unstable routes
- background jobs sharing memory with the web app
- session or exam attempt state stored in process memory

Implement permanent memory-safe patterns:
- chunking/pagination
- lazy loading
- bounded caches with eviction
- request timeouts
- route concurrency caps
- route-level memory logging
- load shedding when memory thresholds are exceeded
- memory thresholds with warning/critical actions
- safe process health endpoints

Add instrumentation for:
- RSS
- heap used / heap total
- route-level latency
- payload size
- concurrent requests
- concurrent heavy jobs
- memory trend over time
- restart frequency

Identify the top memory offenders and fix them first.

SECTION 3 — EXAM SYSTEM REDESIGN FOR RELIABILITY
==================================================

Redesign the exam delivery path to be production-safe.

- frontend initiation
- auth/session
- entitlement checks
- exam lookup
- content/question retrieval
- validation
- serialization
- response delivery
- frontend rendering

Implement permanent rules:
