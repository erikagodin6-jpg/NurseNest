MASTER PROMPT: Implement AI-Verified User-Generated Flashcards (Non-Tiered, Revenue-Protective)

You are my senior full-stack engineer working inside my existing NurseNest codebase (Vite + React frontend, Express backend, DB already supports flashcards). Implement an AI-powered medical accuracy verification layer for USER-GENERATED flashcards, integrated into the existing flashcard feature. This is a FEATURE ADDITION, not a rebuild.

HARD REQUIREMENTS (Do Not Violate)
1) Non-tiered by scope: Do NOT apply RPN/RN/NP scope logic. Users can create cards for ANY body system/topic they want.
2) Revenue-protective: This verification must NOT replace or undermine my paid tier features (scope tiering, exam simulator, AI tutor, adaptive plans, etc.). Keep this as “accuracy validation” only.
3) Public indexing policy: Only AI-Verified cards can be publicly searchable / shareable / indexed. Unverified or flagged cards must remain private by default.
4) Safety-first: Dangerous misinformation must be blocked from public sharing and routed to an admin queue.
5) No long-form teaching: The verification output must be concise. No full lessons, no step-by-step tutoring, no exam coaching. Provide short corrections only.
6) Works across all systems: cardiac, respiratory, renal, endocrine, OB, peds, pharm, labs, acid-base, etc. No restrictions.

CORE FEATURE: AI VERIFICATION WORKFLOW
When a user creates or edits a flashcard:
A) Send the flashcard content (front, back, optional image metadata, tags) to a backend verification endpoint.
B) The AI returns a STRUCTURED JSON response that includes:
   - status: "VERIFIED" | "PARTIALLY_ACCURATE" | "INACCURATE" | "UNSAFE"
   - confidence: 0–100
   - issues: array of { field: "front"|"back"|"both", type: "FACT_ERROR"|"OUTDATED"|"AMBIGUOUS"|"MISSING_CONTEXT"|"UNSAFE", snippet, explanation_short, suggested_fix }
   - corrected_version: { front, back } (only if not VERIFIED; keep concise, do not expand into a lesson)
   - safety_flags: array of { category: "DOSING"|"CONTRAINDICATION"|"EMERGENCY_ACTION"|"CRITICAL_VALUE"|"ACID_BASE"|"OTHER", severity: "HIGH"|"MED"|"LOW", note_short }
   - publish_eligibility: "PUBLIC_OK" | "PRIVATE_ONLY" | "ADMIN_REVIEW_REQUIRED"
   - disclaimer_short: one sentence displayed to user (“AI verification supports study but does not replace clinical judgment.”)
C) Store verification results in DB with timestamps and model version.
D) UI displays:
   - badge labels: "AI Verified", "AI Flagged (Private)", "Not Verified"
   - if not verified: show highlighted issues + suggested fix + allow user “Apply Fix” or “Edit Manually” then “Re-Verify”
E) Only when status is VERIFIED and publish_eligibility is PUBLIC_OK may a flashcard set be shared publicly or indexed.

SAFETY ESCALATION RULES
Automatically classify and block “UNSAFE” when content includes likely harm risk, including:
- medication dosing errors (units, frequency, route, unsafe max doses)
- contraindicated medication claims (e.g., pregnancy contraindications misstated)
- incorrect emergency/ACLS interventions
- incorrect critical lab values (K+, Na+, glucose, etc.)
- dangerous acid-base interpretations or DKA/HHS mismanagement claims
Unsafe cards:
- cannot be made public
- must be marked “ADMIN_REVIEW_REQUIRED”
- must appear in an admin moderation queue with filters by severity and category

FREE VS PAID (REVENUE-PROTECTIVE GATING)
Implement capability gating without scope tiering:
- Free users: allow limited verifications per day (default 5–10/day configurable). Show only status + very minimal issue summary (no detailed fixes).
- Paid subscribers: unlimited verifications; show full concise correction suggestions; allow bulk verification of a set; allow “export verified-only set” feature; priority queue processing.
IMPORTANT: This gating should integrate with my existing subscription system. Do not create a new standalone purchase or micro-transaction.

IMAGE REQUIREMENTS (NO TEXT IN IMAGES)
Users may attach images to flashcards. For PAID users only, add an “image medical accuracy check” step:
- Verify images are not misleading anatomically/physiologically (lightweight check)
- Enforce: no embedded text in images for any images created by our generator
- If image check fails, keep card PRIVATE_ONLY and flag issue type "IMAGE_ACCURACY"
NOTE: Do not generate images in this task; only validate. If an image generator exists already, ensure it outputs images without embedded text and stores labels separately as HTML.

SEO / INDEXING REQUIREMENTS
- Only AI-Verified public sets can be indexed and appear in public browsing/search pages.
- Add structured data (schema) on public set pages to support SEO (Educational content / Q&A style where appropriate).
- Add canonical URLs, breadcrumbs, and internal links from public set pages to related lessons if those exist.
- Ensure robots rules do not index private/unverified/flagged content.

ADMIN DASHBOARD REQUIREMENTS
Add an admin panel page with:
- list of flagged cards requiring review
- filters: severity, category, date, user, system tags
- ability to open a card, see original, AI findings, and corrected suggestion
- actions: approve for public (force VERIFIED), keep private, request user edit (send message template), or delete if violates policy

  - If set contains any non-VERIFIED cards, disable “Publish/Share Publicly” and show which cards block publication.

- never fabricates citations; if you include any “source type,” keep it generic (e.g., “standard nursing pharm reference”)
