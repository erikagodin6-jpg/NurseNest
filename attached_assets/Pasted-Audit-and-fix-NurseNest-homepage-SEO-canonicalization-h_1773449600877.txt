Audit and fix NurseNest homepage SEO, canonicalization, hreflang, sitemap consistency, and branded search visibility for the published production site only.

Goal:
Make the main NurseNest homepage become the strongest branded result for searches like “nursenest” by consolidating technical SEO signals, reducing duplicate/conflicting homepage variants, and strengthening brand metadata.

Important:
• work on the production/published site only
• do not point anything to development database
• do not break auth, subscriptions, lessons, flashcards, questions, mock exams, analytics, or admin
• preserve existing multilingual support

==================================================
PART 1 — CHOOSE AND ENFORCE ONE PRIMARY HOMEPAGE URL
==================================================

1. Determine the intended canonical homepage.
   Preferred example:
   https://www.nursenest.ca/en

2. Enforce this consistently:
   • root / must 301 redirect to the canonical homepage if that is the intended default
   • all internal logo links, homepage buttons, footer links, and nav links must point to the same canonical homepage URL
   • sitemap homepage entry must use the same canonical homepage URL
   • organization schema and website schema must use the same canonical homepage URL

3. Ensure no conflicting canonical tags exist between:
   • /
   • /en
   • www vs non-www
   • trailing slash vs no trailing slash variants

==================================================
PART 2 — FIX CANONICAL TAGS SITEWIDE
==================================================

1. Add or verify correct canonical tags on all pages.
2. Every page must self-canonicalize to its own preferred URL unless there is a valid alternate consolidation rule.
3. English homepage canonical must point to the chosen homepage URL only.
4. Avoid canonicals that point multiple different language pages to the English page unless that is explicitly intended.
5. Make sure paginated, lesson, blog, flashcard, and simulator pages do not accidentally canonicalize to the wrong route.

==================================================
PART 3 — FIX HREFLANG AND X-DEFAULT
==================================================

1. For every translated equivalent page, output proper hreflang annotations.
2. Each page should include:
   • self hreflang
   • reciprocal hreflang references to alternate language versions
   • x-default pointing to the default landing page
3. Ensure hreflang clusters are valid and complete for:
   • homepage
   • major landing pages
   • lessons
   • flashcards
   • simulators
   • blog or learning hubs if translated
4. Do not generate hreflang tags for pages that do not have true equivalents.

==================================================
PART 4 — STRENGTHEN HOMEPAGE TITLE, META, H1, AND BRAND SIGNALS
==================================================

1. Update the homepage title so it begins with NurseNest.
   Recommended format:
   NurseNest | NCLEX, REx-PN & Nursing Exam Prep

2. Update the homepage meta description so it clearly states:
   • NurseNest brand
   • core audience
   • core products
   • Canada/NCLEX/REx-PN relevance if appropriate

3. Ensure the homepage H1 prominently includes NurseNest.

4. Ensure homepage intro copy naturally includes brand terms such as:
   • NurseNest
   • nursing exam prep
   • NCLEX
   • REx-PN
   • nursing question bank
   • nursing flashcards
   • nursing mock exams

5. Avoid keyword stuffing.

==================================================
PART 5 — ADD/VERIFY STRUCTURED DATA
==================================================

1. Add or verify Organization schema.
2. Add or verify WebSite schema.
3. Include:
   • organization name: NurseNest
   • canonical homepage URL
   • logo
   • sameAs links for official social profiles if available
4. Ensure there is no conflicting schema using different URLs or brand spellings.

==================================================
PART 6 — CLEAN UP SITEMAPS
==================================================

1. Audit all sitemap files currently generated.
2. Ensure only valid sitemap URLs are linked in robots.txt and submitted to Google Search Console.
3. Remove broken, duplicate, typo, or deprecated sitemap references.
4. Ensure sitemap indexes reference only existing child sitemaps.
5. Ensure canonical homepage URL is present in the sitemap.
6. Ensure multilingual URLs in sitemaps match canonical and hreflang logic.

==================================================
PART 7 — ROBOTS.TXT AND INDEXING HYGIENE
==================================================

1. Verify robots.txt is accessible and references the correct sitemap URL(s).
2. Ensure important public pages are crawlable.
3. Ensure dev/staging/test/admin-only URLs are not exposed to indexing unintentionally.
4. Ensure there are no accidental noindex tags on important brand pages.

==================================================
PART 8 — BRAND CONSOLIDATION SIGNALS
==================================================

1. Add a reusable sitewide footer/about snippet that consistently reinforces the exact brand name NurseNest.
2. Ensure logo alt text and key brand images use the exact spelling NurseNest.
3. Ensure homepage and key landing pages internally link back to the homepage with branded anchor text such as NurseNest.
4. Add or improve contact/about pages so Google can better understand the brand entity.

==================================================
PART 9 — OUTPUT REQUIRED
==================================================

After making changes, provide:
1. the chosen canonical homepage URL
2. all files changed
3. summary of canonical fixes
4. summary of hreflang fixes
5. summary of sitemap and robots fixes
6. updated homepage title, meta description, and H1
7. structured data added or updated
8. anything that still needs manual submission in Google Search Console
9. how to test that branded search signals are now consistent
