Fix multilingual SEO and incomplete translation issues across the entire NurseNest site.

Goal:
Every language route must be a fully translated, self-canonical, indexable page with proper hreflang linking, no accidental duplication, no mixed-language content leakage, and no /index.html duplicates. We need Google to recognize each locale page as a legitimate language version, not a duplicate of English.

Critical context:
Search Console is showing many URLs as “Alternative page with proper canonical tag.”
This suggests our current locale URLs are being clustered as alternates instead of being indexed independently.
A major likely cause is that some localized pages are only partially translated, and/or their canonical tags are pointing to the wrong version.

Implement all of the following exactly:

PART 1 — FULL TRANSLATION ENFORCEMENT
1. Create a site-wide i18n audit that scans every route and every translatable content field.
2. Detect untranslated or fallback English strings on non-English pages.
3. Detect mixed-language pages where the route locale does not match the visible content.
4. Detect untranslated UI elements including:
   - nav
   - footer
   - buttons
   - badges
   - breadcrumbs
   - lesson headers
   - rationale text
   - flashcard labels
   - test bank labels
   - exam UI
   - simulator UI
   - blog cards
   - pricing cards
   - FAQ
   - modals
   - validation messages
   - forms
   - empty states
   - error pages
   - dashboard labels
5. Detect untranslated database-driven content, not just hardcoded interface strings.
6. Detect untranslated lesson body sections, rationale text, flashcard text, question explanations, and blog content.
7. Generate an admin-only “Translation Coverage Dashboard” showing:
   - route
   - locale
   - translation completion percentage
   - untranslated field count
   - fallback-English field count
   - last updated date
   - publish readiness status
8. Add status labels:
   - Draft Translation
   - Partial Translation
   - Ready for Indexing
   - Hidden from Sitemap
9. Any non-English page below 95% translation completion must automatically be marked:
   - noindex
   - excluded from sitemap
   - not eligible for canonical self-indexing
10. Only pages with near-complete translation should be indexable.

PART 2 — CANONICAL RULES
11. Every indexable page must output exactly one canonical tag in the server-rendered HTML head.
12. Canonicals must be self-referencing for fully translated locale pages:
   - /en/page -> canonical /en/page
   - /es/page -> canonical /es/page
   - /fr/page -> canonical /fr/page
   - /ar/page -> canonical /ar/page
   etc.
13. Do not canonicalize translated pages to English unless the translated page is intentionally incomplete and non-indexable.
14. If a locale page is incomplete, either:
   - set noindex and remove from sitemap, or
   - redirect to the canonical source language page if that is the intended behavior
15. Do not let the frontend hydrate and replace canonical tags after render.
16. Ensure canonical in raw HTML matches canonical after rendering.

PART 3 — HREFLANG IMPLEMENTATION
17. Add hreflang tags for every localized equivalent, including self-reference.
18. Add x-default for the main language selector or default landing page if appropriate.
19. Ensure hreflang sets are bidirectional and complete.
20. Do not include hreflang entries for pages that are noindex or absent from sitemap.
21. Each locale page must point only to true equivalents, not loosely related pages.

PART 4 — URL NORMALIZATION
22. Remove /index.html duplication site-wide.
23. 301 redirect:
   - /en/index.html -> /en/
   - /es/index.html -> /es/
   - and same rule for all locales
24. Normalize trailing slashes consistently.
25. Ensure internal links always use the preferred canonical form only.
26. Update router helpers so no component ever generates duplicate route variants.

PART 5 — SITEMAP RULES
27. Generate sitemap entries only for canonical, indexable, fully translated URLs.
28. Exclude:
   - partial translations
   - noindex pages
   - duplicate route variants
   - /index.html pages
   - preview/test/admin-only pages
29. If using hreflang sitemap annotations, ensure they match page-level hreflang exactly.
30. Add a sitemap validation step in build/deploy.

PART 6 — CONTENT PARITY AND LOCALE INTEGRITY
31. Enforce that each localized route loads locale-specific content from the database or translation source.
32. Prevent silent fallback to English on production locale pages unless the page is noindex.
33. Add a “missing translation guard”:
   - if key content is missing for a locale, do not publish as indexable
34. For lessons, blogs, exam rationales, and study pages, ensure:
   - title translated
   - meta title translated
   - meta description translated
   - headings translated
   - body translated
   - CTA translated
   - schema fields translated where appropriate
35. Add locale-aware slug mapping where needed so translated pages are not just English content on a different URL.

PART 7 — SEO DEBUG TOOLING
36. Create an admin-only “SEO + Translation Inspector” per page showing:
   - current URL
   - locale
   - source language
   - canonical URL
   - robots status
   - hreflang set
   - sitemap inclusion
   - translation completion %
   - untranslated field list
   - raw HTML head preview
   - rendered head preview
37. Add a warning if:
   - canonical points to another language
   - page contains too much English fallback
   - hreflang is incomplete
   - page is in sitemap but translation is incomplete
   - route has duplicate variants

PART 8 — DEPLOYMENT SAFETY
38. Add a pre-deploy validation script that fails deployment if:
   - any indexable page has wrong canonical
   - any locale page marked indexable is under 95% translated
   - hreflang sets are broken
   - /index.html duplicates are still reachable
   - sitemap contains non-canonical URLs
39. Add structured logs for translation completeness and canonical generation.
40. Output a final audit report listing all affected routes and what was fixed.

PART 9 — WHAT I WANT TO SEE AFTER THIS CHANGE
41. Fully translated locale pages should be self-canonical and indexable.
42. Incomplete locale pages should not be indexed until translation is complete.
43. Search Console should gradually stop classifying true localized pages as alternates just because of mixed-language or duplicate signals.
44. All page templates must work for:
   - lessons
