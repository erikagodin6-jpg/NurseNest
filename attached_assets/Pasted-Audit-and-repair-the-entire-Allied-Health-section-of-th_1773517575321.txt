Audit and repair the entire Allied Health section of the NurseNest platform to eliminate broken routes and 404 errors.

There are currently multiple broken links across the Allied Health site, including the Respiratory Therapy (RRT) link that does not resolve correctly. The goal of this task is to locate every broken link, standardize route naming, repair navigation paths, and ensure that all public Allied Health pages load correctly both through navigation and direct URL entry.

OBJECTIVES

1. Identify all broken links across the Allied Health site.
2. Repair routing inconsistencies that produce 404 errors.
3. Standardize Allied Health URL slugs so they follow one consistent pattern.
4. Ensure all navigation components point to valid routes.
5. Ensure deep links work in production without returning 404 errors.
6. Add automatic detection tools to prevent future broken links.

ROUTE STRUCTURE STANDARDIZATION

All Allied Health routes should follow a consistent pattern:

/allied/paramedic
/allied/respiratory-therapy
/allied/medical-lab-tech
/allied/diagnostic-imaging
/allied/occupational-therapy
/allied/physical-therapy

Do not mix abbreviations like /rrt or /mlt in the public URLs unless they redirect to the canonical slug.

For example:
If /rrt exists, redirect it to:
/allied/respiratory-therapy

Ensure canonical URLs match the chosen slug structure.

BROKEN LINK DETECTION

Scan all Allied Health components for invalid links.

Check:

• navigation menus
• dropdown menus
• hero buttons
• CTA buttons
• feature cards
• program cards
• breadcrumbs
• footer links
• related content sections
• internal blog links
• sitemap links

Identify any links that resolve to 404 and correct them.

RRT LINK FIX

Locate the broken Respiratory Therapy link on the Allied Health page.

Determine whether the correct destination page should be:
/allied/respiratory-therapy

Update every link that points to an incorrect RRT route.

Ensure the Respiratory Therapy page exists and renders properly.

ROUTER CONFIGURATION

Verify the router configuration supports all Allied Health routes.

Ensure each page has a registered route and is not being overridden by broader wildcard routes.

Check that nested routes do not conflict with parent routes.

If using a client-side router such as Wouter or React Router, verify that all Allied Health routes are defined explicitly.

SERVER FALLBACK SUPPORT

Ensure the production server correctly handles deep links.

Direct navigation to routes such as:
/allied/paramedic
/allied/respiratory-therapy

must not return a 404.

If necessary, configure server fallback so all frontend routes resolve to index.html and are handled by the client router.

FOOTER AND SITEMAP CLEANUP

Verify that all Allied Health URLs in the footer and sitemap are valid.

Remove any sitemap entries that point to non-existent pages.

Ensure the sitemap only contains valid routes.

CARD AND TILE LINKS

Audit all cards on the Allied Health landing page and subpages.

Ensure every clickable card points to a valid destination.

Remove any placeholder routes.

REDIRECT STRATEGY

Where old URLs exist, add redirects instead of allowing them to return 404 errors.

Examples:

/rrt → redirect to /allied/respiratory-therapy
/mlt → redirect to /allied/medical-lab-tech

This prevents external links or bookmarks from breaking.

MOBILE NAVIGATION CHECK

Verify that mobile navigation menus contain the same valid links as the desktop navigation.

Often mobile menus have duplicated route definitions that may still reference outdated URLs.

BROKEN ROUTE DETECTION TOOL

Add a simple internal route validation utility that checks internal links against the router configuration.

This tool should detect:

• missing routes
• invalid internal links
• links pointing to 404 pages

It should run during development and optionally log warnings during production builds.

SEO AND CANONICAL FIXES

Ensure each Allied Health page includes:

• correct canonical URL
• correct page title
• correct meta description

Prevent search engines from indexing 404 pages.

VERIFICATION CHECKLIST

After implementing fixes confirm:

• RRT link works
• all Allied Health program links work
• no navigation links return 404
• direct URL entry works
• sitemap URLs resolve correctly
• canonical URLs match the route structure
• mobile and desktop navigation links both function correctly

SUMMARY REPORT

After completing the audit provide a short report including:

1. The root cause of the RRT broken link.
2. How many broken routes were found.

5. Confirmation that the Allied Health site is free of 404 navigation errors.

GOAL

The final result should be an Allied Health section where every navigation element, card, and route resolves correctly without producing 404 errors, and future route inconsistencies are prevented through automated validation.
