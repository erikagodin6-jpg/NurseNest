I need to refactor the entire NurseNest platform into a strict, production-ready role-gated architecture so every content type is separated correctly by tier and no user can see content outside their allowed role.

Current problem:
Role gating keeps breaking across the platform. Exams were leaking across tiers, lessons were moved into free, and the system is no longer reliably separating content by role. I need a single master architecture that enforces access control everywhere, not patch-by-patch.

GOAL:
Create a durable platform-wide role-gating system for all user-facing and admin-facing content.

Canonical roles:
- free
- rpn
- rn
- np
- admin

Canonical tiers:
- free
- rpn
- rn
- np

==================================================
PART 1 — SINGLE SOURCE OF TRUTH
==================================================

Create a centralized access control system.

Implement:
- roleAccessConfig.ts
- getAllowedTiersForRole(role)
- canAccessTier(role, tier)
- canAccessContent(user, contentItem)
- filterContentForRole(user, contentList)
- getDefaultTierForRole(role)
- getDefaultDashboardForRole(role)
- getDefaultRoutesForRole(role)

This must be the single source of truth used across frontend and backend.

Example default rules:
- free -> ["free"]
- rpn -> ["free", "rpn"]
- rn -> ["free", "rn"] by default, optional ["rpn"] review only if explicitly enabled
- np -> ["free", "np"] by default, optional ["rn"] review only if explicitly enabled
- admin -> ["free", "rpn", "rn", "np"]

Cross-tier visibility must never happen by accident.
If cross-tier review is allowed, it must be explicit in config.

PART 3 — ROLE ACCESS RULES
==================================================

Enforce strict visibility by role.

FREE:
- can see only free content and intentional free previews
- cannot access paid tier routes or APIs
- can see locked upgrade prompts only where intentionally designed

RPN:
- can see free + rpn content
- cannot see rn or np content
- cannot see rn or np titles in search, recommendations, or related content

RN:
- can see free + rn content
- cannot see np content
- cannot see rpn content unless explicit review mode is enabled in config

NP:
- can see free + np content
- cannot see rpn content by default
- cannot see rn content unless explicit review mode is enabled in config

ADMIN:
- can see all
- can filter all
- can preview the product as any role
